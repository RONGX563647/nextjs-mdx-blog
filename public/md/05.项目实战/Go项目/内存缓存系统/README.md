# 项目名称：基于 TCP 的分布式内存数据缓存系统（MiniDistCache）

## 一、项目核心定位

该项目是一个轻量级分布式内存缓存系统，基于 C 语言实现，**深度融合计算机组成原理（内存管理、缓存策略）、计算机网络（TCP 通信、协议解析）、基础数据结构（哈希表、链表、队列）** 三大领域知识。核心功能是通过 TCP 网络接收客户端请求，提供键值对（Key-Value）的缓存存储与查询服务，并通过高效内存管理和缓存淘汰策略优化性能，同时支持数据持久化与基础监控。

## 二、项目核心功能与扩展功能

### 1. 核心功能

- **TCP 客户端 - 服务端通信**：服务端监听端口，支持多客户端并发连接，处理缓存操作请求（SET/GET/DELETE/EXPIRE）。
- **内存缓存 CRUD**：实现键值对的新增（SET）、查询（GET）、删除（DELETE）、过期设置（EXPIRE）。
- **缓存淘汰策略**：支持 LRU（最近最少使用）、LFU（最不经常使用）两种淘汰算法，解决内存不足时的空间回收问题。
- **数据校验**：对传输的键值数据进行 CRC32 校验，避免网络传输错误。
- **基础日志**：记录客户端请求、缓存操作结果、错误信息（如内存分配失败、网络异常）。

### 2. 扩展功能

- **数据持久化**：定期将缓存数据写入本地文件（二进制 / 文本格式），服务重启后可恢复数据。
- **配置文件解析**：支持通过 ini 格式配置文件设置监听端口、缓存最大内存、淘汰策略、持久化间隔。
- **客户端命令行工具**：提供命令行客户端，支持手动输入缓存指令（如`./client SET name Alice`）。
- **超时连接管理**：通过定时器清理长时间无交互的客户端连接，释放资源。
- **简单监控**：统计缓存命中率、当前缓存键值对数量、已使用内存大小，通过`STAT`指令查询。

## 四、项目整体难度与工程量

### 1. 整体难度：中等

- **低难度技术**（8 项）：占比约 35%，如内存对齐、动态内存分配、日志系统、文件 I/O，主要依赖 C 语言基础 API。

1. **知识融合**：将计算机组成原理（内存管理）、计算机网络（TCP 通信）、数据结构（哈希表 / 链表 / 堆）的理论知识落地为可运行的系统，避免 “知识碎片化”。

## 代码说明

### 1. 核心模块组成

- **数据结构层**：实现哈希表（`HashTable`）用于快速查找，双向链表用于 LRU 策略维护访问顺序。
- **缓存逻辑层**：实现`lru_put`/`lru_get`/`lru_delete`核心操作，包含过期检查和 LRU 淘汰逻辑。
- **网络通信层**：基于 TCP 的服务端（多线程处理并发连接）和客户端（发送缓存命令）。
- **工具层**：日志系统（记录操作和错误）、内存管理（封装`malloc`带日志）、哈希函数（DJB2 算法）。

### 2. 编译与使用方法

#### 编译命令

bash

```bash
gcc minidistcache.c -o minidistcache -lpthread  # 需要链接pthread库（多线程支持）
```

#### 启动服务端

bash

```bash
./minidistcache  # 服务端启动，监听8888端口
```

#### 客户端操作示例

bash

```bash
# 设置缓存（key=name，value=Alice，300秒过期）
./minidistcache 127.0.0.1 "SET name Alice 300"

# 获取缓存
./minidistcache 127.0.0.1 "GET name"

# 删除缓存
./minidistcache 127.0.0.1 "DELETE name"
```

### 3. 功能限制与扩展方向

#### 现有限制

- 简化了网络协议（无消息头，不处理粘包），生产环境需完善。
- 仅实现 LRU 淘汰策略，未实现 LFU。
- 无数据持久化和配置文件解析。

#### 扩展方向

1. 网络层：添加消息头（4 字节长度）解决粘包问题，支持更复杂的协议。
2. 缓存层：实现 LFU 策略（添加访问计数和小根堆），支持缓存持久化到文件。
3. 并发层：使用线程池替代单线程创建，优化高并发性能。
4. 监控层：添加`STAT`命令，返回缓存命中率、内存使用等统计信息。
