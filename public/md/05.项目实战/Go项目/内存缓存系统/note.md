由于该项目完整实现代码量较大（约2000-3000行），以下提供**核心模块的简化实现代码**，包含TCP服务端/客户端通信、LRU缓存策略、哈希表存储等核心功能，可直接编译运行。代码保留关键逻辑，省略部分扩展功能（如LFU、持久化），适合作为基础版本学习。







## 代码说明

### 1. 核心模块组成
- **数据结构层**：实现哈希表（`HashTable`）用于快速查找，双向链表用于LRU策略维护访问顺序。
- **缓存逻辑层**：实现`lru_put`/`lru_get`/`lru_delete`核心操作，包含过期检查和LRU淘汰逻辑。
- **网络通信层**：基于TCP的服务端（多线程处理并发连接）和客户端（发送缓存命令）。
- **工具层**：日志系统（记录操作和错误）、内存管理（封装`malloc`带日志）、哈希函数（DJB2算法）。


### 2. 编译与使用方法
#### 编译命令
```bash
gcc minidistcache.c -o minidistcache -lpthread  # 需要链接pthread库（多线程支持）
```

#### 启动服务端
```bash
./minidistcache  # 服务端启动，监听8888端口
```

#### 客户端操作示例
```bash
# 设置缓存（key=name，value=Alice，300秒过期）
./minidistcache 127.0.0.1 "SET name Alice 300"

# 获取缓存
./minidistcache 127.0.0.1 "GET name"

# 删除缓存
./minidistcache 127.0.0.1 "DELETE name"
```


### 3. 功能限制与扩展方向
#### 现有限制
- 简化了网络协议（无消息头，不处理粘包），生产环境需完善。
- 仅实现LRU淘汰策略，未实现LFU。
- 无数据持久化和配置文件解析。

#### 扩展方向
1. 网络层：添加消息头（4字节长度）解决粘包问题，支持更复杂的协议。
2. 缓存层：实现LFU策略（添加访问计数和小根堆），支持缓存持久化到文件。
3. 并发层：使用线程池替代单线程创建，优化高并发性能。
4. 监控层：添加`STAT`命令，返回缓存命中率、内存使用等统计信息。


该代码完整实现了“基于TCP的内存缓存系统”的核心逻辑，融合了计算机网络（TCP编程）、计算机组成原理（内存管理、缓存策略）、数据结构（哈希表、双向链表）的基础知识，可作为学习和扩展的基础版本。
