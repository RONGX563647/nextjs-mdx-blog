```markdown
本文围绕 Selenium 的使用展开详细讲解，旨在解决因 Ajax 接口加密参数（如 token）难以直接爬取数据的问题，通过模拟浏览器操作实现 “可见即可爬”。以下是核心内容总结：
一、Selenium 核心价值
当网站 Ajax 接口含加密参数（如 token），难以直接模拟请求时，Selenium 可驱动浏览器执行操作（点击、输入等），直接获取浏览器渲染后的页面数据，适用于 JavaScript 动态渲染页面的爬取。
二、基础准备
环境配置：安装 Chrome 浏览器、对应版本的 ChromeDriver，以及 Python 的 Selenium 库。
核心对象：通过webdriver.Chrome()等方法声明浏览器对象（支持 Chrome、Firefox、Edge 等），后续操作基于此对象执行。
三、核心功能与用法
1. 页面访问与基础操作
用get(url)访问网页，通过page_source获取页面源码、current_url获取当前 URL、get_cookies()获取 Cookies。
示例：打开百度并搜索关键词，获取搜索结果页信息。
2. 节点定位
单个节点：通过find_element_by_id()、find_element_by_css_selector()、find_element_by_xpath()等方法，或通用的find_element(By.方式, 值)定位，返回WebElement对象。
多个节点：对应方法加s（如find_elements_by_css_selector()），返回包含多个WebElement的列表。
3. 节点交互与动作链
节点交互：通过send_keys()输入文本、clear()清空、click()点击按钮等。
动作链：无特定执行对象的操作（如拖拽），用ActionChains类，示例：将元素从一处拖拽到另一处。
4. 执行 JavaScript
通过execute_script(js代码)实现 API 未覆盖的功能，如下拉进度条（window.scrollTo(0, document.body.scrollHeight)）、弹出提示框等。
5. 节点信息提取
属性：get_attribute(属性名)获取节点属性（如src）。
文本：text属性获取节点内部文本。
其他信息：id（节点 ID）、location（位置）、tag_name（标签名）、size（宽高）。
6. Frame 切换
页面含子 Frame 时，需用switch_to.frame(Frame名/索引)切换到子 Frame；返回父 Frame 用switch_to.parent_frame()，否则无法定位子 Frame 内节点。
7. 延时等待
隐式等待：implicitly_wait(秒数)，设置全局等待时间，未找到节点时等待至超时。
显式等待：WebDriverWait(browser, 秒数).until(条件)，针对特定节点设置等待（如presence_of_element_located节点出现、element_to_be_clickable节点可点击）。
8. 浏览器控制
前进后退：forward()前进、back()后退。
选项卡管理：用window.open()新建选项卡，window_handles获取所有选项卡代号，switch_to.window(代号)切换选项卡。
Cookies 操作：get_cookies()获取、add_cookie(字典)添加、delete_all_cookies()删除所有 Cookies。
9. 异常处理
捕获常见异常（如TimeoutException超时、NoSuchElementException节点未找到），用try-except避免程序中断。
10. 反屏蔽与无头模式
反屏蔽：网站可能检测navigator.webdriver等特征，通过 CDP 命令在页面加载前置空该属性，配合ChromeOptions隐藏自动化标识，示例：
python
运行
option.add_experimental_option('excludeSwitches', ['enable-automation'])
browser.execute_cdp_cmd('Page.addScriptToEvaluateOnNewDocument', {'source': 'Object.defineProperty(navigator, "webdriver", {get: () => undefined})'})


无头模式：无界面运行，节省资源，用ChromeOptions().add_argument('--headless')开启，建议设置窗口大小（避免默认尺寸导致元素定位问题）。
四、常见问题与注意事项
淘宝等网站反爬严格，可能需要进一步隐藏浏览器指纹或处理登录、验证码。
无头模式默认窗口较小，需手动设置尺寸以匹配正常浏览场景。
频繁操作可能触发验证码，需结合行为模拟（如随机延迟）规避。
通过以上内容，可掌握 Selenium 从基础到进阶的用法，解决动态渲染页面和加密接口的爬取问题。
```
