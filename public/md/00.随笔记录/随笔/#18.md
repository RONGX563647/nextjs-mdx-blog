## 1.python自动化脚本

```markdown
我用python selenium做了一个自动化脚本用来填写问卷星的问卷 但是我发现填完提交了却显示没有作答这是为什么呢
使用 Selenium 自动填写问卷星问卷后提交却显示未作答，可能由以下原因导致：
1. 页面元素加载延迟
问卷星可能使用了异步加载（如 AJAX），元素虽已显示但实际未完全初始化。
解决方案：
使用显式等待确保元素可交互：
python
运行
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# 等待单选按钮可点击
radio_button = WebDriverWait(driver, 10).until(
    EC.element_to_be_clickable((By.ID, "radio_option_id"))
)
radio_button.click()



2. 表单验证机制未触发
问卷星可能有前端验证逻辑（如必填项检查、JavaScript 校验），而自动化操作未触发这些验证。
解决方案：
模拟真实用户操作，例如填写完输入框后按Tab键或触发change事件：
python
运行
from selenium.webdriver.common.keys import Keys

input_element = driver.find_element(By.ID, "input_id")
input_element.send_keys("答案内容")
input_element.send_keys(Keys.TAB)  # 触发验证

3. 提交方式不正确
问卷星可能使用 JavaScript 或 AJAX 处理提交，简单点击按钮可能无法完成真正的提交。
解决方案：
使用 JavaScript 执行提交：
python
运行
submit_button = driver.find_element(By.ID, "submit_button_id")
driver.execute_script("arguments[0].click();", submit_button)

检查是否有二次确认弹窗，若有则处理弹窗：
python
运行
try:
    confirm_button = WebDriverWait(driver, 5).until(
        EC.element_to_be_clickable((By.ID, "confirm_button_id"))
    )
    confirm_button.click()
except:
    pass

4. 反爬虫机制限制
问卷星可能检测到自动化操作（如异常的鼠标移动轨迹、请求频率过快）。
解决方案：
添加随机延迟模拟人工操作：
python
运行
import time
import random

time.sleep(random.uniform(1, 3))  # 随机等待1-3秒

使用无头浏览器时添加浏览器指纹：
python
运行
from selenium import webdriver

options = webdriver.ChromeOptions()
options.add_argument("--disable-blink-features=AutomationControlled")
options.add_argument("user-agent=Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36...")
driver = webdriver.Chrome(options=options)

5. 验证码或滑块验证
问卷星可能在提交时要求人机验证（尤其是频繁提交时）。
解决方案：
手动处理验证码：
python
运行
# 遇到验证码时暂停，人工干预
input("请手动完成验证码，完成后按Enter继续...")

复杂验证码可使用第三方服务（如打码平台），但需付费。
6. 表单数据未正确填充
某些选项可能需要特殊处理（如下拉选择、复选框组）。
解决方案：
检查元素结构，确保正确选择：
python
运行
# 下拉选择示例
from selenium.webdriver.support.ui import Select

select_element = Select(driver.find_element(By.ID, "dropdown_id"))
select_element.select_by_visible_text("选项文本")

7. 会话或 Cookie 问题
问卷星可能依赖会话状态或 Cookie 跟踪作答。
解决方案：
确保使用同一个浏览器实例，避免中途关闭：
python
运行
# 保持driver实例的连续性
driver = webdriver.Chrome()
# 完成所有操作后再关闭
driver.quit()

调试建议
打印页面源代码：提交后获取页面源码，检查是否有错误提示：
python
运行
print(driver.page_source)

使用开发者工具：手动操作问卷星，通过浏览器开发者工具（如 Chrome 的 Network 面板）观察真实请求和响应，对比自动化脚本的行为。
分步测试：逐个验证每个问题的填写和提交，定位具体失效步骤。
如果问题仍然存在，可能是问卷星针对自动化操作加强了防刷机制，建议减少提交频率或改用人工方式
```
