```markdown
### 核心技术快速回顾  
1. **Tomcat**  
   - 开源Servlet容器，核心组件：Connector（接收请求）、Engine（调度请求）、Host（虚拟主机）、Context（Web应用）。  
   - 支持BIO/NIO线程模型，默认NIO适合高并发，Tomcat 10+适配Jakarta EE，需注意包名兼容性。  

2. **Spring Boot Web**  
   - 基于Spring MVC，通过自动配置和Starter简化开发，内置Tomcat/Jetty等容器。  
   - 核心特性：注解驱动（@RestController）、响应式编程（WebFlux）、Actuator监控，适合快速构建RESTful API和微服务。  

3. **Spring MVC**  
   - MVC框架，流程：DispatcherServlet接收请求→HandlerMapping路由→Controller处理→ViewResolver渲染响应。  
   - 关键注解：@Controller、@RequestMapping、@RequestBody，与Spring Boot集成后自动配置，提升开发效率。  

4. **HTTP协议**  
   - 应用层协议，基于请求-响应模型，无状态，默认端口80（HTTPS 443）。  
   - 核心要素：请求/响应行、头、体；状态码（2xx成功，4xx客户端错误）；HTTP/2多路复用优化性能，HTTPS加密传输。  

5. **Java请求响应**  
   - 客户端：Java 11+ HttpClient支持同步/异步请求，OkHttp高效处理HTTP/2；服务端：Servlet/Spring MVC处理请求，返回ModelAndView或JSON数据。  
   - 关键：异常处理（try-with-resources）、JSON解析（Jackson）、异步编程（CompletableFuture）。  

6. **架构与工具**  
   - **Postman**：API调试工具，支持请求模拟、断言、集合管理，适合接口联调与文档生成。  
   - **三层架构**：表现层（UI）、业务层（逻辑）、数据层（存储），解耦提升可维护性，需注意单向依赖。  
   - **IoC（控制反转）**：Spring通过容器管理Bean依赖，依赖注入（构造器/Setter）降低耦合，提升可测试性。
```

~~~markdown
MyBatis 是一个流行的 Java 持久层框架，它将 SQL 查询与 Java 对象映射，简化了数据库操作。下面从核心概念、工作流程到实际应用进行全面讲解：


### **1. 核心概念**
#### **1.1 ORM（对象关系映射）**
- **作用**：将数据库表与 Java 对象绑定，避免手动编写 JDBC 代码。
- **MyBatis 特点**：半自动化 ORM（需手动编写 SQL，但可灵活控制查询）。

#### **1.2 核心组件**
- **SqlSession**：执行 SQL 的核心接口，类似 JDBC 的 `Connection`。
- **Mapper 接口**：定义数据库操作的方法（如 `UserMapper.java`）。
- **映射文件**：XML 或注解形式，定义 SQL 语句与对象的映射关系。
- **Configuration**：全局配置，包含数据源、插件等信息。


### **2. 工作流程**
1. **配置加载**：读取 `mybatis-config.xml` 和映射文件（或注解）。
2. **创建 SqlSessionFactory**：通过配置构建工厂。
3. **获取 SqlSession**：从工厂获取会话实例。
4. **执行 SQL**：通过 Mapper 接口或直接执行 SQL。
5. **结果映射**：将查询结果自动映射到 Java 对象。


### **3. 快速上手**
#### **3.1 依赖配置（Maven）**
```xml
<dependency>
    <groupId>org.mybatis</groupId>
    <artifactId>mybatis</artifactId>
    <version>3.5.7</version>
</dependency>
```

#### **3.2 配置文件 `mybatis-config.xml`**
```xml
<?xml version="1.0" encoding="UTF-8" ?>
<configuration>
  <environments default="development">
    <environment id="development">
      <transactionManager type="JDBC"/>
      <dataSource type="POOLED">
        <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://localhost:3306/test"/>
        <property name="username" value="root"/>
        <property name="password" value="password"/>
      </dataSource>
    </environment>
  </environments>
  <mappers>
    <mapper resource="UserMapper.xml"/>
  </mappers>
</configuration>
```

#### **3.3 映射文件 `UserMapper.xml`**
```xml
<mapper namespace="com.example.mapper.UserMapper">
  <select id="selectUser" parameterType="int" resultType="User">
    SELECT * FROM users WHERE id = #{id}
  </select>
</mapper>
```

#### **3.4 Java 代码**
```java
// 1. 加载配置
InputStream inputStream = Resources.getResourceAsStream("mybatis-config.xml");
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

// 2. 获取 SqlSession 并执行查询
try (SqlSession session = sqlSessionFactory.openSession()) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    User user = mapper.selectUser(1);
}
```


### **4. 高级特性**
#### **4.1 动态 SQL**
- **标签**：`<if>`, `<where>`, `<foreach>`, `<set>` 等。
- **示例**：
  ```xml
  <select id="findUsers" resultType="User">
    SELECT * FROM users
    <where>
      <if test="name != null">
        AND name = #{name}
      </if>
      <if test="age != null">
        AND age > #{age}
      </if>
    </where>
  </select>
  ```

#### **4.2 关联映射**
- **一对一**：通过 `<association>` 标签。
- **一对多**：通过 `<collection>` 标签。

#### **4.3 缓存机制**
- **一级缓存**：基于 SqlSession 的本地缓存。
- **二级缓存**：全局缓存，需手动配置（如 Ehcache、Redis）。


### **5. 最佳实践**
1. **使用注解替代 XML**（简单查询）：
   ```java
   @Select("SELECT * FROM users WHERE id = #{id}")
   User selectUser(int id);
   ```

2. **分页插件**：集成 PageHelper 实现分页查询。

3. **事务管理**：
   ```java
   try (SqlSession session = sqlSessionFactory.openSession(false)) {
       // 手动提交事务
       session.commit();
   }
   ```

4. **插件机制**：自定义拦截器（如 SQL 性能监控）。


### **6. 与 Spring 集成**
#### **6.1 依赖**
```xml
<dependency>
    <groupId>org.mybatis.spring</groupId>
    <artifactId>mybatis-spring</artifactId>
    <version>2.0.6</version>
</dependency>
```

#### **6.2 配置**
```java
@Configuration
public class MyBatisConfig {
    @Bean
    public SqlSessionFactory sqlSessionFactory(DataSource dataSource) throws Exception {
        SqlSessionFactoryBean factoryBean = new SqlSessionFactoryBean();
        factoryBean.setDataSource(dataSource);
        factoryBean.setMapperLocations(new PathMatchingResourcePatternResolver()
                .getResources("classpath:mapper/*.xml"));
        return factoryBean.getObject();
    }
}
```


### **7. 优缺点**
#### **优点**
- 灵活控制 SQL，适合复杂查询。
- 性能优化空间大。
- 学习曲线平缓。

#### **缺点**
- 大量 SQL 需手动编写。
- 复杂关联查询处理较繁琐。


### **8. 常见问题**
1. **N+1 查询问题**：通过 `<fetchType="eager">` 或批量查询解决。
2. **类型转换**：使用 `TypeHandler` 自定义类型映射。
3. **SQL 注入**：使用 `#{}` 而非 `${}` 占位符。


### **总结**
MyBatis 适合需要精细控制 SQL 的项目，尤其在处理复杂查询时优势明显。通过合理配置和最佳实践，可以高效地实现数据库访问层。建议结合 Spring Boot 和 MyBatis-Plus 进一步简化开发。
~~~

> MyBatis 是 Java 持久层框架，实现对象与数据库表的映射。核心组件有 SqlSession、Mapper 接口等，工作流程包括配置加载、获取会话、执行 SQL 及结果映射。具备动态 SQL、关联映射、缓存等高级特性。优点是灵活、性能易优化、学习成本低；缺点是需手写大量 SQL，复杂关联处理麻烦。常与 Spring 集成，使用时需注意 N+1 查询、类型转换、SQL 注入等问题 。
