# 【Java +AI ｜基础篇day1】

## 一、Java 概述

### 1. Java 的核心定义与发展

- **本质**：1995 年由美国 Sun 公司（Stanford University Network）推出的高级编程语言，2009 年 Sun 被 Oracle（甲骨文）收购。
- **核心人物**：联合创始人詹姆斯・高斯林（James Gosling），被誉为 “Java 之父”。
- **行业地位**：世界最流行的编程语言之一，国内企业使用最广泛，开发社区完善、功能丰富。
- **核心特性**：可移植性强（跨平台）、安全可靠、性能优良，支持多场景开发。

### 2. Java 的三大技术平台

| 平台名称 | 英文全称                      | 核心定位             | 应用场景                       |
| -------- | ----------------------------- | -------------------- | ------------------------------ |
| 标准版   | Java SE（Standard Edition）   | Java 技术核心与基础  | 桌面应用、控制台程序开发       |
| 企业版   | Java EE（Enterprise Edition） | 企业级应用解决方案   | 服务器端开发（电商、金融）     |
| 小型版   | Java ME（Micro Edition）      | 移动设备应用解决方案 | 早期移动端开发（已逐步被替代） |

### 3. Java 的核心能力

- 主要应用：企业服务器端开发（核心场景）、大数据与云计算、桌面应用开发、游戏开发、科学计算等。
- 跨平台原理：Java 程序经`javac`编译为`.class`字节码文件，通过不同操作系统的 JVM（Java 虚拟机）解释执行，实现 “一次编译，处处可用”。

## 二、Java 开发环境搭建

### 1. 核心工具：JDK

- **定义**：Java Development Kit（Java 开发者工具包），是使用 Java 的基础，包含 JRE（运行环境）、JVM（虚拟机）、核心类库及开发工具（`javac`、`java`等）。
- **版本选择**：优先选择 LTS（长期支持版），主流版本为 JDK 8、11、17、21，多数企业仍在使用 JDK 8/11。
- **下载渠道**：Oracle 官方网站（[www.oracle.com](https://www.doubao.com/chat/www.oracle.com)），建议从官方获取安全稳定版本。

### 2. 安装与验证

#### （1）安装步骤

1. 下载对应操作系统（Windows/Mac/Linux）的 JDK 安装包。
2. 运行安装程序，选择安装路径（建议路径无中文、空格）。
3. 配置环境变量（部分新版本 JDK 自动配置，老版本需手动设置）。

#### （2）环境变量配置

- 核心配置项

  ：

  1. `JAVA_HOME`：指向 JDK 安装根目录（如`D:\develop\java\jdk-21`），供其他技术（如 Tomcat）识别 JDK 位置。
  2. `PATH`：添加`%JAVA_HOME%\bin`（Windows）或`$JAVA_HOME/bin`（Mac/Linux），确保在任意目录可执行`javac`、`java`命令。

- **配置路径（Windows）**：此电脑→属性→高级系统设置→环境变量→系统变量。

#### （3）验证安装成功

1. 按下`Win+R`，输入`cmd`打开命令行。
2. 输入`java -version`和`javac -version`，若显示版本号则安装成功。

### 3. 开发工具选择

- 基础工具：文本编辑器（Notepad++、Visual Studio Code）+ 命令行（适合入门熟悉流程）。
- 集成开发环境（IDE）：IntelliJ IDEA、Eclipse、NetBeans，提供代码高亮、自动补全、调试功能，推荐初学者使用 IntelliJ IDEA。

## 三、第一个 Java 程序（HelloWorld）

### 1. 开发流程（文本编辑器 + 命令行）

#### （1）编写代码

- 创建文本文件，命名为`HelloWorld.java`（文件名必须与类名一致，后缀为`.java`）。

- 输入代码：

  java

  运行

  ```java
  public class HelloWorld {
      public static void main(String[] args) {
          System.out.println("Hello, World!");
      }
  }
  ```

  

- 注意事项：文件名与类名一致（大小写敏感），使用英文半角符号，代码缩进规范。

#### （2）编译代码

- 打开命令行，切换到`HelloWorld.java`所在目录。
- 执行命令：`javac HelloWorld.java`（`javac`为编译工具，将`.java`源文件转为`.class`字节码文件）。
- 编译成功后，目录下会生成`HelloWorld.class`文件。

#### （3）运行程序

- 执行命令：`java HelloWorld`（`java`为执行工具，无需加`.class`后缀）。
- 成功输出：`Hello, World!`，标志程序运行正常。

### 2. 核心命令说明

| 命令    | 作用                              | 格式示例              |
| ------- | --------------------------------- | --------------------- |
| `javac` | Java 编译器，将源文件编译为字节码 | `javac 文件名.java`   |
| `java`  | Java 解释器，执行字节码文件       | `java 类名`（无后缀） |

### 3. 常见问题

- 文件名与类名不一致：编译报错 “找不到类”。
- 环境变量未配置：命令行提示 “不是内部或外部命令”。
- 符号使用中文全角：编译报错 “非法字符”。

## 四、Java 基础语法

### 1. 注释（代码解释说明）

#### （1）三种注释类型

| 注释类型 | 语法格式          | 适用场景     | 特点                                  |
| -------- | ----------------- | ------------ | ------------------------------------- |
| 单行注释 | `// 注释内容`     | 单行解释     | 仅作用于当前行，编译后删除            |
| 多行注释 | `/* 注释内容 */`  | 多行解释     | 不可嵌套，编译后删除                  |
| 文档注释 | `/** 注释内容 */` | 类、方法说明 | 可通过`javadoc`生成文档，保留在文档中 |

#### （2）示例代码

java

```java
/**
 * 文档注释：HelloWorld入门程序
 * 功能：输出问候语
 */
public class CommentDemo {
    public static void main(String[] args) {
        // 单行注释：输出HelloWorld
        System.out.println("HelloWorld");
        
        /* 多行注释：
           这是多行解释
           用于说明代码块功能
        */
        System.out.println("Java注释示例");
    }
}
```

#### （3）核心特点：注释不影响程序执行，编译后的`.class`文件中无注释内容。

### 2. 字面量（直接书写的数据）

#### （1）定义：程序中可直接使用的数据，需遵循固定书写格式。

#### （2）分类与书写规则

| 数据类型 | 生活写法       | 程序写法                       | 说明                     |
| -------- | -------------- | ------------------------------ | ------------------------ |
| 整数     | 666、-88       | 666、-88                       | 与生活写法一致           |
| 小数     | 13.14、-5.21   | 13.14、-5.21                   | 与生活写法一致           |
| 字符     | A、0、我       | `'A'`、`'0'`、`'我'`           | 单引号包裹，仅含一个字符 |
| 字符串   | 黑马程序员     | `"HelloWorld"`、`"黑马程序员"` | 双引号包裹，内容可空     |
| 布尔值   | 真、假         | `true`、`false`                | 仅两个取值               |
| 空值     | -              | `null`                         | 特殊值，代表无数据       |
| 特殊字符 | 制表符、换行符 | `\t`（制表符）、`\n`（换行）   | 转义字符，用于格式化输出 |

### 3. 变量（内存中的数据存储区域）

#### （1）核心概念

- 本质：内存中的一块区域（类似 “盒子”），用于存储程序处理的数据。
- 作用：使代码更灵活，便于数据管理和修改。
- 特点：存储的数据可替换（覆盖赋值）。

#### （2）定义格式

java

```java
数据类型 变量名称 = 初始值; // 完整定义（声明+赋值）
数据类型 变量名称; // 仅声明，使用前必须赋值
```

- 示例：`int age = 18;`（定义整数变量`age`，初始值为 18）、`String name = "Java";`（定义字符串变量`name`）。

#### （3）使用注意事项

1. 先声明后使用，未声明直接使用会报错。
2. 变量类型与存储数据类型必须一致（如`int`不能存小数）。
3. 作用域：从定义处到所在代码块（`{}`）结束，同一作用域内变量名不能重复。
4. 声明时可不赋值，但使用前必须有值（否则编译报错）。

#### （4）变量命名规范

- 首字母小写，遵循 “驼峰模式”（如`studyNumber`、`userName`）。
- 名称有意义，避免使用`a`、`b`等无意义字符。
- 由字母、数字、下划线（`_`）、美元符（`$`）组成，不能以数字开头，不能是关键字。

### 4. 数据存储原理

#### （1）二进制基础

- 计算机底层存储：所有数据均以二进制（0、1）存储，遵循 “逢 2 进 1” 规则。
- 进制转换：
  - 十进制转二进制：除二取余法（如 13→1101）。
  - 二进制转十进制：按位加权求和（如 1101→1×2³+1×2²+0×2¹+1×2⁰=13）。

#### （2）存储单位

- 最小单位：比特（bit，简称 b），表示 1 位二进制数（0 或 1）。
- 基本单位：字节（byte，简称 B），1B=8b。
- 常用单位：1KB=1024B、1MB=1024KB、1GB=1024MB、1TB=1024GB。

#### （3）不同数据的存储方式

- 字符数据：通过 ASCII 编码表映射为数字，再存储其二进制（如 'A'→65→01000001）。
  - 关键编码：'A'=65、'a'=97、'0'=48。
- 图片数据：由像素点组成，每个像素点的颜色通过 0~255×255×255 的数值表示，存储其二进制。
- 音视频数据：通过采样、编码转换为二进制流存储。

#### （4）进制表示（Java 支持）

| 进制类型 | 前缀标识 | 示例              |
| -------- | -------- | ----------------- |
| 二进制   | 0B/0b    | 0B1101（表示 13） |
| 八进制   | 0        | 015（表示 13）    |
| 十六进制 | 0X/0x    | 0xD（表示 13）    |

### 5. 基本数据类型（4 大类 8 种）

#### （1）分类与特性

| 数据类型大类   | 具体类型       | 内存占用 | 数据范围                                 | 默认值   |
| -------------- | -------------- | -------- | ---------------------------------------- | -------- |
| 整型           | byte           | 1 字节   | -128~127                                 | 0        |
|                | short          | 2 字节   | -32768~32767                             | 0        |
|                | int（默认）    | 4 字节   | -2147483648~2147483647（约 21 亿）       | 0        |
|                | long           | 8 字节   | -9223372036854775808~9223372036854775807 | 0L       |
| 浮点型（小数） | float          | 4 字节   | 1.401298E-45~3.4028235E+38               | 0.0F     |
|                | double（默认） | 8 字节   | 4.9000000E-324~1.797693E+308             | 0.0D     |
| 字符型         | char           | 2 字节   | 0~65535（支持中文）                      | '\u0000' |
| 布尔型         | boolean        | 1 字节   | true（真）、false（假）                  | false    |

#### （2）使用说明

- 整数默认类型为`int`，声明`long`类型需加后缀`L/l`（如`long num = 100L`）。
- 小数默认类型为`double`，声明`float`类型需加后缀`F/f`（如`float price = 13.14F`）。
- `char`类型可存储单个中文（如`char c = '我'`），因占用 2 字节，支持 Unicode 编码。

### 6. 关键字与标识符

#### （1）关键字

- 定义：Java 语言预定义的具有特殊含义的单词，用于表示语法结构、数据类型等。
- 核心特点：不能用作类名、变量名、方法名等标识符，否则编译报错。
- 常见关键字：`public`、`class`、`int`、`double`、`if`、`for`、`return`、`void`等（无需刻意记忆，使用时避免即可）。

#### （2）标识符

- 定义：程序员为类、变量、方法、包等自定义的名称（本质是 “名字”）。
- 命名规则：
  1. 组成：字母、数字、下划线（`_`）、美元符（`$`）。
  2. 开头：不能以数字开头。
  3. 禁忌：不能是关键字，不能包含特殊字符（&、%、# 等）。
  4. 大小写敏感：`myVariable`与`myvariable`是两个不同标识符。
- 命名规范（建议）：
  - 类名：首字母大写，驼峰模式（如`HelloWorld`、`Student`）。
  - 变量名 / 方法名：首字母小写，驼峰模式（如`userAge`、`calculateSum`）。

## 五、核心小结

1. Java 核心优势：跨平台、安全可靠、生态完善，核心应用于企业服务器端开发。
2. 开发流程：安装 JDK→配置环境变量→编写代码→编译（`javac`）→运行（`java`）。
3. 语法核心：注释（3 种类型）、字面量（固定书写格式）、变量（先声明后使用）、基本数据类型（4 大类 8 种）。
4. 存储本质：所有数据底层均为二进制，通过编码、进制转换实现不同类型数据的存储。
5. 命名规范：遵循驼峰模式，名称有意义，避免关键字和特殊字符。

# 参考答案

## 面试题 1 参考答案

1. 核心原理：Java 通过 “一次编译，处处可用” 实现跨平台，核心是**JVM（Java 虚拟机）** —— 不同操作系统（Windows、Linux、Mac）提供对应的 JVM，JVM 负责将 Java 编译后的字节码文件（.class）解释为当前系统可执行的机器指令，屏蔽了底层操作系统的差异。

2. 完整流程：

   

   （1）编写代码：使用文本编辑器或 IDE 创建.java 源文件，编写 Java 代码；

   

   （2）编译：通过

   ```
   javac
   ```

   命令将.java 源文件编译为字节码文件（.class）；

   

   （3）运行：通过

   ```
   java
   ```

   命令启动 JVM，JVM 加载并解释执行.class 文件，输出运行结果。

## 面试题 2 参考答案

1. 核心作用：

   - JVM（Java Virtual Machine）：Java 虚拟机，是运行 Java 字节码的核心，负责将字节码解释为机器指令，实现跨平台；
   - JRE（Java Runtime Environment）：Java 运行环境，包含 JVM 和核心类库（如 String、System 等），能运行已编译的 Java 程序，但无编译工具；
   - JDK（Java Development Kit）：Java 开发工具包，包含 JRE、编译工具（javac）、运行工具（java）等开发所需工具，是开发 Java 程序的基础。

2. 包含关系：JDK ⊇ JRE ⊇ JVM；

   

   开发需安装 JDK 的原因：开发过程中需要使用

   ```
   javac
   ```

   等编译工具将.java 源文件转为字节码，而 JRE 仅能运行程序，无编译功能，因此必须安装 JDK。

## 面试题 3 参考答案

1. 错误及原因：
   - `int 1age = 20;`：变量名以数字开头，违反标识符命名规则（标识符不能以数字开头）；
   - `long price = 10000000000;`：10000000000 默认是 int 类型，超出 int 的取值范围（-2147483648~2147483647），需加 L 后缀标识为 long 类型；
   - `float score = 95.5;`：95.5 默认是 double 类型，不能直接赋值给 float 类型，需加 F 后缀标识为 float 类型；
   - `char c = "A";`：char 类型需用单引号包裹（仅含一个字符），双引号是字符串的标识，违反字符字面量书写规则。
2. 修正后代码：

java



运行

```java
int age1 = 20;
long price = 10000000000L;
float score = 95.5F;
char c = 'A';
```

## 面试题 5 参考答案

1. 存在的问题：

   - 文件名与公共类名不一致：Java 规定，公共类（public 修饰的类）的类名必须与文件名完全一致（大小写敏感），小明的文件名为 hello.java，类名为 HelloWorld，导致编译报错；
   - 代码语法错误：`System.out.println("Hello World")`末尾缺少分号（;），Java 语句必须以分号结束。

2. 完整修正步骤：

   

   （1）修改文件名：将 hello.java 改为 HelloWorld.java（与公共类名一致）；

   

   （2）修正代码：在

   ```
   System.out.println("Hello World")
   ```

   末尾添加分号，修正后代码：

java

运行

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
```

（3）编译：打开命令行，切换到 HelloWorld.java 所在目录，执行命令`javac HelloWorld.java`，编译成功后生成 HelloWorld.class 文件；

（4）运行：在同一命令行执行命令`java HelloWorld`，成功输出`Hello World`。
