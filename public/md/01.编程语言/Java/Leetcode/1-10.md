# LeetCode Hot100 第一题：两数之和

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
你可以按任意顺序返回答案。

## 解法一：暴力枚举法

### 解题思路

- 双重循环遍历数组，外层循环遍历每个元素，内层循环寻找与当前元素之和等于目标值的元素
- 找到后返回两个元素的索引

### 代码实现

两数之和的暴力枚举解法

V1

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 外层循环遍历每个元素
        for (int i = 0; i < nums.length; i++) {
            // 内层循环查找与当前元素匹配的元素
            for (int j = i + 1; j < nums.length; j++) {
                // 检查两数之和是否等于目标值
                if (nums[i] + nums[j] == target) {
                    return new int[] {i, j};
                }
            }
        }
        // 根据题目要求，输入一定有解，所以这里只是为了满足语法要求
        return new int[] {-1, -1};
    }
}
```

- **时间复杂度**：O (n²)，其中 n 是数组的长度。需要双重循环遍历数组。
- **空间复杂度**：O (1)，只使用了常数级别的额外空间。

### 优缺点

- **优点**：实现简单，不需要额外的数据结构
- **缺点**：时间复杂度高，在数组较大时效率低下
- **适用场景**：数组规模较小的情况

## 解法二：哈希表一次遍历法

### 解题思路

- 使用哈希表（HashMap）存储已经遍历过的元素及其索引
- 对于每个元素，计算目标值与当前元素的差值
- 检查差值是否在哈希表中，如果存在则返回两个元素的索引
- 如果不存在，则将当前元素及其索引存入哈希表

### 代码实现

```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 创建哈希表存储元素值和对应的索引
        Map<Integer, Integer> map = new HashMap<>();
        
        // 遍历数组
        for (int i = 0; i < nums.length; i++) {
            // 计算需要找到的目标差值
            int complement = target - nums[i];
            
            // 检查差值是否已经在哈希表中
            if (map.containsKey(complement)) {
                // 如果存在，返回两个元素的索引
                return new int[] {map.get(complement), i};
            }
            
            // 如果不存在，将当前元素和索引存入哈希表
            map.put(nums[i], i);
        }
        
        // 根据题目要求，输入一定有解，所以这里只是为了满足语法要求
        return new int[] {-1, -1};
    }
}
```

## 解法三：排序 + 双指针法

### 解题思路

- 首先创建一个包含原始值和索引的对象数组
- 对该数组按照值进行排序
- 使用双指针，一个指向数组开头，一个指向数组结尾
- 根据两个指针所指元素的和与目标值的大小关系，移动指针
- 找到符合条件的元素后，返回它们的原始索引

### 代码实现

```java
import java.util.Arrays;
import java.util.Comparator;

class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 创建一个包含值和原始索引的数组
        int[][] valueIndex = new int[nums.length][2];
        for (int i = 0; i < nums.length; i++) {
            valueIndex[i][0] = nums[i];  // 存储值
            valueIndex[i][1] = i;        // 存储原始索引
        }
        
        // 按照值对数组进行排序
        Arrays.sort(valueIndex, Comparator.comparingInt(a -> a[0]));
        
        // 初始化双指针
        int left = 0;
        int right = nums.length - 1;
        
        // 使用双指针查找目标值
        while (left < right) {
            int sum = valueIndex[left][0] + valueIndex[right][0];
            
            if (sum == target) {
                // 找到目标，返回原始索引
                return new int[] {valueIndex[left][1], valueIndex[right][1]};
            } else if (sum < target) {
                // 和小于目标值，移动左指针增大和
                left++;
            } else {
                // 和大于目标值，移动右指针减小和
                right--;
            }
        }
        
        // 根据题目要求，输入一定有解，所以这里只是为了满足语法要求
        return new int[] {-1, -1};
    }
}
```

### 复杂度分析

- **时间复杂度**：O (n log n)，其中 n 是数组的长度。排序操作占主导地位，双指针遍历是 O (n)。
- **空间复杂度**：O (n)，需要额外的空间存储值和索引的对应关系。

### 优缺点

- **优点**：时间复杂度介于暴力法和哈希表法之间，不需要处理哈希冲突
- **缺点**：需要排序，实现相对复杂，且需要额外空间存储原始索引
- **适用场景**：当内存有限但可以接受稍高的时间复杂度时

## 三种解法对比总结

| 解法         | 时间复杂度 | 空间复杂度 | 优点                     | 缺点                   |
| ------------ | ---------- | ---------- | ------------------------ | ---------------------- |
| 暴力枚举法   | O(n²)      | O(1)       | 实现简单，不需要额外空间 | 效率低，不适合大数据量 |
| 哈希表法     | O(n)       | O(n)       | 效率高，一次遍历即可     | 需要额外空间存储哈希表 |
| 排序双指针法 | O(n log n) | O(n)       | 无需处理哈希冲突         | 实现复杂，需要排序     |

## 关键注意事项

1. 所有解法都要注意不能使用同一个元素两次（即 i ≠ j）
2. 哈希表法中，先检查后插入可以避免使用同一个元素
3. 排序双指针法需要保存原始索引，因为排序会改变元素位置
4. 根据题目约束，输入保证有且仅有一个解，所以不需要处理无解情况

在实际面试中，哈希表法是最优解，既体现了对数据结构的掌握，又有良好的时间复杂度，是面试官期望看到的解法。

# 2

LeetCode Hot100 第二题是 **两数相加**，题目要求：给定两个非空链表，分别代表两个非负整数（逆序存储，每个节点存一位数字），求两数之和并以同样逆序链表返回。

### 核心分析

- 链表逆序存储：如数字 `123` 存储为 `3 → 2 → 1`，相加时从链表头（个位）开始，符合竖式加法 “从低位到高位” 的逻辑。
- 关键问题：处理 **进位**、**链表长度不一致**、**最后一位进位**（如 `999 + 1 = 1000`，需补新节点）。

## 解法一：迭代法（模拟竖式加法，最优解）

### 思路

1. **虚拟头节点**：简化结果链表的创建（无需单独处理头节点为空的情况）。

2. **进位变量**：记录每轮相加的进位（初始为 0）。

3. **循环遍历**：只要 `l1` 未遍历完、`l2` 未遍历完或仍有进位，就继续计算。

4. 每轮计算

   ：

   - 取 `l1`/`l2` 当前值（为空则取 0），加上进位得到总和 `sum`。
   - 当前位数字 = `sum % 10`，新进位 = `sum / 10`。
   - 创建新节点接入结果链表，更新指针和链表。

### 代码实现

javaSP

```java
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        // 1. 虚拟头节点：避免处理头节点为空的麻烦
        ListNode dummyHead = new ListNode(0);
        // 2. curr 指针：跟踪结果链表的当前位置
        ListNode curr = dummyHead;
        // 3. carry 变量：记录每轮相加的进位
        int carry = 0;

        // 循环条件：l1/l2未遍历完 或 仍有进位（确保最后一位进位被处理）
        while (l1 != null || l2 != null || carry != 0) {
            // 取当前节点值（为空则补0）
            int val1 = (l1 != null) ? l1.val : 0;
            int val2 = (l2 != null) ? l2.val : 0;

            // 计算总和（包含上一轮进位）
            int sum = val1 + val2 + carry;
            // 更新进位（取商，如 sum=15 → carry=1）
            carry = sum / 10;
            // 当前位数字（取余，如 sum=15 → 当前位=5）
            int currentVal = sum % 10;

            // 创建新节点并接入结果链表
            curr.next = new ListNode(currentVal);
            curr = curr.next;

            // l1、l2 后移（若不为空）
            if (l1 != null) l1 = l1.next;
            if (l2 != null) l2 = l2.next;
        }

        // 返回结果链表的头（虚拟头的下一个节点）
        return dummyHead.next;
    }
}
```

### 复杂度

- 时间：`O(max(m, n))`，`m`/`n` 为两链表长度，需遍历最长链表一次。
- 空间：`O(max(m, n))`，结果链表长度最多为最长链表长度 + 1（最后进位）。

## 解法二：递归法（利用递归栈处理进位）

### 思路

1. **递归核心**：将 “处理当前位” 与 “递归处理后续位” 分离，用递归栈替代迭代的循环。

2. **终止条件**：`l1`、`l2` 均遍历完且无进位（无需再创建节点）。

3. 每轮递归

   ：

   - 计算当前位总和（含进位），得到当前节点值和新进位。
   - 递归调用自身处理 `l1.next`、`l2.next` 和新进位，得到后续节点链。
   - 将当前节点的 `next` 指向后续节点链，返回当前节点。

### 代码实现

```java
class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        // 调用递归辅助函数，初始进位为 0
        return addRecursively(l1, l2, 0);
    }

    /**
     * 递归辅助函数
     * @param l1 当前待相加的 l1 节点
     * @param l2 当前待相加的 l2 节点
     * @param carry 上一轮的进位
     * @return 当前节点及后续的结果链表
     */
    private ListNode addRecursively(ListNode l1, ListNode l2, int carry) {
        // 终止条件：l1、l2 为空且无进位 → 无需创建节点
        if (l1 == null && l2 == null && carry == 0) {
            return null;
        }

        // 取当前节点值（为空则补0）
        int val1 = (l1 != null) ? l1.val : 0;
        int val2 = (l2 != null) ? l2.val : 0;

        // 计算总和与新进位
        int sum = val1 + val2 + carry;
        int currentVal = sum % 10;
        int newCarry = sum / 10;

        // 创建当前节点，next 指向递归处理的后续节点
        ListNode currentNode = new ListNode(currentVal);
        currentNode.next = addRecursively(
            (l1 != null) ? l1.next : null,  // l1 后移（为空则传 null）
            (l2 != null) ? l2.next : null,  // l2 后移（为空则传 null）
            newCarry                        // 新进位
        );

        // 返回当前节点（作为上一层的 next 节点）
        return currentNode;
    }
}
```

### 复杂度

- 时间：`O(max(m, n))`，递归深度等于最长链表长度 + 1（进位）。
- 空间：`O(max(m, n))`，递归栈深度与最长链表长度一致（链表过长可能栈溢出）。

## 解法三：栈辅助迭代法（用栈存储节点值，头插法构建结果）

### 思路

1. **栈的作用**：利用 “后进先出” 特性，将链表节点值压栈后，弹出顺序对应原数字的 “高位到低位”。

2. **头插法构建结果**：每次相加得到的当前位（从高位到低位）插入结果链表头部，最终得到逆序结果（符合题目要求）。

3. 步骤

   ：

   - 遍历两链表，将节点值分别压入两个栈。
   - 弹出栈顶值相加（含进位），头插法创建结果节点。
   - 处理剩余进位，返回结果链表头。

### 代码实现

```java
import java.util.Stack;

class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        // 1. 初始化栈，存储 l1 和 l2 的节点值
        Stack<Integer> s1 = new Stack<>();
        Stack<Integer> s2 = new Stack<>();

        // 将 l1 节点值压栈（如 l1=3→2→1，栈 s1 为 [3,2,1]）
        while (l1 != null) {
            s1.push(l1.val);
            l1 = l1.next;
        }
        // 将 l2 节点值压栈
        while (l2 != null) {
            s2.push(l2.val);
            l2 = l2.next;
        }

        // 2. 计算并构建结果链表
        int carry = 0;
        ListNode resultHead = null;  // 结果链表头（初始为空）

        // 循环条件：栈不为空 或 仍有进位
        while (!s1.isEmpty() || !s2.isEmpty() || carry != 0) {
            // 弹出栈顶值（为空则补0）
            int val1 = s1.isEmpty() ? 0 : s1.pop();
            int val2 = s2.isEmpty() ? 0 : s2.pop();

            // 计算总和与进位
            int sum = val1 + val2 + carry;
            carry = sum / 10;
            int currentVal = sum % 10;

            // 头插法：新节点作为新头，next 指向原头
            ListNode newNode = new ListNode(currentVal);
            newNode.next = resultHead;
            resultHead = newNode;
        }

        // 返回结果链表头
        return resultHead;
    }
}
```

### 复杂度

- 时间：`O(max(m, n))`，需遍历两链表压栈，再遍历栈计算。
- 空间：`O(m + n)`，栈存储两链表所有节点值，额外空间开销较大。

## 三种解法对比

| 解法     | 优点                                     | 缺点                 | 适用场景                 |
| -------- | ---------------------------------------- | -------------------- | ------------------------ |

# 3

LeetCode Hot100 第三题是 **无重复字符的最长子串**，题目要求：给定一个字符串 `s`，找出其中不含有重复字符的 **最长子串** 的长度（子串是连续的，区别于子序列）。

### 核心分析

- 关键约束：子串 **连续** + 无重复字符。
- 核心问题：如何高效维护一个 “无重复字符的窗口”，并动态更新窗口的最大长度。
- 常见误区：混淆 “子串”（连续）和 “子序列”（非连续），需确保窗口内字符始终连续且唯一。

## 解法一：暴力枚举法（基础思路，理解问题本质）

### 思路

1. **枚举所有子串**：通过双重循环，外层确定子串的起始位置 `i`，内层扩展子串的结束位置 `j`（`j >= i`）。
2. **检查子串唯一性**：用一个集合（`HashSet`）存储当前子串的字符，若 `s[j]` 已在集合中，说明子串 `s[i..j]` 有重复，终止内层循环。
3. **记录最大长度**：每次找到无重复子串时，更新最大长度。

### 代码实现

java



运行









```java
import java.util.HashSet;
import java.util.Set;

class Solution {
    public int lengthOfLongestSubstring(String s) {
        int n = s.length();
        int maxLen = 0; // 记录最长无重复子串长度

        // 外层循环：子串起始位置 i
        for (int i = 0; i < n; i++) {
            // 用集合存储当前子串的字符，确保无重复
            Set<Character> charSet = new HashSet<>();
            int currentLen = 0; // 当前子串长度

            // 内层循环：子串结束位置 j（从 i 开始扩展）
            for (int j = i; j < n; j++) {
                char c = s.charAt(j);
                // 若字符已存在，说明当前子串 s[i..j] 有重复，终止扩展
                if (charSet.contains(c)) {
                    break;
                }
                // 字符加入集合，更新当前子串长度
                charSet.add(c);
                currentLen++;
                // 更新最大长度
                maxLen = Math.max(maxLen, currentLen);
            }
        }
        return maxLen;
    }
}
```

### 复杂度

- 时间：`O(n²)`，`n` 为字符串长度，双重循环枚举所有子串，每个子串检查需 `O(1)`（集合查找）。
- 空间：`O(min(m, n))`，`m` 为字符集大小（如 ASCII 为 128），集合最多存储 `min(m, n)` 个不重复字符。
- 优缺点：思路简单易懂，但效率低，仅适合理解问题，无法通过长字符串测试用例（如 `n=10^4` 会超时）。

## 解法二：滑动窗口（哈希表优化，高效解法）

### 思路

滑动窗口是处理 “连续子串 / 子数组” 问题的核心技巧，本质是维护一个 **[left, right]** 的窗口，确保窗口内无重复字符，通过动态移动窗口边界优化效率。

1. **窗口定义**：`left` 是窗口左边界（无重复子串的起始位置），`right` 是窗口右边界（当前遍历到的字符位置）。

2. **哈希表作用**：存储字符 **最近一次出现的索引**，用于快速判断字符是否在当前窗口内，以及更新左边界 `left`。

3. 窗口移动逻辑

   ：

   - 当 `s[right]` 不在哈希表中，或其最近出现的索引 < `left`（说明不在当前窗口内）：直接将 `s[right]` 加入哈希表，扩展窗口右边界，更新最大长度。
   - 当 `s[right]` 在当前窗口内（最近出现的索引 >= `left`）：将 `left` 跳至 “重复字符的下一个位置”（避免窗口内有重复），再更新 `s[right]` 的索引。

4. **遍历结束**：`right` 遍历完字符串，最大长度即为结果。

### 代码实现

java



运行









```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    public int lengthOfLongestSubstring(String s) {
        int n = s.length();
        int maxLen = 0;
        // 哈希表：key=字符，value=该字符最近一次出现的索引
        Map<Character, Integer> charIndexMap = new HashMap<>();
        // left：窗口左边界（初始为 0）
        int left = 0;

        // 遍历字符串，right 为窗口右边界
        for (int right = 0; right < n; right++) {
            char c = s.charAt(right);
            // 关键：若 c 已出现且索引 >= left（在当前窗口内），更新 left
            if (charIndexMap.containsKey(c) && charIndexMap.get(c) >= left) {
                left = charIndexMap.get(c) + 1;
            }
            // 更新 c 的最近出现索引为当前 right
            charIndexMap.put(c, right);
            // 计算当前窗口长度（right - left + 1），更新最大长度
            maxLen = Math.max(maxLen, right - left + 1);
        }
        return maxLen;
    }
}
```

### 复杂度

- 时间：`O(n)`，仅遍历字符串一次，哈希表的插入和查找均为 `O(1)`。
- 空间：`O(min(m, n))`，哈希表存储字符的最近索引，最多占用 `min(m, n)` 空间（`m` 为字符集大小）。
- 优缺点：效率极高，是面试中的主流解法；逻辑需注意 `left` 的更新不能回退（如字符串 “abba”，需避免 `left` 从 2 退到 1）。

## 解法三：滑动窗口（数组优化，极致效率）

### 思路

由于字符串的字符本质是 ASCII 码（范围 0-127，若含 Unicode 可扩展至 65536），可用 **数组** 代替哈希表存储字符的最近索引。数组的下标对应字符的 ASCII 码，值对应字符最近出现的索引，访问速度比哈希表更快。

1. **数组初始化**：创建大小为 128 的数组 `lastIndex`，初始值为 `-1`（表示字符未出现过）。
2. **窗口逻辑**：与哈希表版本完全一致，仅将 “哈希表操作” 替换为 “数组操作”。
3. **核心优化**：数组的随机访问速度远高于哈希表的哈希计算和冲突处理，在字符集固定时性能更优。

### 代码实现

java



运行









```java
class Solution {
    public int lengthOfLongestSubstring(String s) {
        int n = s.length();
        int maxLen = 0;
        // 数组：下标=字符ASCII码，值=字符最近出现的索引，初始为-1（未出现）
        int[] lastIndex = new int[128];
        for (int i = 0; i < 128; i++) {
            lastIndex[i] = -1;
        }
        int left = 0; // 窗口左边界

        for (int right = 0; right < n; right++) {
            char c = s.charAt(right);
            int ascii = (int) c; // 获取字符的ASCII码

            // 若 c 已出现且在当前窗口内（lastIndex[ascii] >= left），更新 left
            if (lastIndex[ascii] != -1 && lastIndex[ascii] >= left) {
                left = lastIndex[ascii] + 1;
            }
            // 更新 c 的最近出现索引为当前 right
            lastIndex[ascii] = right;
            // 计算窗口长度，更新最大长度
            maxLen = Math.max(maxLen, right - left + 1);
        }
        return maxLen;
    }
}
```

### 复杂度

- 时间：`O(n)`，与哈希表版本一致，但实际运行速度更快（数组访问无哈希开销）。
- 空间：`O(1)`，数组大小固定为 128（或 65536 用于 Unicode），属于常数空间。
- 优缺点：性能最优，适合字符集已知的场景；若字符串含罕见 Unicode 字符，需扩大数组大小，但仍为常数空间。

## 三种解法对比

| 解法               | 时间复杂度 | 空间复杂度        | 核心优势                   | 适用场景                             |
| ------------------ | ---------- | ----------------- | -------------------------- | ------------------------------------ |
| 暴力枚举法         | O(n²)      | O(min(m,n))       | 思路简单，易理解           | 理解问题本质，小规模字符串           |
| 滑动窗口（哈希表） | O(n)       | O(min(m,n))       | 通用性强（支持任意字符集） | 面试首选，字符集不确定的场景         |
| 滑动窗口（数组）   | O(n)       | O (1)（常数空间） | 速度最快，开销最小         | 字符集为 ASCII/Unicode，追求极致性能 |

实际刷题 / 面试中，**滑动窗口（数组优化版）** 是最优选择（性能和空间均占优）；若字符串含特殊 Unicode 字符，可改用哈希表版本，逻辑完全复用。
