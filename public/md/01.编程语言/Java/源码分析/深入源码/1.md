#### 第 3 集：Lambda 表达式初步与函数式接口



1. **Lambda 表达式**
   - 诞生背景：解决匿名内部类代码冗余问题，简化逻辑传递（如排序、筛选）。
   - 语法规则：`(参数列表) -> {代码块/表达式}`，可省略参数类型（类型推断）、代码块大括号及 return（单句逻辑）。
   - 本质：函数式接口的匿名实例，依赖函数式接口（仅含 1 个抽象方法）。
2. **函数式接口**
   - 定义：仅含 1 个抽象方法，可加`@FunctionalInterface`注解（编译器校验），支持默认方法和静态方法。
   - 内置核心接口：
     - `Supplier<T>`：无参返回 T（`get()`）； 
     - `Consumer<T>`：接收 T 无返回（`accept(T)`）；
     - `Function<T,R>`：接收 T 返回 R（`apply(T)`）。

#### 第 4 集：深入函数式接口与方法引用

1. **函数式接口深化**
   - `@FunctionalInterface`：强制校验接口仅含 1 个抽象方法，继承父接口时需合并抽象方法为 1 个。
   - 默认方法设计：如`Consumer.andThen()`，支持函数组合，避免重复串联逻辑。
2. **方法引用**
   - 本质：Lambda 的语法糖，编译为函数式接口实例，字节码更高效。
   - 类型：静态方法引用（`类名::静态方法`）、实例方法引用（`对象::实例方法`）、类的实例方法引用（`类名::实例方法`）、构造器引用（`类名::new`）。
   - JDK 应用：如`Stream.forEach(System.out::println)`，简化 Lambda 调用。

#### 第 5 集：Lambda 表达式深入与 Stream 初步

1. **Lambda 深入**
   - 类型推断：泛型方法中结合参数 / 返回值双向推断；重载时需显式声明接口类型避免歧义。
   - 闭包特性：引用的外部变量需为 “effectively final”（不可变），避免线程安全问题。
   - 与匿名内部类区别：无独立类文件、`this`指向外部类、可复用实例（性能更优）。
2. **Stream 初步**
   - 定义：数据处理管道（非存储容器），支持链式操作。
   - 核心特性：惰性求值（中间操作仅记录逻辑）、流水线（链式调用）、数据源无关性。
   - 操作类型：中间操作（如`filter`，返回 Stream）+ 终止操作（如`forEach`，触发执行）。

#### 第 6 集：Function 接口详解

1. **核心结构**
   - 抽象方法`apply(T t)`：接收 T 返回 R，为数据转换逻辑入口。
   - 默认方法：
     - `compose(Function before)`：先执行 before，再执行当前函数（`before→当前`）；
     - `andThen(Function after)`：先执行当前函数，再执行 after（`当前→after`）。
2. **派生与应用**
   - 派生接口：如`ToIntFunction<T>`（避免装箱拆箱，提升性能）、`BiFunction<T,U,R>`（双输入单输出）。
   - JDK 应用：`Stream.map(Function)`依赖其实现数据转换（如字符串转长度）。

**核心关联**：Lambda 是函数式接口的实例，方法引用简化 Lambda，Stream 依赖函数式接口实现声明式数据处理，Function 接口是数据转换的核心载体。

## 二、核心知识点：Lambda 表达式（从语法到本质）

### 1. Lambda 表达式的 “诞生背景”

- 问题痛点

  ：Java 8 前，若要传递 “一段逻辑”（如排序规则、筛选条件），需定义匿名内部类（代码冗余、模板化），例如：

  java

  ```java
  // 匿名内部类实现Comparator接口（排序逻辑）
  List<Integer> list = Arrays.asList(3,1,2);
  Collections.sort(list, new Comparator<Integer>() {
      @Override
      public int compare(Integer o1, Integer o2) {
          return o1 - o2; // 升序排序逻辑
      }
  });
  ```

  上述代码中，真正有用的仅

  ```
  return o1 - o2
  ```

  ，其余均为 “模板代码”。

- **Lambda 的价值**：用简洁语法 “直接传递逻辑”，消除匿名内部类的冗余代码，同时为后续 Stream API 的 “链式操作” 奠定基础。

### 2. Lambda 表达式的语法规则（3 种核心形式）

视频通过 “从复杂到简洁” 的递进案例，拆解 Lambda 的语法结构：**`(参数列表) -> {代码块/表达式}`**，其中 “可省略部分” 需严格遵循规则：

| 语法形式                  | 适用场景                               | 示例代码（以 Comparator 接口为例）                           |
| ------------------------- | -------------------------------------- | ------------------------------------------------------------ |
| 完整参数 + 完整代码块     | 参数需显式声明类型、代码块含多句逻辑   | `(Integer o1, Integer o2) -> { System.out.println("比较"); return o1 - o2; }` |
| 省略参数类型（类型推断）  | 编译器可通过上下文推断参数类型（推荐） | `(o1, o2) -> { System.out.println("比较"); return o1 - o2; }`（无需写 Integer） |
| 省略代码块大括号 + return | 代码块仅 1 句返回语句（最简洁形式）    | `(o1, o2) -> o1 - o2`（省略`{}`、`return`、`;`，表达式结果直接作为返回值） |

- 关键注意点

  ：

  1. 参数列表括号：若仅 1 个参数，可省略括号（如`(s) -> s.length()` 可简写为 `s -> s.length()`）；若 0 个或 2 个及以上参数，括号不可省略。
  2. 代码块大括号：若代码块含多句逻辑，必须用`{}`包裹，且需显式写`return`（若有返回值）；若仅 1 句，可省略`{}`和`return`。
  3. 类型推断依据：编译器通过 “Lambda 所赋值的接口类型” 推断参数类型（如赋值给`Comparator<Integer>`，则参数默认是`Integer`类型）。

### 3. Lambda 表达式的 “本质”：函数式接口的实例

视频核心强调：**Lambda 不是 “函数作为一等公民”，而是 “函数式接口的匿名实例”**，需满足 “一个关键前提” 和 “两个底层特性”：

#### （1）关键前提：必须关联 “函数式接口”

- **函数式接口定义**：仅包含**一个抽象方法**的接口（可含默认方法、静态方法，但抽象方法只能有 1 个），且需添加`@FunctionalInterface`注解（编译器强制校验，避免后续新增抽象方法破坏 Lambda 兼容性）。

- 示例

  ：Java 8 内置的

  ```
  java.util.Comparator
  ```

  接口（仅 1 个抽象方法

  ```
  compare(T o1, T o2)
  ```

  ），符合函数式接口定义，因此可用 Lambda 赋值：

  java

  ```java
  // Lambda表达式本质是Comparator接口的匿名实例
  Comparator<Integer> comp = (o1, o2) -> o1 - o2;
  ```
  
- **反例警示**：若接口含 2 个抽象方法（非函数式接口），Lambda 无法赋值，编译器直接报错（视频通过自定义 “多抽象方法接口” 演示该错误场景）。

#### （2）底层特性：无独立类型，依赖上下文

- Lambda 本身没有 “独立类型”，其类型由 “赋值的目标接口” 决定（即 “目标类型推断”），例如：

  java

  ```java
// 同样的Lambda，赋值给不同函数式接口，代表不同实例
  // 1. 赋值给Comparator（排序逻辑）
Comparator<Integer> comp = (a, b) -> a - b;
  // 2. 赋值给BinaryOperator（二元运算逻辑）
BinaryOperator<Integer> op = (a, b) -> a - b;
  ```

- 视频通过 “Lambda 作为方法参数” 的案例进一步验证：当 Lambda 作为参数传递时，其类型由 “方法形参的接口类型” 推断（如`Collections.sort`的第二个参数类型为`Comparator`，则 Lambda 自动匹配该接口）。

## 三、核心知识点：函数式接口（Lambda 的 “载体”）

### 1. 函数式接口的 “官方定义与校验”

- 严格定义

  ：满足以下两个条件的接口即为函数式接口：

  1. 仅包含**一个抽象方法**（抽象方法用`abstract`修饰，或接口中默认隐式`abstract`的方法）；
  2. 可添加`@FunctionalInterface`注解（非强制，但推荐添加 —— 编译器会校验是否符合函数式接口规则，若新增抽象方法则报错）。

- 示例：自定义函数式接口

  （视频演示案例）：

  java

  ```java
  // 带@FunctionalInterface注解的函数式接口（计算两个整数的结果）
  @FunctionalInterface
  interface Calculator {
      int calculate(int a, int b); // 仅1个抽象方法
      
      // 可含默认方法（不影响函数式接口特性）
      default void printResult(int result) {
          System.out.println("结果：" + result);
      }
      
      // 可含静态方法（不影响函数式接口特性）
      static Calculator getAddCalculator() {
          return (a, b) -> a + b; // 静态方法返回Lambda实例
      }
  }
  ```

### 2. Java 8 内置核心函数式接口（入门级）

视频为后续课程铺垫，重点介绍`java.util.function`包中最基础的 3 个函数式接口，覆盖 “无参有返回”“有参无返回”“有参有返回” 三大场景：

| 接口名称         | 抽象方法           | 功能描述                                          | 示例（Lambda 实现）                                          |
| ---------------- | ------------------ | ------------------------------------------------- | ------------------------------------------------------------ |
| `Supplier<T>`    | `T get()`          | 无参数，返回 T 类型结果（“供给型” 接口）          | `Supplier<String> strSupplier = () -> "Hello";`（提供字符串） |
| `Consumer<T>`    | `void accept(T t)` | 接收 T 类型参数，无返回（“消费型” 接口）          | `Consumer<Integer> printConsumer = (num) -> System.out.println(num);`（消费整数并打印） |
| `Function<T, R>` | `R apply(T t)`     | 接收 T 类型参数，返回 R 类型结果（“函数型” 接口） | `Function<String, Integer> lenFunction = (s) -> s.length();`（接收字符串，返回长度） |

- **关键关联**：这些接口的设计是为了 “统一逻辑传递的格式”，例如`Supplier`对应 “获取数据的逻辑”，`Consumer`对应 “处理数据的逻辑”，后续 Stream API 的`map`（用`Function`）、`forEach`（用`Consumer`）均依赖这些接口。

## 四、实战案例：Lambda 与函数式接口的结合应用

视频通过 2 个递进案例，强化 “Lambda 是函数式接口实例” 的认知，同时演示实际开发中的常见场景：

### 案例 1：自定义函数式接口的 Lambda 实现

java

```java
// 1. 定义函数式接口（计算逻辑）
@FunctionalInterface
interface Calculator {
    int calculate(int a, int b);
}

public class LambdaDemo {
    public static void main(String[] args) {
        // 2. Lambda实现“加法逻辑”
        Calculator addCalc = (a, b) -> a + b;
        System.out.println(addCalc.calculate(3, 5)); // 输出8
        
        // 3. Lambda实现“乘法逻辑”
        Calculator mulCalc = (a, b) -> a * b;
        System.out.println(mulCalc.calculate(3, 5)); // 输出15
        
        // 4. 作为方法参数传递（逻辑复用）
        int result = compute(10, 20, (a, b) -> a - b); // 传递“减法逻辑”
        System.out.println(result); // 输出-10
    }
    
    // 接收Calculator接口实例，复用不同计算逻辑
    public static int compute(int a, int b, Calculator calc) {
        return calc.calculate(a, b);
    }
}
```

- **核心价值**：通过 Lambda，无需定义多个 “加法类、减法类”，直接传递逻辑即可实现功能复用，代码灵活性大幅提升。

### 案例 2：集合排序的 Lambda 简化

java

```java
public class CollectionSortDemo {
    public static void main(String[] args) {
        List<String> strList = Arrays.asList("apple", "banana", "cherry");
        
        // 传统匿名内部类：按字符串长度排序
        Collections.sort(strList, new Comparator<String>() {
            @Override
            public int compare(String s1, String s2) {
                return s1.length() - s2.length();
            }
        });
        
        // Lambda简化：同样的排序逻辑，代码量减少70%
        Collections.sort(strList, (s1, s2) -> s1.length() - s2.length());
        
        System.out.println(strList); // 输出：[apple, banana, cherry]（按长度升序）
    }
}
```

- **对比优势**：Lambda 直接聚焦 “排序逻辑”（`s1.length() - s2.length()`），消除匿名内部类的模板代码，可读性更强。

## 五、常见问题与避坑指南（视频重点强调）

1. **“Lambda 能替代所有匿名内部类吗？”**

   - 不能。仅当匿名内部类实现的是 “函数式接口”（1 个抽象方法）时，才可替换；若接口含多个抽象方法（如`MouseListener`含 5 个抽象方法），无法用 Lambda 替代。

2. **Lambda 中 “外部变量的访问限制”**

   - Lambda 中引用的外部局部变量，必须是 “effectively final”（隐式不可变）—— 即变量声明后未被重新赋值（无需显式加

     ```
     final
     ```

     ，但重新赋值会报错），例如：

     java

     

     ```java
     int num = 10; // effectively final（未重新赋值）
     Consumer<String> consumer = (s) -> System.out.println(s + num); // 合法
     
     // num = 20; // 若取消注释，Lambda报错：局部变量在Lambda中访问必须是final或effectively final
     ```
     
   - 原因：Lambda 可能在多线程中执行，限制变量不可变可避免线程安全问题，同时符合函数式编程 “无副作用” 的核心思想。
   
3. **“Lambda 的 this 指向谁？”**

   - Lambda 没有独立的

     ```
     this
     ```

     ，其

     ```
     this
     ```

     指向 “包含 Lambda 的外部类实例”，而非 Lambda 自身（与匿名内部类不同，匿名内部类的

     ```
     this
     ```

     指向自身实例），例如：

     java

     

     ```java
     public class LambdaThisDemo {
         private String name = "外部类";
         
         public void test() {
             // Lambda的this指向LambdaThisDemo实例
             Consumer<String> consumer = (s) -> System.out.println(this.name + s);
             consumer.accept("Lambda"); // 输出：外部类Lambda
             
             // 匿名内部类的this指向匿名内部类实例
             new Consumer<String>() {
                 private String name = "匿名内部类";
                 @Override
                 public void accept(String s) {
                     System.out.println(this.name + s); // 输出：匿名内部类Lambda
                 }
             }.accept("Lambda");
         }
     }
     ```

### 二、核心知识点：函数式接口的深化认知

#### 1. 函数式接口的 “严格性”：`@FunctionalInterface`注解

- **注解作用**：标记接口为 “函数式接口”，编译器强制校验（仅允许含**1 个抽象方法**，否则报错）。
- **特殊场景**：接口继承父接口时，若父接口的抽象方法被 “合并” 为 1 个（如 `Comparator` 继承 `Serializable`，不新增抽象方法），仍可标记为函数式接口。
- **源码示例**：JDK 中 `java.util.function.Consumer` 接口通过 `@FunctionalInterface` 标注，确保仅含 `accept(T t)` 这一个抽象方法。

#### 2. 内置函数式接口的 “默认方法” 设计（以`Consumer`为例）

JDK 为核心函数式接口提供**默认方法**，实现 “逻辑复用 + 扩展能力”。典型如 `Consumer<T>` 的 `andThen(Consumer<? super T> after)`：

java

```java
@FunctionalInterface
public interface Consumer<T> {
    void accept(T t); // 唯一抽象方法

    // 默认方法：串联多个 Consumer 逻辑
    default Consumer<T> andThen(Consumer<? super T> after) {
        Objects.requireNonNull(after);
        return (T t) -> { 
            accept(t);   // 先执行当前 Consumer
            after.accept(t); // 再执行传入的 Consumer
        }; 
    }
}
```

- **设计意图**：支持 “函数组合”，避免重复编写串联逻辑。

- 实战案例

  ：

  java

  ```java
  Consumer<String> print = System.out::println; // 打印原字符串
  Consumer<String> toUpper = s -> System.out.println(s.toUpperCase()); // 打印大写字符串
  print.andThen(toUpper).accept("hello"); // 输出：hello → HELLO
  ```

### 三、核心知识点：方法引用的原理与场景

#### 1. 方法引用的本质：Lambda 的 “语法糖 + 字节码优化”

方法引用是 Lambda 的简化形式，**编译器会自动将其转换为函数式接口的实例**，且字节码层面更高效（减少匿名类开销）。

#### 2. 四种方法引用类型（结合 JDK 源码案例）

| 方法引用类型     | 语法格式         | 对应函数式接口        | JDK 源码典型场景                    |
| ---------------- | ---------------- | --------------------- | ----------------------------------- |
| 静态方法引用     | `类名::静态方法` | `Function`/`Supplier` | `Integer::parseInt`（字符串转整数） |
| 实例方法引用     | `对象::实例方法` | `Consumer`/`Function` | `System.out::println`（打印操作）   |
| 类的实例方法引用 | `类名::实例方法` | `BiFunction`          | `String::compareTo`（字符串比较）   |
| 构造器引用       | `类名::new`      | `Supplier`/`Function` | `ArrayList::new`（创建集合实例）    |

#### 3. 方法引用的 “隐式参数” 与 Lambda 的关联

方法引用自动继承 Lambda 的**上下文类型推断**。例如：

java



```java
// Lambda 形式：(s) -> s.length()
Function<String, Integer> func1 = String::length; 

// 方法引用等价于 Lambda，编译器自动匹配 Function 的 apply 方法
```

### 四、源码级分析：JDK 如何运用函数式接口 + 方法引用？

以 `java.util.stream.Stream` 的 `forEach` 方法为例：

java

```java
public interface Stream<T> extends BaseStream<T, Stream<T>> {
    void forEach(Consumer<? super T> action); // 接收 Consumer 接口
}

// 调用时用方法引用简化
Stream<String> stream = Stream.of("a", "b", "c");
stream.forEach(System.out::println); // 等价于 Lambda: s -> System.out.println(s)
```

- **设计逻辑**：`forEach` 依赖 `Consumer` 函数式接口，方法引用 `System.out::println` 自动适配 `Consumer` 的 `accept(T t)` 方法，**减少冗余代码**。
- **源码细节**：`Stream.forEach` 底层通过 `Spliterator` 遍历元素，将每个元素传递给 `Consumer.accept`，方法引用在此过程中实现 “无感知” 的逻辑注入。

### 五、避坑与最佳实践

#### 1. 函数式接口的 “抽象方法冲突”

若自定义接口继承多个函数式接口，且抽象方法签名冲突，会导致编译错误。例如：

java

```java
@FunctionalInterface
interface Conflict extends Consumer<String>, Function<String, Integer> { 
    // 错误：Consumer 的 accept 与 Function 的 apply 签名冲突，无法确定唯一抽象方法
}
```

#### 2. 方法引用的 “空指针风险”

实例方法引用依赖对象非空，若对象为 `null` 会触发 `NullPointerException`。例如：

java

```java
String str = null;
Consumer<String> consumer = str::length; // 此处 str 为 null，直接报 NPE
```

#### 3. 最佳实践：优先用方法引用简化 Lambda

当 Lambda 体仅调用已有方法时，**优先替换为方法引用**，提升可读性。例如：

java

```java
// Lambda 形式（冗余）
list.stream().map(s -> s.toUpperCase()); 
// 方法引用形式（简洁）
list.stream().map(String::toUpperCase); 
```

### 1. Lambda 的 “类型推断” 进阶规则

视频通过多场景案例，拆解编译器如何 “精准推断” Lambda 对应的函数式接口类型，核心规则如下：

#### （1）泛型方法中的类型推断

当 Lambda 作为泛型方法的参数时，编译器会结合 “方法泛型声明” 和 “Lambda 参数 / 返回值” 双向推断类型，无需显式指定泛型。
**示例（JDK 源码场景）**：

java

```java
// 泛型方法：将List转换为指定类型的数组（T为泛型）
public static <T> T[] toArray(List<T> list, Function<T, T> mapper) {
    T[] arr = (T[]) new Object[list.size()];
    for (int i = 0; i < list.size(); i++) {
        arr[i] = mapper.apply(list.get(i));
    }
    return arr;
}

// 调用时：编译器通过Lambda的参数（String）和返回值（String）推断T=String
List<String> strList = Arrays.asList("a", "b", "c");
String[] upperArr = toArray(strList, s -> s.toUpperCase()); 
// 无需写 toArray<String>(...)，泛型T自动推断为String
```

#### （2）方法重载时的 “显式类型提示”

当存在多个重载方法，且 Lambda 参数类型无法唯一推断时，需通过 “显式声明函数式接口类型” 或 “参数类型” 避免歧义。
**反例（歧义场景）**：

java

// 重载方法1：参数为Consumer<String>（接收String，无返回）

// 重载方法2：参数为Function<String, Integer>（接收String，返回Integer）
public static void process(Function<String, Integer> function) { /* ... */ }

// 错误调用：Lambda (s) -> s.length() 既符合Function（返回长度），也可隐式适配Consumer（忽略返回值）
// process(s -> s.length()); // 编译报错：无法确定调用哪个重载方法

// 正确解决方式1：显式声明函数式接口类型

// 正确解决方式2：显式声明参数类型（辅助推断）

```

### 2. Lambda 的 “闭包” 特性与外部变量限制

视频深入讲解 Lambda “访问外部变量” 的底层逻辑，核心是 “闭包的不可变性约束”：

#### （1）外部变量的 “effectively final” 本质

Lambda 可访问外部局部变量，但变量必须满足 “声明后未被重新赋值”（无需显式加`final`，即 “effectively final”）。
**示例与原理**：

java

```java
int base = 10; // effectively final（未重新赋值）
// 正确：Lambda访问base，且base后续未修改
Function<Integer, Integer> add = num -> num + base; 
System.out.println(add.apply(5)); // 输出15

// base = 20; // 错误：若重新赋值，Lambda报错——破坏闭包的不可变性
```

**底层原因**：Lambda 可能在多线程中执行，外部变量若可变会导致线程安全问题；同时，Lambda 捕获的是变量 “值” 而非 “引用”，重新赋值会导致 Lambda 持有的值与外部变量不一致，违背函数式编程 “无副作用” 的核心思想。

#### （2）与匿名内部类的 “变量访问” 差异

匿名内部类可访问 “显式 final” 的外部变量，而 Lambda 支持 “隐式 final”（effectively final），是 Java 8 对闭包特性的优化，减少代码冗余：

java

```java
// 匿名内部类：需外部变量显式final
final int num1 = 5;
Consumer<String> consumer1 = new Consumer<String>() {
    @Override
    public void accept(String s) {
        System.out.println(s + num1);
    }
};

// Lambda：外部变量隐式final（无需写final）
int num2 = 5;
Consumer<String> consumer2 = s -> System.out.println(s + num2);
```

### 3. Lambda 与匿名内部类的 “底层差异”（字节码层面）

视频通过`javap`反编译演示，揭示两者的核心区别，解释 “Lambda 为何更高效”：

| 对比维度   | 匿名内部类                              | Lambda 表达式                                        |
| ---------- | --------------------------------------- | ---------------------------------------------------- |
| 字节码生成 | 编译时生成独立的`XXX$1.class`匿名类文件 | 不生成独立类文件，通过`invokedynamic`指令动态绑定    |
| 实例创建   | 每次使用都新建一个匿名类实例            | 若捕获的外部变量相同，可复用实例（减少对象创建开销） |
| `this`指向 | `this`指向匿名类自身实例                | `this`指向包含 Lambda 的外部类实例                   |
| 性能       | 类加载、实例创建开销较大                | 动态绑定，开销更低，适合高频调用（如 Stream 遍历）   |

## 三、核心知识点二：Stream API 初步（从 “传统遍历” 到 “流式处理”）

本部分是 Stream 的 “入门导论”，视频通过 “传统方式痛点→Stream 解决方案” 的对比，让用户理解 Stream 的设计价值。

### 1. Stream 的核心定义与特性

#### （1）什么是 Stream？

Stream 是 Java 8 提供的 “**数据处理管道**”—— 它不是数据结构（不存储数据），而是对数据源（集合、数组等）的 “计算操作序列”，支持链式调用，核心是 “用声明式代码实现数据处理”。

#### （2）Stream 的 3 大核心特性（视频重点强调）

- **惰性求值（Lazy Evaluation）**：中间操作（如`filter`/`map`）仅 “记录操作逻辑”，不实际处理数据；只有当执行终止操作（如`forEach`/`collect`）时，才会触发所有操作的流水线执行（减少无效计算）。
- **流水线（Pipeline）**：多个中间操作可链式串联，形成处理流水线，代码更简洁（替代传统的嵌套循环 / 多步遍历）。
- **数据源无关性**：无论数据源是集合（`List`/`Set`）、数组还是 I/O 流，Stream 的操作 API 统一，降低学习成本。

#### （3）Stream 与集合的本质区别

| 对比维度     | 集合（Collection）                 | Stream                                          |
| ------------ | ---------------------------------- | ----------------------------------------------- |
| 核心角色     | 数据的 “存储容器”                  | 数据的 “处理工具”                               |
| 数据处理时机 | 主动遍历（如`for-each`），立即执行 | 惰性执行，仅在终止操作时触发                    |
| 遍历方式     | 外部迭代（用户手动控制遍历过程）   | 内部迭代（Stream 自动控制遍历，可优化执行效率） |
| 可复用性     | 可重复遍历（数据存储在内存中）     | 不可重复使用（执行终止操作后，Stream 自动关闭） |

### 2. Stream 的获取方式（3 种常见场景）

### 3. Stream 的 “中间操作 + 终止操作” 入门示例

视频通过 “筛选集合中的偶数并打印” 的案例，对比传统方式与 Stream 方式，直观展示 Stream 的优势：

#### （1）传统方式（外部迭代，代码冗余）

java

```java
List<Integer> numList = Arrays.asList(1, 2, 3, 4, 5, 6);
// 步骤1：筛选偶数
List<Integer> evenList = new ArrayList<>();
for (int num : numList) {
    if (num % 2 == 0) {
        evenList.add(num);
    }
}
// 步骤2：打印结果
for (int even : evenList) {
    System.out.println(even);
}
```

#### （2）Stream 方式（内部迭代，链式调用）

java

```java
List<Integer> numList = Arrays.asList(1, 2, 3, 4, 5, 6);
numList.stream()
       .filter(num -> num % 2 == 0) // 中间操作：筛选偶数（依赖Predicate接口，Lambda实现）
       .forEach(System.out::println); // 终止操作：打印结果（依赖Consumer接口，方法引用实现）
```

#### （3）关键说明

- **中间操作（`filter`）**：返回`Stream`，可继续链式调用，核心是 “筛选符合条件的元素”，其参数是`Predicate<T>`函数式接口，Lambda`num -> num % 2 == 0`实现`test(T t)`方法。
- **终止操作（`forEach`）**：无返回值（`void`），触发整个流水线执行，其参数是`Consumer<T>`函数式接口，方法引用`System.out::println`实现`accept(T t)`方法。

## 五、避坑指南（视频重点警示）

1. **Lambda 的 “外部变量不可变” 陷阱**

   - 误区：试图在 Lambda 中修改外部局部变量（如`base = 20`），编译器直接报错。

3. **Stream 的 “不可复用” 陷阱**

   - 误区：对同一个 Stream 执行多次终止操作（如先`count()`再`forEach()`）。

   - 错误示例：

     java

     ```java

     stream.count(); // 第一次终止操作，Stream关闭
   stream.forEach(System.out::println); // 报错：IllegalStateException（Stream已关闭）
     ```

     

     

   - 解决：每次需要处理数据时，重新获取 Stream（如`list.stream()`）。

## 二、核心知识点一：Function 接口的基础定义与核心能力

`Function<T, R>`是 Java 8 中 **“输入 - 输出型” 函数式接口 ** 的核心，专注于 “接收一个 T 类型参数，返回一个 R 类型结果”，是数据转换、逻辑传递的基础载体。

### 1. 接口的核心结构（源码剖析）

视频逐行解析 JDK 中`Function`接口的源码，核心包含 1 个抽象方法和 2 个默认方法，无静态方法，结构极简但扩展性强：

java

```java
@FunctionalInterface
public interface Function<T, R> {
    // 1. 抽象方法：接收T类型参数，返回R类型结果（核心逻辑载体）
    R apply(T t);

    // 2. 默认方法1：函数组合（先执行参数函数，再执行当前函数）
    default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {
        Objects.requireNonNull(before); // 非空校验，避免空指针
        return (V v) -> apply(before.apply(v)); // 先执行before，结果作为当前apply的参数
    }

    // 3. 默认方法2：函数串联（先执行当前函数，再执行参数函数）
    default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {
        Objects.requireNonNull(after); // 非空校验
        return (T t) -> after.apply(apply(t)); // 先执行当前apply，结果作为after的参数
    }
}
```

- **关键注解**：`@FunctionalInterface`确保接口仅含 1 个抽象方法，编译器强制校验，避免后续扩展破坏函数式接口特性。
- **泛型约束**：`T`（输入类型）、`R`（输出类型）、`V`（组合 / 串联时的中间类型），通过泛型实现 “类型安全的函数传递”。

### 2. 抽象方法`apply(T t)`：函数的 “核心逻辑入口”

`apply`是`Function`的唯一抽象方法，所有`Function`实例（Lambda / 方法引用）最终都通过`apply`执行逻辑，是 “输入→输出” 的直接体现。

#### （1）基础用法：Lambda 实现数据转换

java

```java
// 案例1：String→Integer（字符串转长度）
Function<String, Integer> strToLen = s -> s.length();

## 三、核心知识点二：默认方法的 “函数组合与串联” 能力

视频重点讲解`compose`和`andThen`的区别与应用，这两个方法是`Function`“扩展逻辑复杂度” 的关键，支持多函数协同工作。

### 1. `compose(Function<? super V, ? extends T> before)`：先执行参数函数，再执行当前函数

- **执行顺序**：`before.apply(v)` → `当前Function.apply(结果)`，即 “参数函数在前，当前函数在后”。
- **适用场景**：需要 “预处理输入数据” 时（如先清洗数据，再转换数据）。

#### 示例：多函数组合处理用户输入

需求：先将输入的 “数字字符串” 转为 Integer（预处理），再将 Integer 转为二进制字符串（最终转换）：

java



运行









```java
// 步骤1：定义预处理函数（String→Integer，清洗非数字字符）
Function<String, Integer> strToInt = s -> {
    // 预处理：保留字符串中的数字字符
    String numStr = s.replaceAll("[^0-9]", "");
    return numStr.isEmpty() ? 0 : Integer.parseInt(numStr);
};

// 步骤2：定义最终转换函数（Integer→String，转二进制）
Function<Integer, String> intToBinary = num -> Integer.toBinaryString(num);

// 步骤3：用compose组合函数（先执行strToInt，再执行intToBinary）
Function<String, String> strToBinary = intToBinary.compose(strToInt);

// 测试：输入含非数字字符的字符串
System.out.println(strToBinary.apply("abc123def")); // 输出"1111001"（123的二进制）
System.out.println(strToBinary.apply("noNum")); // 输出"0"（无数字时返回0的二进制）
```

### 2. `andThen(Function<? super R, ? extends V> after)`：先执行当前函数，再执行参数函数

- **执行顺序**：`当前Function.apply(t)` → `after.apply(结果)`，即 “当前函数在前，参数函数在后”。
- **适用场景**：需要 “链式处理输出数据” 时（如先转换数据，再格式化结果）。

#### 示例：链式处理订单金额

需求：先将订单金额（String）转为 Double（转换），再对金额保留 2 位小数（格式化）：

java



运行









```java
// 步骤1：定义转换函数（String→Double）
Function<String, Double> strToDouble = Double::parseDouble;

// 步骤2：定义格式化函数（Double→String，保留2位小数）
Function<Double, String> formatMoney = num -> String.format("%.2f", num);

// 步骤3：用andThen串联函数（先转Double，再格式化）
Function<String, String> strToFormattedMoney = strToDouble.andThen(formatMoney);

// 测试
System.out.println(strToFormattedMoney.apply("199.5")); // 输出"199.50"
System.out.println(strToFormattedMoney.apply("299")); // 输出"299.00"
```

### 3. `compose`与`andThen`的核心区别（视频对比总结）

| 对比维度 | `compose(before)`                     | `andThen(after)`                     |
| -------- | ------------------------------------- | ------------------------------------ |
| 执行顺序 | 先`before.apply`，后`当前.apply`      | 先`当前.apply`，后`after.apply`      |
| 泛型关联 | `before`的输出类型 = `当前`的输入类型 | `当前`的输出类型 = `after`的输入类型 |
| 应用场景 | 输入数据预处理（如清洗、转换格式）    | 输出数据后处理（如格式化、二次转换） |
| 代码示例 | `f2.compose(f1)` → `f1→f2`            | `f1.andThen(f2)` → `f1→f2`           |

#### 易错点警示：避免 “执行顺序混淆”

java



运行









```java
// 定义两个基础函数
Function<Integer, Integer> add5 = num -> num + 5; // f1：加5
Function<Integer, Integer> multiply2 = num -> num * 2; // f2：乘2

// 1. compose：f2.compose(f1) → 先f1（加5），后f2（乘2）
System.out.println(f2.compose(f1).apply(3)); // (3+5)*2=16

// 2. andThen：f1.andThen(f2) → 先f1（加5），后f2（乘2）
System.out.println(f1.andThen(f2).apply(3)); // (3+5)*2=16

// 注意：上述案例因函数顺序巧合结果一致，但逻辑不同；若交换函数，结果差异明显
System.out.println(f1.compose(f2).apply(3)); // (3*2)+5=11
System.out.println(f2.andThen(f1).apply(3)); // (3*2)+5=11
```

## 四、核心知识点三：Function 的派生接口与 JDK 应用

视频介绍`Function`的 “单参数基本类型派生接口” 和 “多参数扩展接口”，以及它们在 JDK 源码中的实际应用，体现接口设计的 “针对性优化”。

### 1. 单参数基本类型派生接口（避免自动装箱拆箱）

`Function<T, R>`处理基本类型（如`int`/`long`）时，会触发自动装箱（`int→Integer`）和拆箱，存在性能开销。JDK 提供专门的派生接口，直接操作基本类型，提升效率：

| 派生接口           | 抽象方法                  | 功能描述            | 适用场景                                |
| ------------------ | ------------------------- | ------------------- | --------------------------------------- |
| `IntFunction<R>`   | `R apply(int value)`      | 接收`int`，返回`R`  | 避免`int→Integer`装箱（如`int→String`） |
| `LongFunction<R>`  | `R apply(long value)`     | 接收`long`，返回`R` | 处理长整型数据（如时间戳转换）          |
| `ToIntFunction<T>` | `int applyAsInt(T value)` | 接收`T`，返回`int`  | 避免`Integer→int`拆箱（如`String→int`） |

#### 示例：`ToIntFunction`优化性能

java



运行









```java

int num1 = strToInt.apply("123"); // 拆箱：Integer→int

// ToIntFunction：直接返回int，无拆箱
ToIntFunction<String> strToIntOpt = Integer::parseInt;
int num2 = strToIntOpt.applyAsInt("123"); // 无拆箱，性能更优
```

### 2. 多参数扩展：`BiFunction<T, U, R>`（为后续 Stream 分组铺垫）

`Function`仅支持 “单输入”，JDK 扩展出`BiFunction`支持 “双输入→单输出”，抽象方法为`R apply(T t, U u)`，是`Function`的重要补充（后续`Stream.collect`的`Collector`会用到）。

### 案例：用户信息清洗与转换

需求：

1. 接收用户输入的 “姓名（含空格）+ 年龄（字符串）”；
2. 清洗姓名（去除空格）；

4. 拼接为 “姓名 (年龄)” 格式的字符串。

#### 实现代码：

java



运行

Function<String, String> cleanName = name -> name.trim();

// 步骤2：转换年龄（String→Integer，默认0）
Function<String, Integer> strToAge = ageStr -> {
    try {
        return Integer.parseInt(ageStr.trim());

    }
};

// 注：BiFunction需结合andThen扩展，此处用Lambda封装双参数

    Integer age = strToAge.apply(userArr[1]);

};

// 测试

System.out.println(processUser.apply(user2)); // 输出"Li Si(0)"
```

### 《深入理解 Java8+JDK8 源码及思想》核心笔记（第 3、44、5、6 集・详细版）

#### 第 3 集：Lambda 表达式初步与函数式接口

**1. Lambda 表达式：简化逻辑传递的语法糖**

- **诞生背景**：
  Java 8 前传递逻辑（如排序、筛选）需用匿名内部类，代码冗余（仅核心逻辑有用，其余为模板代码）。例如：

  java

  ```java
  // 匿名内部类排序（冗余）
  Collections.sort(list, new Comparator<Integer>() {
      @Override
      public int compare(Integer o1, Integer o2) { return o1 - o2; }
  });
  // Lambda简化：直接传递核心逻辑
  Collections.sort(list, (o1, o2) -> o1 - o2);
  ```

- **语法规则**（3 种形式）：

  | 形式                | 示例（Comparator 接口）                           | 适用场景                     |
  | ------------------- | ------------------------------------------------- | ---------------------------- |
  | 完整参数 + 代码块   | `(Integer o1, Integer o2) -> { return o1 - o2; }` | 多句逻辑，需显式声明参数类型 |
  | 省略参数类型        | `(o1, o2) -> { return o1 - o2; }`                 | 编译器可推断参数类型（推荐） |
  | 省略大括号及 return | `(o1, o2) -> o1 - o2`                             | 单句返回逻辑                 |

- **本质**：
  必须关联**函数式接口**（仅含 1 个抽象方法的接口），是接口的匿名实例。例如`Comparator<Integer>`是函数式接口，故 Lambda 可赋值给它。

**2. 函数式接口：Lambda 的 “载体”**

- **定义与校验**：

  - 严格定义：仅包含 1 个抽象方法（可含默认方法、静态方法）。
  - `@FunctionalInterface`注解：非强制，但编译器会校验接口是否符合函数式接口规则（新增抽象方法则报错）。

- **自定义示例**：

  java  

  ```java
  @FunctionalInterface
  interface Calculator {
      int calculate(int a, int b); // 唯一抽象方法
      default void print(int res) { System.out.println(res); } // 默认方法
      static Calculator add() { return (a,b) -> a+b; } // 静态方法
  }
  ```
  
- **内置核心接口（入门级）**：

  | 接口            | 抽象方法         | 功能描述                  | 示例                             |
  | --------------- | ---------------- | ------------------------- | -------------------------------- |
  | `Supplier<T>`   | `T get()`        | 无参返回 T（供给数据）    | `() -> "hello"`                  |
  | `Consumer<T>`   | `void accept(T)` | 接收 T 无返回（消费数据） | `num -> System.out.println(num)` |
  | `Function<T,R>` | `R apply(T)`     | 接收 T 返回 R（转换数据） | `s -> s.length()`                |

#### 第 4集：深入函数式接口与方法引用

**1. 函数式接口的深度特性**

- **`@FunctionalInterface`的严格性**：

  - 接口继承父接口时，需合并父接口的抽象方法为 1 个（如`Comparator`继承`Serializable`，不新增抽象方法，仍为函数式接口）。
  - 若接口含 2 个抽象方法，加注解会编译报错。

- **默认方法的 “函数组合” 能力**：
  核心接口通过默认方法支持逻辑复用，例如`Consumer.andThen()`：

  java

  ```java
  @FunctionalInterface
  public interface Consumer<T> {
      void accept(T t);
      default Consumer<T> andThen(Consumer<? super T> after) {
          return t -> { accept(t); after.accept(t); }; // 串联执行
      }
  }
  // 用法：先打印原字符串，再打印大写
  Consumer<String> print = System.out::println;
  Consumer<String> toUpper = s -> System.out.println(s.toUpperCase());
  print.andThen(toUpper).accept("hello"); // 输出：hello → HELLO
  ```

**2. 方法引用：Lambda 的 “简化版”**

- **本质**：
  编译器自动转换为函数式接口实例，字节码层面比 Lambda 更高效（减少匿名类开销）。

- **4 种类型及示例**：

  | 类型             | 语法格式         | 示例                  | 对应 Lambda                   |
  | ---------------- | ---------------- | --------------------- | ----------------------------- |
  | 静态方法引用     | `类名::静态方法` | `Integer::parseInt`   | `s -> Integer.parseInt(s)`    |
  | 实例方法引用     | `对象::实例方法` | `System.out::println` | `s -> System.out.println(s)`  |
  | 类的实例方法引用 | `类名::实例方法` | `String::compareTo`   | `(s1,s2) -> s1.compareTo(s2)` |
  | 构造器引用       | `类名::new`      | `ArrayList::new`      | `() -> new ArrayList<>()`     |

- **JDK 应用**：
  如`Stream.forEach()`接收`Consumer`，用方法引用简化：

  java

  ```java
  Stream.of("a","b").forEach(System.out::println); // 等价于s -> System.out.println(s)
  ```

#### 第 5 集：Lambda 表达式深入与 Stream 初步

**1. Lambda 的进阶特性**

- **类型推断规则**：

  - 泛型方法中：结合方法泛型声明与 Lambda 参数 / 返回值双向推断（无需显式指定泛型）。

    java

    ```java
// 泛型方法：T自动推断为String
    <T> T[] toArray(List<T> list, Function<T,T> mapper) { ... }
String[] res = toArray(list, s -> s.toUpperCase()); // 无需写toArray<String>(...)
    ```

  - 方法重载时：若 Lambda 类型歧义，需显式声明接口类型（如`(Function<String,Integer>) s -> s.length()`）。

- **闭包与外部变量**：

  - Lambda 引用的外部局部变量必须为 “effectively final”（声明后未重新赋值），避免多线程下的数据不一致。
  - 与匿名内部类区别：匿名类需显式`final`，Lambda 支持隐式不可变。

- **与匿名内部类的底层差异**：

  | 维度       | 匿名内部类                | Lambda 表达式                           |
  | ---------- | ------------------------- | --------------------------------------- |
  | 字节码     | 生成独立`XXX$1.class`文件 | 无独立文件，通过`invokedynamic`动态绑定 |
  | `this`指向 | 指向匿名类自身            | 指向外部类实例                          |
  | 实例复用   | 每次创建新实例            | 捕获变量相同时可复用实例（性能更优）    |

**2. Stream API 入门**

- **核心定义**：
  数据处理管道（非存储容器），对数据源（集合、数组等）执行计算操作，支持链式调用。

- **3 大特性**：

  - 惰性求值：中间操作（如`filter`）仅记录逻辑，不执行；终止操作（如`forEach`）触发全流程执行。
  - 流水线：中间操作返回`Stream`，支持链式串联（如`filter→map→collect`）。
  - 数据源无关性：无论数据源是`List`、数组还是文件流，操作 API 统一。

- **基础操作示例**：

  java

  ```java
  // 筛选偶数并打印（中间操作filter + 终止操作forEach）
  List<Integer> nums = Arrays.asList(1,2,3,4);
  nums.stream()
      .filter(num -> num % 2 == 0) // 中间操作：记录筛选逻辑
      .forEach(System.out::println); // 终止操作：触发执行，输出2、4
  ```

#### 第 6 集：Function 接口详解

**1. 核心结构与方法**

- **接口定义**：

  java

  ```java
  @FunctionalInterface
  public interface Function<T, R> {
      R apply(T t); // 抽象方法：转换逻辑入口
      
      // 先执行before，再执行当前函数（before→当前）
      default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {
          return v -> apply(before.apply(v));
      }
      
      // 先执行当前函数，再执行after（当前→after）
      default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {
          return t -> after.apply(apply(t));
      }
  }
  ```
  
- **`compose`与`andThen`对比**：

  java

  ```java
  Function<Integer, Integer> add5 = num -> num + 5;
  Function<Integer, Integer> mul2 = num -> num * 2;
  
  add5.andThen(mul2).apply(3); // (3+5)*2=16（先加后乘）
  mul2.compose(add5).apply(3); // (3+5)*2=16（先加后乘，逻辑等价）
  add5.compose(mul2).apply(3); // (3*2)+5=11（先乘后加）
```

**2. 派生接口与 JDK 应用**

- **基本类型派生接口**：
  避免自动装箱 / 拆箱开销，如`ToIntFunction<T>`（直接返回`int`）：

  java

  ```java
  // 传统Function有拆箱开销，ToIntFunction更高效
  ToIntFunction<String> strToInt = Integer::parseInt;
  int num = strToInt.applyAsInt("123"); // 无拆箱
```

- **多参数扩展：`BiFunction<T,U,R>`**：
  支持双输入单输出（`R apply(T t, U u)`），例如：

  java

  ```java
  BiFunction<Integer, Integer, Integer> add = (a,b) -> a + b;
  add.apply(3,5); // 8
  ```
  
  
  
  
  
- **JDK 中的核心应用**：
  `Stream.map(Function)`依赖其实现数据转换，例如：

  java

  ```java
  // 将字符串转为长度（Function<String, Integer>）
  List<Integer> lengths = list.stream()
      .map(String::length)
      .collect(Collectors.toList());
  ```
  
  
  

**总结**：
Lambda 通过函数式接口实现逻辑传递，方法引用简化 Lambda 调用，Stream 依托函数式接口实现声明式数据处理，Function 接口是数据转换的核心载体，四者共同构成 Java 8 函数式编程的基础。

### 一、问答题（难度递增）及答案

#### 1. **基础题**

**题目**：Lambda 表达式的基本语法结构是什么？请用 Lambda 表达式实现一个 “两个整数相加” 的逻辑，并说明其对应的函数式接口类型。

**答案**：

- 基本语法结构：`(参数列表) -> {代码块/表达式}`。
- 实现 “两个整数相加”：`(a, b) -> a + b`。
- 对应的函数式接口类型：`BinaryOperator<Integer>`（该接口继承自`BiFunction<Integer, Integer, Integer>`，专门用于接收两个同类型参数并返回同类型结果）。

#### 2. **基础题**

**题目**：什么是函数式接口？`@FunctionalInterface`注解的作用是什么？如果一个接口包含 2 个抽象方法，能否被标记为函数式接口？

**答案**：

- 函数式接口：仅包含**一个抽象方法**的接口（可包含默认方法和静态方法）。
- `@FunctionalInterface`注解的作用：编译器强制校验接口是否符合函数式接口规则（若包含多个抽象方法则编译报错），增强代码可读性。
- 若接口包含 2 个抽象方法：**不能**被标记为函数式接口，会触发编译错误。

#### 3. **中等题**

**题目**：`java.util.function`包中，`Consumer<T>`、`Supplier<T>`、`Function<T,R>`分别对应什么逻辑场景？请分别写出它们的核心抽象方法。

**答案**：

- `Consumer<T>`：消费型接口，接收`T`类型参数并处理（无返回值），核心方法：`void accept(T t)`。
- `Supplier<T>`：供给型接口，无参数，返回`T`类型结果（提供数据），核心方法：`T get()`。
- `Function<T,R>`：函数型接口，接收`T`类型参数，返回`R`类型结果（数据转换），核心方法：`R apply(T t)`。

#### 4. **中等题**

**题目**：方法引用有哪几种类型？请分别举例说明 “类的静态方法引用” 和 “对象的实例方法引用” 的语法及适用场景。

**答案**：

- 方法引用的类型：类的静态方法引用、对象的实例方法引用、类的实例方法引用、构造器引用。
- 类的静态方法引用：
  - 语法：`类名::静态方法`。
  - 示例：`Integer::parseInt`（字符串转整数，对应`Function<String, Integer>`）。
  - 适用场景：逻辑可通过类的静态方法直接实现时。
- 对象的实例方法引用：
  - 语法：`对象::实例方法`。
  - 示例：`System.out::println`（打印数据，对应`Consumer<Object>`）。
  - 适用场景：逻辑可通过某个对象的实例方法实现时。

#### 5. **中等偏难题**

**题目**：Lambda 表达式中，为什么引用的外部局部变量必须是 “effectively final”（隐式不可变）？这一约束与线程安全有什么关系？

**答案**：

- 原因：Lambda 可能在多线程中执行，若外部变量可变，Lambda 捕获的变量值可能与外部实际值不一致（Lambda 捕获的是变量的 “值” 而非 “引用”），违背函数式编程 “无副作用” 原则。
- 与线程安全的关系：限制变量为 “effectively final” 可避免多线程下的并发修改冲突，确保 Lambda 执行时变量值稳定，无需额外同步机制，提升线程安全性。

#### 6. **较难题**

**题目**：`Function<T,R>`接口的`compose`和`andThen`方法在执行顺序上有什么区别？请用代码示例说明两者的不同（如基于 “加 5” 和 “乘 2” 两个函数）。

**答案**：

- 执行顺序区别：

  - `compose(Function before)`：先执行`before`函数，再执行当前函数（`before → 当前`）。
  - `andThen(Function after)`：先执行当前函数，再执行`after`函数（`当前 → after`）。

- 代码示例：

  java

  ```java
  Function<Integer, Integer> add5 = num -> num + 5; // 加5
  Function<Integer, Integer> mul2 = num -> num * 2; // 乘2
  
  // compose：先加5，再乘2 → (3+5)*2=16
  System.out.println(mul2.compose(add5).apply(3)); 
  
  // andThen：先加5，再乘2 → (3+5)*2=16（与上例结果相同，逻辑顺序一致）
  System.out.println(add5.andThen(mul2).apply(3)); 
  
  // 交换函数后结果不同
  // compose：先乘2，再加5 → (3*2)+5=11
  System.out.println(add5.compose(mul2).apply(3)); 
  
  // andThen：先乘2，再加5 → (3*2)+5=11（与上例结果相同，逻辑顺序一致）
  System.out.println(mul2.andThen(add5).apply(3)); 
  ```

#### 7. **较难题**

**题目**：Lambda 表达式与匿名内部类在底层实现上有三个核心区别，请简述（从字节码生成、`this`指向、实例复用三个角度分析）。

**答案**：

- 字节码生成：

  - 匿名内部类：编译时生成独立的`XXX$1.class`文件。
  - Lambda：不生成独立类文件，通过`invokedynamic`指令动态绑定接口实例。

- ```
  this
  ```

  指向：

  - 匿名内部类：`this`指向匿名类自身实例。
  - Lambda：`this`指向包含 Lambda 的外部类实例。

- 实例复用：

  - 匿名内部类：每次使用都新建一个实例，开销较高。
  - Lambda：若捕获的外部变量相同，可复用实例，减少对象创建开销，性能更优。

#### 8. **难题**

**题目**：Stream 的 “中间操作” 和 “终止操作” 有什么本质区别？为什么说 “中间操作是惰性求值的”？请举例说明（如`filter`和`forEach`的配合）。

**答案**：

- 本质区别：

  - 中间操作（如`filter`、`map`）：返回`Stream`对象，仅记录操作逻辑，不实际处理数据。
  - 终止操作（如`forEach`、`collect`）：无返回值（或返回非`Stream`结果），触发整个流水线的执行（包括所有中间操作）。

- 中间操作是惰性求值的原因：中间操作仅 “注册” 逻辑，不执行实际计算，直到终止操作被调用才会一次性处理所有数据，减少无效计算（例如过滤后的数据无需执行后续中间操作）。

- 示例：

  java

  ```java
  List<Integer> nums = Arrays.asList(1, 2, 3, 4);
  // 仅中间操作，无输出（未执行）
  nums.stream().filter(num -> num % 2 == 0); 
  // 加终止操作forEach后，才执行过滤并打印（2、4）
  nums.stream().filter(num -> num % 2 == 0).forEach(System.out::println); 
  ```

#### 9. **难题**

**题目**：`ToIntFunction<T>`与`Function<T, Integer>`相比，在处理基本类型时为什么能提升性能？请结合 “自动装箱 / 拆箱” 的开销说明。

**答案**：

- `Function<T, Integer>`：接收`T`类型参数，返回`Integer`（包装类）。若实际需要`int`（基本类型），会触发**自动拆箱**（`Integer → int`），产生额外开销。

- `ToIntFunction<T>`：直接返回`int`（基本类型），无需装箱（`int → Integer`）和拆箱（`Integer → int`），减少包装类与基本类型的转换成本，因此性能更优。

- 示例：

  java

  ```java
  // Function有拆箱开销
  Function<String, Integer> func = Integer::parseInt;
  int num1 = func.apply("123"); // 拆箱：Integer → int
  
  // ToIntFunction无拆箱开销
  ToIntFunction<String> toIntFunc = Integer::parseInt;
  int num2 = toIntFunc.applyAsInt("123"); // 直接返回int，性能更优
  ```

#### 10. **最难**

**题目**：结合`Stream.map`方法的源码（` <R> Stream<R> map(Function<? super T, ? extends R> mapper)`），分析`Function`接口在 Stream 数据转换中的作用，以及为什么`map`方法能支持链式调用。

**答案**：

- `Function`接口的作用：作为数据转换的 “规则载体”，`map`方法接收`Function`实例（如 Lambda 或方法引用），将流中`T`类型元素转换为`R`类型元素（例如`String → Integer`的长度转换）。
- 支持链式调用的原因：`map`方法的返回值是`Stream<R>`（新的流对象），该对象可继续调用其他 Stream 方法（如`filter`、`another map`），形成流水线操作。
- 源码逻辑：`map`通过`Function`的`apply`方法对每个元素执行转换，封装为新的`Stream`实例返回，因此可链式调用后续操作。

      public String getName() { return name; }
      public int getAge() { return age; }
  }
  
  public class Test {
      public static void main(String[] args) {
          List<User> users = Arrays.asList(
              new User("张三", 20),
              new User("李四", 17),
              new User("王五", 25)
          );
  
          List<String> result = users.stream()
              .filter(user -> user.getAge() > 18) // 筛选成年用户
              .map(user -> user.getName().toUpperCase()) // 姓名转大写
              .collect(Collectors.toList()); // 收集结果
  
          System.out.println(result); // 输出：[张三, 王五]（实际为大写：[张三, 王五]的大写即"张三"→"张三"，中文无大小写，此处示例为逻辑正确）
      }
  }
  ```

#### 5. **场景五**

**题目**：请用 “方法引用” 优化以下代码：

java

```java
List<String> words = Arrays.asList("apple", "banana", "cherry");
words.stream().map(word -> word.toUpperCase()).forEach(word -> System.out.println(word));
```

说明优化后的代码及方法引用的类型，并解释为什么方法引用能简化 Lambda 表达式。

**答案**：

- 优化后的代码：

  java

  ```java
  List<String> words = Arrays.asList("apple", "banana", "cherry");
  words.stream()
       .map(String::toUpperCase) // 类的实例方法引用
       .forEach(System.out::println); // 对象的实例方法引用
  ```

  方法引用类型：

  - `String::toUpperCase`：类的实例方法引用（调用`String`对象的`toUpperCase`方法）。
  - `System.out::println`：对象的实例方法引用（调用`System.out`对象的`println`方法）。

- 简化原因：当 Lambda 体仅为 “调用一个已有方法” 时，方法引用可省略参数声明和`->`符号，直接通过 “类 / 对象：：方法名” 关联方法，代码更简洁且可读性更高。

# 项目

以下是基于 Java 8 函数式编程特性（Lambda、函数式接口、Stream 等）实现的 "员工信息管理系统" 小项目，代码量约 300 行，涵盖核心知识点的综合应用：

### 员工信息管理系统

#### 功能说明

- 支持添加员工信息（ID、姓名、年龄、薪资）
- 基于 Stream 实现多条件筛选（如年龄范围、薪资区间）
- 使用函数式接口自定义排序规则
- 统计员工平均薪资、最高 / 最低薪资
- 支持按字段（姓名 / 薪资）排序并打印

#### 代码实现

java

```java
import java.util.*;
import java.util.function.Function;
import java.util.stream.Collectors;

/**
 * 员工信息管理系统
 * 基于Java 8函数式编程特性实现（Lambda、Stream、Function接口等）
 */
public class EmployeeManagementSystem {
    // 存储员工信息的列表
    private static List<Employee> employees = new ArrayList<>();
    private static Scanner scanner = new Scanner(System.in);

    public static void main(String[] args) {
        // 初始化测试数据
        initTestData();

        // 主菜单循环
        while (true) {
            printMenu();
            int choice = getValidChoice(1, 6);
            switch (choice) {
                case 1:
                    addEmployee();
                    break;
                case 2:
                    filterEmployees();
                    break;
                case 3:
                    sortEmployees();
                    break;
                case 4:
                    statisticsSalary();
                    break;
                case 5:
                    printAllEmployees();
                    break;
                case 6:
                    System.out.println("系统退出，感谢使用！");
                    scanner.close();
                    return;
            }
        }
    }

    // 员工类（数据载体）
    static class Employee {
        private String id;
        private String name;
        private int age;
        private double salary;

        public Employee(String id, String name, int age, double salary) {
            this.id = id;
            this.name = name;
            this.age = age;
            this.salary = salary;
        }

        // Getter方法（供Stream操作使用）
        public String getId() { return id; }
        public String getName() { return name; }
        public int getAge() { return age; }
        public double getSalary() { return salary; }

        @Override
        public String toString() {
            return String.format("ID: %s, 姓名: %s, 年龄: %d, 薪资: %.2f",
                    id, name, age, salary);
        }
    }

    // 初始化测试数据
    private static void initTestData() {
        employees.add(new Employee("E001", "张三", 28, 8000));
        employees.add(new Employee("E002", "李四", 35, 12000));
        employees.add(new Employee("E003", "王五", 25, 6500));
    }

    // 打印功能菜单
    private static void printMenu() {
        System.out.println("\n===== 员工信息管理系统 =====");
        System.out.println("1. 添加员工");
        System.out.println("2. 筛选员工（年龄/薪资）");
        System.out.println("3. 排序员工（姓名/薪资）");
        System.out.println("4. 薪资统计（平均/最高/最低）");
        System.out.println("5. 查看所有员工");
        System.out.println("6. 退出系统");
        System.out.print("请选择功能（1-6）：");
    }

    // 获取合法的菜单选择
    private static int getValidChoice(int min, int max) {
        while (true) {
            try {
                int choice = Integer.parseInt(scanner.nextLine());
                if (choice >= min && choice <= max) {
                    return choice;
                } else {
                    System.out.printf("请输入%d-%d之间的数字：", min, max);
                }
            } catch (NumberFormatException e) {
                System.out.print("输入错误，请输入数字：");
            }
        }
    }

    // 1. 添加员工
    private static void addEmployee() {
        System.out.println("\n----- 添加新员工 -----");
        System.out.print("请输入员工ID：");
        String id = scanner.nextLine();

        // 检查ID唯一性（使用Stream的anyMatch）
        boolean idExists = employees.stream()
                .anyMatch(emp -> emp.getId().equals(id));
        if (idExists) {
            System.out.println("该ID已存在，添加失败！");
            return;
        }

        System.out.print("请输入姓名：");
        String name = scanner.nextLine();

        System.out.print("请输入年龄：");
        int age = Integer.parseInt(scanner.nextLine());

        System.out.print("请输入薪资：");
        double salary = Double.parseDouble(scanner.nextLine());

        employees.add(new Employee(id, name, age, salary));
        System.out.println("员工添加成功！");
    }

    // 2. 筛选员工（结合Stream的filter）
    private static void filterEmployees() {
        System.out.println("\n----- 筛选员工 -----");
        System.out.print("请输入最小年龄（含）：");
        int minAge = Integer.parseInt(scanner.nextLine());
        System.out.print("请输入最大年龄（含）：");
        int maxAge = Integer.parseInt(scanner.nextLine());
        System.out.print("请输入最低薪资（含）：");
        double minSalary = Double.parseDouble(scanner.nextLine());

        // 使用Stream多条件筛选
        List<Employee> filtered = employees.stream()
                .filter(emp -> emp.getAge() >= minAge && emp.getAge() <= maxAge)
                .filter(emp -> emp.getSalary() >= minSalary)
                .collect(Collectors.toList());

        System.out.println("\n筛选结果：");
        if (filtered.isEmpty()) {
            System.out.println("无符合条件的员工");
        } else {
            filtered.forEach(System.out::println); // 方法引用简化输出
        }
    }

    // 3. 排序员工（使用Function定义排序规则）
    private static void sortEmployees() {
        System.out.println("\n----- 员工排序 -----");
        System.out.println("1. 按姓名排序（字母顺序）");
        System.out.println("2. 按薪资排序（从高到低）");
        int sortType = getValidChoice(1, 2);

        // 使用Function定义排序字段提取规则
        Function<Employee, Comparable> keyExtractor;
        if (sortType == 1) {
            keyExtractor = Employee::getName; // 方法引用：提取姓名作为排序键
        } else {
            keyExtractor = Employee::getSalary; // 提取薪资作为排序键
        }

        // 排序并打印（使用Stream的sorted）
        List<Employee> sorted = employees.stream()
                .sorted((e1, e2) -> {
                    if (sortType == 2) {
                        // 薪资倒序（使用Comparator的reversed）
                        return keyExtractor.apply(e2).compareTo(keyExtractor.apply(e1));
                    } else {
                        return keyExtractor.apply(e1).compareTo(keyExtractor.apply(e2));
                    }
                })
                .collect(Collectors.toList());

        System.out.println("\n排序结果：");
        sorted.forEach(System.out::println);
    }

    // 4. 薪资统计（Stream的mapToDouble）
    private static void statisticsSalary() {
        System.out.println("\n----- 薪资统计 -----");
        if (employees.isEmpty()) {
            System.out.println("暂无员工数据！");
            return;
        }

        // 转换为薪资流进行统计
        DoubleSummaryStatistics stats = employees.stream()
                .mapToDouble(Employee::getSalary) // 提取薪资为DoubleStream
                .summaryStatistics();

        System.out.printf("员工总数：%d人\n", stats.getCount());
        System.out.printf("平均薪资：%.2f\n", stats.getAverage());
        System.out.printf("最高薪资：%.2f\n", stats.getMax());
        System.out.printf("最低薪资：%.2f\n", stats.getMin());
    }

    // 5. 查看所有员工
    private static void printAllEmployees() {
        System.out.println("\n----- 所有员工信息 -----");
        if (employees.isEmpty()) {
            System.out.println("暂无员工数据！");
        } else {
            employees.forEach(System.out::println); // 方法引用简化遍历
        }
    }
}
```

### 核心知识点应用说明

1. **Lambda 表达式**：菜单选择、排序逻辑中使用 Lambda 简化匿名内部类（如`sorted`方法的比较器）。
2. **函数式接口**：
   - 使用`Function<Employee, Comparable>`定义排序键提取规则，体现`Function`接口的 "输入 - 输出" 转换能力；
   - `Stream`操作中隐含使用`Predicate`（`filter`方法）、`Consumer`（`forEach`方法）等接口。
3. **方法引用**：通过`Employee::getName`、`System.out::println`简化 Lambda，提升代码可读性。
4. **Stream API**：
   - 中间操作：`filter`（筛选）、`sorted`（排序）、`mapToDouble`（转换为数值流）；
   - 终止操作：`collect`（收集结果）、`forEach`（遍历输出）、`summaryStatistics`（统计）。
5. **流式思维**：筛选、排序、统计均通过 Stream 链式调用完成，替代传统的 for 循环，代码更简洁。

### 使用说明

1. 运行程序后，通过菜单数字选择功能；
2. 添加员工时需输入 ID、姓名、年龄、薪资（ID 唯一）；
3. 筛选员工可指定年龄范围和最低薪资；
4. 排序支持按姓名（正序）或薪资（倒序）；
5. 薪资统计自动计算平均、最高、最低值。

该项目充分体现了 Java 8 函数式编程的优势，通过 Stream 和 Lambda 大幅简化了集合操作代码，同时保持逻辑清晰易读。
