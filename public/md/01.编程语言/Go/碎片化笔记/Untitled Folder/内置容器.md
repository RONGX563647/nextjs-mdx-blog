### 3-1 数组.mp4

#### 核心内容：Go 语言数组的定义、特性与基础使用

- **定义与声明**
  数组是长度固定、元素类型相同的序列，声明格式 `var 数组名 [长度]类型` ，如 `var arr [5]int` ，也可用短语法初始化 `arr := [3]int{1, 2, 3}` ，长度是类型一部分，`[3]int` 和 `[5]int` 是不同类型。

- **内存与特性**
  数组在内存中连续存储，长度编译期确定，一旦声明不可变（长度、类型固定）。通过索引访问元素，`arr[0]` 取第一个元素，索引从 0 开始，越界访问编译不报错，但运行时会 panic 。

- 遍历方式

  支持

  

  ```
  for
  ```

  

  循环遍历（通过索引），如：

  go

  

  ```go
  for i := 0; i < len(arr); i++ {
      fmt.Println(arr[i])
  }
  ```

  也可用

  

  ```
  for range
  ```

   

  简洁遍历：

  go

  ```go
  for index, value := range arr {
      fmt.Printf("索引 %d，值 %d\n", index, value)
  }
  ```

- **使用场景**
  适合存储长度固定、数据类型单一的简单序列，如固定大小的坐标点 `[2]int` 表示二维坐标，但因长度固定，实际开发中切片（动态数组）使用更频繁，数组常作为底层结构支撑切片。

### 3-2 切片的概念.mp4

#### 核心内容：切片的本质、与数组的关系，及核心特性

- **本质与底层结构**
  切片是数组的 “视图”，自身是引用类型，内部包含 **指针（指向数组起始位置）、长度（切片元素个数）、容量（从起始位置到数组末尾的元素个数）** 。声明格式 `var 切片名 []类型` ，如 `var s []int` ，可基于数组或通过 `make` 创建。
- **与数组的关联**
  基于数组创建切片，如 `arr := [5]int{1,2,3,4,5}` ，`s := arr[1:3]` （左闭右开，取索引 1、2 元素），切片修改会影响原数组（因共享底层数组）；用 `make` 创建切片（如 `s := make([]int, 3, 5)` ），底层会隐式创建匿名数组，长度 3（切片可用元素数）、容量 5（底层数组总长度）。
- 核心特性
  - **动态长度**：可通过 `append` 函数扩展，当容量不足时，会触发底层数组扩容（创建新数组，拷贝原数据，更新切片指针、长度、容量 ）。
  - **引用语义**：多个切片可共享同一底层数组，修改切片元素可能影响其他关联切片（需注意并发安全）。
- **使用场景**
  几乎替代数组的日常使用，适合存储动态变化、长度不固定的序列，如处理用户输入的不定长数据、动态构建的列表等，是 Go 语言中最常用的序列类型。

### 3-3 切片的操作.mp4

#### 核心内容：切片的创建、增删改查、扩容等实用操作

- 创建方式

  - 基于数组：`arr := [5]int{1,2,3,4,5}; s := arr[1:4]` （取索引 1 - 3 元素 ）。
  - 基于 `make`：`s := make([]int, 2, 4)` ，参数依次是类型、初始长度、容量，长度决定初始可用元素数（默认初始值，如 `int` 类型为 0 ），容量决定底层数组大小。
  - 直接初始化：`s := []int{1,2,3}` ，长度和容量均为 3 。

- 增（append）

  ```
  append
  ```

   

  函数用于添加元素，语法

  ```
  s = append(s, 新元素)
  ```

   

  。若容量足够，直接在底层数组末尾添加；容量不足时，自动扩容（一般扩容为原容量 2 倍，小切片可能按 1.25 倍等策略 ）。支持一次添加多个元素：

  go

  ```go
  s := []int{1,2}
  s = append(s, 3, 4) // 添加多个元素，s 变为 [1,2,3,4]
  ```

  还可拼接切片：

  go

  ```go
  s1 := []int{1,2}
  s2 := []int{3,4}
  s3 = append(s1, s2...) // s3 为 [1,2,3,4] ，... 展开切片
  ```

- 删（基于切片截取模拟删除）

  Go 无直接删除切片元素的语法，可通过切片截取 +

  ```
  append
  ```

  实现。如删除索引

  ```
  i
  ```

  

  元素：

  go

  ```go
  s := []int{1,2,3,4}
  i := 1
  s = append(s[:i], s[i+1:]...) // 先取 0~i-1 元素，再拼接 i+1~末尾元素，s 变为 [1,3,4]
  ```

- **改（直接索引赋值）**
  类似数组，通过索引修改元素值，`s[0] = 10` ，若索引越界会运行时 panic ，修改会影响共享底层数组的其他切片（若有）。

- **查（遍历或索引访问）**
  同数组，用 `for` 循环或 `for range` 遍历，或直接通过索引 `s[i]` 访问（需确保索引有效 ）。

- **扩容机制**
  当 `append` 导致长度超过容量时触发，新容量计算规则：原容量小于 1024 时，新容量为原容量 2 倍；原容量≥1024 时，新容量按 1.25 倍增长（具体依赖编译器优化，以实际扩容为准 ），扩容会拷贝原数据，有性能开销，需合理规划切片初始容量。

### 3-4 Map.mp4

#### 核心内容：Map 的定义、特性、基础操作，理解键值对存储

- 定义与声明

  Map 是无序的键值对集合，键需支持相等性比较（如基本类型、结构体需全部字段可比较，不支持切片、Map 等作为键 ），值类型不限。声明格式

  

  ```
  var m map[键类型]值类型
  ```

  

  ，如

  ```
  var m map[string]int
  ```

   

  ，注意声明后 Map 为

  ```
  nil
  ```

  ，需初始化才能使用，常用方式：

  - `make` 初始化：`m := make(map[string]int, 10)` ，第二个参数是初始容量（预分配空间，减少扩容次数 ）。
  - 字面量初始化：`m := map[string]int{"a":1, "b":2}` 。

- 核心特性

  - **无序性**：遍历 Map 时，键值对顺序不固定（每次遍历可能不同 ）。
  - **引用类型**：Map 本身是引用，赋值或传递函数参数时，传递的是引用，修改会影响原 Map 。
  - **自动扩容**：添加元素时，若当前容量不足，会自动扩容（类似切片，但扩容策略不同 ）。

- 基础操作

  - **增 / 改**：`m["key"] = value` ，若键存在则修改值，不存在则新增键值对。
  - **查**：`value, ok := m["key"]` ，`ok` 为 `bool` 类型，标记键是否存在，避免直接访问不存在键返回零值（如 `int` 类型返回 0 ，可能与实际值混淆 ）。
  - **删**：`delete(m, "key")` ，删除指定键值对，若键不存在，无报错（安全操作 ）。
  - **遍历**：`for k, v := range m { ... }` ，遍历所有键值对，顺序随机。

- **使用场景**
  适合存储关联型数据，如用户 ID 映射用户信息、统计单词频次（单词为键，次数为值 ）、配置项（键为配置名，值为配置内容 ）等，是处理键值关联数据的核心工具。

### 3-5 Map 例题.mp4

#### 核心内容：通过实际例题，巩固 Map 语法与应用逻辑，常见题型如：

- **题型 1：统计字符串中字符出现次数**
  需求：给定字符串，统计每个字符出现的频次。
  代码示例：

  go

  

  ```go
  s := "hello world"
  m := make(map[rune]int)
  for _, c := range s { // range 遍历字符串，返回 rune 类型字符
      m[c]++
  }
  for k, v := range m {
      fmt.Printf("字符 %c 出现 %d 次\n", k, v)
  }
  ```

  思路：用 Map 键存字符（`rune` 类型适配中文等），值存次数，遍历字符串逐个统计。

- **题型 2：去重并统计元素频次（切片转 Map 去重）**
  需求：给定整数切片，去重并统计每个元素出现次数。
  代码示例：

  go

  ```go
  nums := []int{1,2,2,3,3,3}
  m := make(map[int]int)
  for _, num := range nums {
      m[num]++
  }
  // 去重结果可转切片：
  uniqueNums := make([]int, 0, len(m))
  for k := range m {
      uniqueNums = append(uniqueNums, k)
  }
  fmt.Println("去重后：", uniqueNums)
  fmt.Println("频次：", m)
  ```

  思路：利用 Map 键唯一性去重，同时记录值（次数 ），再按需转换数据结构。

- **题型 3：Map 作为函数参数，修改共享数据**
  需求：定义函数，向 Map 添加键值对，外部 Map 能看到修改。
  代码示例：

  go

  

  ```go
  func addEntry(m map[string]string, key, value string) {
      m[key] = value // Map 是引用类型，直接修改原数据
  }
  func main() {
      m := make(map[string]string)
      addEntry(m, "name", "Alice")
      fmt.Println(m) // 输出 map[name:Alice]
  }
  ```

  思路：体现 Map 引用特性，函数内修改会影响外部，与切片引用语义类似。

这些例题覆盖 Map 常见操作（增、查、遍历 ）、特性（引用类型 ）及与其他数据结构（字符串、切片 ）的结合，帮你掌握实际开发中 Map 的典型用法。

### 3-6 字符和字符串处理.mp4

#### 核心内容：Go 语言中字符串的存储、常用操作（拼接、截取、遍历等 ），及字符（`rune` ）与字符串的转换

- **字符串底层与存储**
  Go 字符串是 **只读的字节切片**（`[]byte` ），默认使用 UTF-8 编码存储字符。每个字符可能占 1 - 4 字节（如英文字母占 1 字节，中文占 3 字节 ），`len("你好")` 结果是 6（2 个中文，每个 3 字节 ）。

- 字符遍历（区分字节与 `rune` ）

  - 按字节遍历：`for i := 0; i < len(s); i++ { c := s[i]; ... }` ，获取的是字节值，非字符实际编码，适合处理 ASCII 字符。
  - 按 `rune` 遍历：`for _, c := range s { ... }` ，`range` 会自动解码 UTF-8 ，`c` 是 `rune` 类型（对应 Unicode 码点 ），能正确遍历中文、 emoji 等字符。

- 常用操作

  - 拼接

    ：

    - 用 `+` 拼接：`s := "hello" + " " + "world"` ，简单但多次拼接（如循环内 ）效率低（每次创建新字符串 ）。

    - 用

      

      ```
      strings.Builder
      ```

       

      高效拼接：

      go

      ```go
      var b strings.Builder
      b.WriteString("hello")
      b.WriteByte(' ') // 单个字节
      b.WriteRune('世') // 单个 rune（支持中文）
      result := b.String() // 输出 "hello 世"
      ```

  - 截取

    ：

    ```
    s := "hello world"; sub := s[0:5]
    ```

     

    （左闭右开，取前 5 字节 ），注意按字节截取，若处理多字节字符（如中文 ），直接截取可能导致乱码，需结合

     

    ```
    rune
    ```

     

    转换：

    go

    

    ```go
    s := "你好，世界"
    runes := []rune(s)
    subRune := runes[0:2] // 取前 2 个字符（"你好"）
    subStr := string(subRune)
    ```

  - 查找与替换

    ：

    - 查找子串：`idx := strings.Index(s, "world")` ，返回子串起始索引，不存在返回 -1 ；`strings.Contains(s, "world")` 判断是否包含。
    - 替换：`newS := strings.Replace(s, "old", "new", -1)` ，最后参数是替换次数，-1 表示全部替换。

  - 分割与连接

    ：

    - 分割：`parts := strings.Split(s, ",")` ，按分隔符分割为切片。
    - 连接：`joined := strings.Join(parts, ";")` ，用分隔符连接切片元素为字符串。

- 字符与字符串转换

  - `rune` 转字符串：`s := string('A')` 、`s := string(22823)` （直接用 Unicode 码点 ）。
  - 字符串转 `rune` 切片：`runes := []rune(s)` ，方便按字符处理多语言内容。

- **使用场景**
  涵盖日常文本处理（日志分析、用户输入解析 ）、多语言支持（中文、 emoji 等字符操作 ）、高效字符串构建（如拼接大量字符串用 `strings.Builder` ），是处理文本数据的基础工具，与切片、Map 结合可实现复杂文本处理逻辑（如统计词频结合 Map ，分割字符串结合切片 ）。

通过系统学习这些内容，能掌握 Go 语言中数组、切片、Map、字符串的核心用法，构建起处理基础数据结构的知识体系，为后续开发复杂程序（如 Web 服务、工具脚本 ）打好基础，实际开发中多结合场景练习（如用切片处理动态列表、用 Map 做缓存 ），可加深理解～
