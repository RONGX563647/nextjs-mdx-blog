  该视频主要讲述了Go语言中的for循环。Go语言只提供了一种循环结构，即for循环，而没有像其他编程语言那样的while循环。视频详细解释了for循环的语法结构，并通过一个例子演示了如何使用for循环打印一系列数字。此外，视频还提到，虽然for循环在Go语言中是唯一的循环结构，但其灵活性和强大的功能使其能够替代其他编程语言中的多种循环结构。

  00:01for循环的基础用法

  1.for循环是大多数编程语言中唯一的循环结构，不同于其他语言提供for和while两种循环。 2.Go语言设计之初为了减少关键字，只提供了for循环，认为其可以替代while循环。 3.for循环的写法与其他编程语言类似，包括初始化语句、条件判断和循环逻辑。

  01:56for循环的初始化语句

  1.初始化语句用于定义循环变量，可以在循环体内定义临时变量，其作用域限于for循环。 2.初始化语句不是必须的，如果循环变量有默认值（如int类型的默认值为0），则可以省略。 3.初始化语句的灵活性使得for循环能够适应不同的循环场景。

  04:59for循环的死循环实现

  1.通过在循环体内使用sleep函数，可以实现在Go语言中的死循环，类似于while循环。 2.sleep函数的参数为duration类型，本质上是int64，常用time.Second或time.Millisecond指定时间单位。 3.通过比较循环变量的值，可以在for循环中实现类似于while循环的条件判断和退出逻辑。

  该视频主要讲述了`go to`语句在编程中的用法及其特点。首先，介绍了`go to`语句可以实现代码的灵活跳转，但其使用可能导致代码可读性降低和难以维护。其次，通过嵌套循环的例子，展示了`break`语句无法满足某些情况下结束整个循环的需求，而`go to`语句可以实现这一功能。最后，强调了尽管`go to`语句在某些场景下（如错误处理）可能有用，但通常并不建议使用，因为其可能导致代码结构混乱。

## Go 语言的条件语句（`if`）和循环语句（`for`）

是控制程序流程的核心结构。与其他语言相比，Go 的语法更加简洁，去掉了冗余的括号和分号，同时保留了强大的流程控制能力。下面详细讲解这两种语句的用法、特性及最佳实践。

### 一、条件语句（`if`）

`if` 语句用于根据条件执行不同的代码分支，Go 中的 `if` 语法简洁且灵活，支持在条件判断前执行一个初始化语句。

#### 1. 基本语法

go

```go
if 条件表达式 {
    // 条件为 true 时执行
} else if 另一个条件表达式 {
    // 前一个条件为 false，当前条件为 true 时执行
} else {
    // 所有条件都为 false 时执行
}
```

**特点**：

- 条件表达式不需要用括号 `()` 包裹（但表达式本身可以包含括号）
- 代码块必须用大括号 `{}` 包裹，即使只有一行代码
- `else` 和 `else if` 必须与上一个代码块的右大括号 `}` 在同一行（否则会编译错误）

**基础示例**：

go

```go
package main

import "fmt"

func main() {
	score := 85

	// 基本 if-else
	if score >= 90 {
		fmt.Println("优秀")
	} else if score >= 60 {
		fmt.Println("及格")
	} else {
		fmt.Println("不及格")
	}
	// 输出：及格
}
```

#### 2. 带初始化语句的 `if`

Go 允许在 `if` 条件前添加一个初始化语句，用于声明或计算临时变量，语法：

go

```go
if 初始化语句; 条件表达式 {
    // 代码块
}
```

初始化语句声明的变量**仅在当前 `if` 代码块（包括 `else`/`else if`）中有效**。

**示例**：

go

```go
func main() {
	// 初始化语句：计算平均分并声明变量 avg
	if avg := (85 + 92 + 78) / 3; avg >= 80 {
		fmt.Printf("平均分 %d，优秀\n", avg) // 变量 avg 在此处可用
	} else {
		fmt.Printf("平均分 %d，良好\n", avg) // else 中也可用
	}

	// fmt.Println(avg) // 错误：avg 在此处已超出作用域
}
// 输出：平均分 85，优秀
```

**常见用途**：

- 简化代码（将变量声明和条件判断合并）

- 限制变量作用域（避免污染外部命名空间）

- 配合错误处理（检查函数返回的

   

  ```
  error
  ```

  ）：

  go

  ```go
  // 示例：检查文件打开错误
  if file, err := os.Open("data.txt"); err != nil {
      fmt.Println("文件打开失败：", err)
  } else {
      defer file.Close()
      fmt.Println("文件打开成功")
      // 操作文件...
  }
  ```

#### 3. 特殊注意事项

- **无括号要求**：条件表达式周围不需要括号，但逻辑复杂时可加括号提高可读性：

  go

  ```go
  if (a > b && c == d) || (e != f) {
      // 复杂条件加括号更清晰
  }
  ```
  
- **`else` 位置**：`else` 必须紧跟在 `if` 代码块的 `}` 之后，否则会报错：

  go

  ```go
  // 错误写法（else 单独成行）
  if score > 60 {
      fmt.Println("及格")
  }
  else { // 编译错误：syntax error: unexpected else, expecting }
      fmt.Println("不及格")
  }
  
  // 正确写法（else 与 } 在同一行）
  if score > 60 {
      fmt.Println("及格")
  } else {
      fmt.Println("不及格")
  }
  ```

### 二、循环语句（`for`）

Go 语言中**只有 `for` 一种循环语句**，但通过不同的语法形式，可以实现其他语言中 `while`、`do-while` 等循环的功能，非常灵活。

#### 1. 基本 for 循环（类似 C 语言的 for）

语法：

go

```go
for 初始化语句; 条件表达式; 后置语句 {
    // 循环体
}
```

- 初始化语句：在循环开始前执行一次（通常用于声明循环变量）
- 条件表达式：每次循环前判断，为 `true` 则执行循环体
- 后置语句：每次循环体执行后执行（通常用于更新循环变量）

**示例**：

go

```go
func main() {
	// 打印 0-4 的数字
	for i := 0; i < 5; i++ {
		fmt.Printf("%d ", i)
	}
	// 输出：0 1 2 3 4
}
```

#### 2. 类似 while 的循环（省略初始化和后置语句）

如果省略初始化和后置语句，`for` 就变成了 `while` 循环：

go

```go
for 条件表达式 {
    // 循环体
}
```

**示例**：

go

```go
func main() {
	count := 0
	// 类似 while (count < 5)
	for count < 5 {
		fmt.Printf("%d ", count)
		count++
	}
	// 输出：0 1 2 3 4
}
```

#### 3. 无限循环（省略所有条件）

如果省略所有三个部分（初始化、条件、后置），`for` 就变成无限循环，需在循环体内用 `break` 终止：

go

```go
for {
    // 循环体（需包含 break 或 return 否则会无限执行）
}
```

**示例**：

go

```go
func main() {
	count := 0
	// 无限循环
	for {
		if count >= 5 {
			break // 满足条件时退出循环
		}
		fmt.Printf("%d ", count)
		count++
	}
	// 输出：0 1 2 3 4
}
```

#### 4. 遍历集合（`for range`）

`for range` 是 Go 中遍历集合的专用语法，支持字符串、数组、切片、map、通道（channel）等。语法：

go

```go
for 索引, 值 := range 集合 {
    // 循环体
}
```

- 对于**字符串**：索引是字符的字节位置，值是 `rune` 类型（Unicode 码点）
- 对于**数组 / 切片**：索引是元素位置，值是元素值
- 对于**map**：索引是键（key），值是值（value）
- 对于**通道**：只有值，没有索引（接收通道中的数据，直到通道关闭）

**示例 1：遍历字符串**

go

```go
func main() {
	s := "hello 世界"
	// 遍历字符串（每个字符是 rune 类型）
	for i, c := range s {
		fmt.Printf("索引 %d: 字符 %c (ASCII码 %d)\n", i, c, c)
	}
}
```

输出：

plaintext

```plaintext
索引 0: 字符 h (ASCII码 104)
索引 1: 字符 e (ASCII码 101)
索引 2: 字符 l (ASCII码 108)
索引 3: 字符 l (ASCII码 108)
索引 4: 字符 o (ASCII码 111)
索引 5: 字符  (ASCII码 32)
索引 6: 字符 世 (ASCII码 19990)
索引 9: 字符 界 (ASCII码 30028)
```

（注意：中文字符占 3 个字节，所以索引会跳变）

**示例 2：遍历切片和 map**

go

```go
func main() {
	// 遍历切片
	nums := []int{10, 20, 30}
	for i, v := range nums {
		fmt.Printf("切片索引 %d: 值 %d\n", i, v)
	}

	// 遍历map（键值对顺序不固定）
	m := map[string]int{"a": 1, "b": 2}
	for k, v := range m {
		fmt.Printf("map键 %s: 值 %d\n", k, v)
	}
}
```

**注意**：

- 遍历中如果不需要索引或值，可用下划线

   

  ```
  _
  ```

   

  忽略（

  ```
  _
  ```

   

  是 Go 中的空白标识符，用于丢弃不需要的返回值）：

  go

  ```go
for _, v := range nums { // 只需要值，忽略索引
      fmt.Println(v)
}
  ```

- 遍历 map 时，每次顺序可能不同（map 是无序的）

- 遍历字符串时，

  ```
  for range
  ```

  会自动处理多字节字符（

  ```
  rune
  ```
  
  ），而普通

   

  ```
  for
  ```
  
   

  循环按字节遍历：

  go

  ```go
  s := "世界"
  // 普通for循环（按字节遍历，得到3个字节）
  for i := 0; i < len(s); i++ {
      fmt.Printf("%x ", s[i]) // 输出：e4 b8 96 e7 95 8c
  }
  // for range（按字符遍历，得到2个rune）
  for _, c := range s {
      fmt.Printf("%c ", c) // 输出：世 界
  }
  ```

#### 5. 循环控制语句（`break`、`continue`、`goto`）

Go 提供了三种循环控制语句，用于改变循环的执行流程：

##### （1）`break`：终止当前循环

`break` 用于立即退出当前循环（不再执行后续迭代）：

go

```go
func main() {
	for i := 0; i < 10; i++ {
		if i == 5 {
			break // 当i=5时，终止循环
		}
		fmt.Printf("%d ", i)
	}
	// 输出：0 1 2 3 4
}
```

##### （2）`continue`：跳过当前迭代

`continue` 用于跳过当前循环体的剩余部分，直接进入下一次迭代：

go

```go
func main() {
	for i := 0; i < 5; i++ {
		if i%2 == 0 {
			continue // 跳过偶数
		}
		fmt.Printf("%d ", i)
	}
	// 输出：1 3
}
```

##### （3）`goto`：跳转到标签位置

`goto` 用于跳转到循环内的标签位置，可实现复杂的循环控制（如跳出多层循环）：

```go
func main() {
	// 外层循环
	for i := 0; i < 3; i++ {
		// 内层循环
		for j := 0; j < 3; j++ {
			if i == 1 && j == 1 {
				goto exitLoop // 跳转到exitLoop标签
			}
			fmt.Printf("(%d,%d) ", i, j)
		}
	}
exitLoop: // 标签定义（需单独成行，后面跟冒号）
	fmt.Println("\n退出循环")
}
```

输出：

plaintext

```plaintext
(0,0) (0,1) (0,2) (1,0) 
退出循环
```

**注意**：`goto` 应谨慎使用，过度使用会导致代码逻辑混乱。

### 三、最佳实践与常见错误

#### 1. 条件语句的最佳实践

- 当条件判断涉及函数调用时，避免重复计算：

  go

  ```go
// 不推荐：重复调用 len(s)
  if len(s) > 0 && s[0] == 'a' { ... }

  // 推荐：用初始化语句缓存结果
if l := len(s); l > 0 && s[0] == 'a' { ... }
  ```

- 错误处理优先返回（提前退出），减少嵌套层级：

  go

  ```go
  // 不推荐：多层嵌套
  func doSomething() error {
      if err := step1(); err == nil {
          if err := step2(); err == nil {
              // 业务逻辑
              return nil
          } else {
              return err
          }
      } else {
          return err
      }
  }
  
  // 推荐：提前返回，减少嵌套
  func doSomething() error {
      if err := step1(); err != nil {
          return err
      }
      if err := step2(); err != nil {
          return err
      }
      // 业务逻辑
      return nil
  }
  ```

#### 2. 循环语句的常见错误

- **在循环中修改循环变量的副本**：`for range` 中的值是元素的副本，修改它不会改变原集合：

  go

  ```go
  nums := []int{1, 2, 3}
  for _, v := range nums {
      v *= 2 // 修改的是副本，原切片不变
  }
  fmt.Println(nums) // 输出：[1 2 3]（未改变）
  
  // 正确做法：通过索引修改
  for i := range nums {
      nums[i] *= 2
  }
  fmt.Println(nums) // 输出：[2 4 6]
  ```
  
- **循环中创建 goroutine 引用循环变量**：循环变量在每次迭代中会被重用，goroutine 可能捕获到同一个变量的地址：

  go

  ```go
  // 错误示例：所有goroutine可能打印相同的i（最后一个值）
  for i := 0; i < 3; i++ {
      go func() {
          fmt.Println(i) // 风险：i是共享变量
      }()
  }
  time.Sleep(time.Second) // 等待goroutine执行
  
  // 正确示例：通过参数传递当前值
  for i := 0; i < 3; i++ {
      go func(num int) {
          fmt.Println(num) // 安全：参数是当前值的副本
      }(i)
  }
  time.Sleep(time.Second)
  ```
  
  ### 总结

Go 语言的 `if` 和 `for` 语句具有简洁、灵活的特点：

- `if` 支持初始化语句，可在条件判断前声明临时变量，简化代码并限制变量作用域
- `for` 是唯一的循环语句，通过不同形式可实现普通循环、`while` 循环和无限循环
- `for range` 专为遍历集合设计，能优雅处理字符串、数组、切片、map 等
- 循环控制可使用 `break`（终止）、`continue`（跳过）和 `goto`（跳转）

掌握这些语句的用法，能帮助你编写清晰、高效的 Go 代码，尤其是 `for range` 遍历和 `if` 初始化语句的使用，是 Go 语言的特色编程风格。

     } else {
         // 所有条件为false时执行
     }
     ```

2. **问题**：Go 的`if`语句与其他语言（如 Java）相比，有什么特殊之处？
   **答案**：

   - 可在条件表达式前加初始化语句，作用域仅限于

     ```
     if
     ```

     块：

     go

     ```go
   if err := doSomething(); err != nil {
         // 处理错误
   }
     ```

   - 无三元运算符（`a ? b : c`），需用`if-else`替代。

   - 必须使用大括号（即使单语句），不允许省略。

3. **问题**：`switch`语句的基本语法是什么？默认情况下`case`分支是否需要`break`？
   **答案**：

   - 基本语法：

     go

     ```go
     switch 表达式 {
     case 值1:
         // 执行语句
     case 值2, 值3: // 多值匹配
         // 执行语句
     default:
         // 无匹配时执行
     }
     ```
     
   - 默认行为：`case`执行完毕后自动`break`（不穿透），如需穿透需显式使用`fallthrough`。
   
4. **问题**：`switch`语句可以省略表达式吗？这种情况下如何使用？
   **答案**：

   - 可以省略表达式，此时

     ```
     switch
     ```

     等价于

     ```
     switch true
     ```

     ，用于多条件判断：

     go

     ```go
     age := 20
     switch {
     case age < 18:
         fmt.Println("未成年")
     case age >= 18 && age < 60:
         fmt.Println("成年")
     default:
         fmt.Println("老年")
     }
     ```

5. **问题**：Go 语言的循环语句有哪几种形式？`for`循环的基本语法是什么？
   **答案**：

   - 形式：仅`for`循环（无`while`、`do-while`，但可模拟）。
   - 基本语法：
     1. 完整形式：`for 初始化; 条件; 后处理 { ... }`
     2. 类似`while`：`for 条件 { ... }`
     3. 无限循环：`for { ... }`（需用`break`终止）

6. **问题**：如何使用`for`循环遍历数组、切片、字符串和`map`？
   **答案**：

   - 数组 / 切片 / 字符串（索引 + 值）：

     go

     ```go
   for i, v := range 集合 { ... } // i:索引, v:值
     ```

   - ```
     map
     ```

     （键 + 值）：

     go

     ```go
     for k, v := range m { ... } // k:键, v:值
     ```

   - 忽略索引 / 键：`for _, v := range 集合 { ... }`
   
   - 忽略值：`for i := range 集合 { ... }`

7. **问题**：`break`和`continue`语句的作用是什么？有什么区别？
   **答案**：

   - `break`：终止当前循环（或`switch`），跳出循环体。
   - `continue`：跳过当前循环的剩余语句，直接进入下一次循环。
   - 区别：`break`完全终止循环，`continue`仅跳过当前迭代。

8. **问题**：如何在多层循环中使用`break`终止外层循环？
   **答案**：
   使用标签（label）配合`break`：

   go

   ```go
   outer:
   for i := 0; i < 3; i++ {
       for j := 0; j < 3; j++ {
           if i*j > 3 {
               break outer // 终止外层循环
           }
           fmt.Println(i, j)
       }
   }
   ```

#### 中级篇（9-15 题）

1. **问题**：`for range`遍历与普通`for`循环（索引）遍历有何异同？适用场景分别是什么？
   **答案**：

   - 相同点：均可遍历集合元素。
   - 不同点：
     - `for range`：简洁，返回值为**副本**（修改不影响原集合），适合读取。
     - 索引遍历：可修改原集合（通过索引），适合需要索引或修改元素的场景。
   - 适用场景：
     - **读取操作：优先`for range`**。
     - 修改操作或需要复杂索引计算：用索引遍历。

2. **问题**：`switch`语句中`case`的值可以是什么类型？对类型有什么要求？
   **答案**：

   - 允许类型：整数、浮点数、字符串、符文（`rune`）、布尔值、常量表达式。
   - 要求：
     - 所有`case`值类型必须与`switch`表达式类型一致。
     - `case`值必须是可比较的（不能是`slice`、`map`、`function`等）。
     - `case`值不能重复。

3. **问题**：`fallthrough`语句的作用是什么？使用时有什么限制？
   **答案**：

   - 作用：使`switch`的`case`执行完毕后继续执行下一个`case`**（穿透）。**
   - 限制：
     - **必须放在`case`块的最后一行**。
     - 不能用于最后一个`case`或`default`。
     - 下一个`case`不会重新判断条件（无条件执行）。

4. **问题**：如何使用循环实现斐波那契数列（前 n 项）？需要注意什么？
   **答案**：
   实现示例：

   go

   运行

   

   ```go
   func fibonacci(n int) []int {
       if n <= 0 {
           return []int{}
       }
       fib := make([]int, n)
       if n >= 1 {
           fib[0] = 1
       }
       if n >= 2 {
           fib[1] = 1
       }
       for i := 2; i < n; i++ {
           fib[i] = fib[i-1] + fib[i-2]
       }
       return fib
   }
   ```
   
   注意：整数溢出（大 n 时），可使用`uint64`或`big.Int`避免。
   
5. **问题**：`for range`遍历字符串时的行为是什么？如何处理中文字符？
   **答案**：

   - 行为：自动按 UTF-8 解码，每次迭代返回字节索引和`rune`（Unicode 码点）。

   - 处理中文：

     ```
     for range
     ```

     天然支持多字节字符，无需额外处理：

     go

     ```go

     for i, r := range s {
       fmt.Printf("索引%d: %c\n", i, r) // 正确输出每个中文字符
     }
   ```
   
6. **问题**：如何使用`for`循环实现`do-while`的功能（至少执行一次循环体）？
   **答案**：
   用无限循环配合`break`模拟：

   go

   ```go

   for {
       // 循环体（至少执行一次）
       fmt.Print("请输入：")

7. **问题**：`switch`与`if-else if`在性能和可读性上有何区别？如何选择？
   **答案**：

   - 性能：`switch`对常量值判断更高效（编译期优化为跳转表），`if-else if`适合范围判断。
   - 可读性：多值离散判断用`switch`更清晰，范围或复杂条件用`if-else if`。
   - 选择原则：离散值匹配优先`switch`，范围或逻辑复杂条件用`if-else`。

1. **问题**：`for range`遍历`map`时的顺序是固定的吗？如何实现`map`的有序遍历？
   **答案**：

   - 顺序：无序（每次遍历顺序可能不同，Go 1.21 + 可通过`maps.Sorted`获取有序键）。

   - 有序遍历：

     go

     ```go
     // 1. 提取所有键
     keys := make([]string, 0, len(m))
     for k := range m {
         keys = append(keys, k)
     }
     // 2. 排序键
     sort.Strings(keys)
     // 3. 按排序后的键遍历
     for _, k := range keys {
         fmt.Println(k, m[k])
     }
     ```
   
2. **问题**：如何在循环中安全地删除`map`元素？删除后继续遍历是否有影响？
   **答案**：

   - 安全删除：遍历中可直接`delete(m, k)`，不会引发 panic。
   - 遍历影响：已遍历的元素不会重复出现，未遍历的元素可能被跳过（不保证全部遍历）。
   - 最佳实践：如需在遍历中删除并处理所有元素，可先复制键列表再遍历删除。

3. **问题**：`select`语句的作用是什么？它与`switch`有何异同？
   **答案**：

   - 作用：用于处理多个 channel 的读写操作，实现非阻塞通信。
   - 异同：
     - 相同：多分支选择，默认`case`可选。
     - 不同：`select`的`case`必须是 channel 操作；随机选择一个就绪的`case`；无`fallthrough`。

4. ### **问题**：如何使用`select`实现超时控制？请举例说明。
   
   **答案**：
   结合`time.After`实现超时：
   
   go
   
   ```go
   ch := make(chan string)
   
   go func() {
       time.Sleep(2 * time.Second)
       ch <- "任务完成"
   }()
   
   select {
   case res := <-ch:
       fmt.Println(res)
   case <-time.After(1 * time.Second): // 1秒超时
       fmt.Println("超时")
   }
   ```
   
5. **问题**：循环中的变量捕获可能导致什么问题？如何避免？
   **答案**：

   - 问题：循环变量在每次迭代中复用同一地址，goroutine 捕获时可能获取到最后一个值：

     go

     ```go
   for i := 0; i < 3; i++ {
         go func() {
           fmt.Println(i) // 可能输出3个3，而非0,1,2
         }()
   }
     ```

   - 避免：将循环变量作为参数传递给 goroutine：

     go

     ```go
     for i := 0; i < 3; i++ {
         go func(val int) {
             fmt.Println(val) // 正确输出0,1,2
         }(i)
     }
     ```

# Go 通道（`chan`）的深入使用指南

通道（`chan`）是 Go 语言实现**并发安全通信**的核心工具，遵循 “*不要通过共享内存通信，而要通过通信共享内存*” 的 CSP（Communicating Sequential Processes）并发模型，让 goroutine 间的协作更简洁可靠。

## 一、通道的基础：创建与核心操作

### 1. 通道的创建

通道是**引用类型**，需通过 `make` 函数初始化，格式为：

go

```go
make(chan 元素类型, [缓冲大小])
```

- **无缓冲通道**：`ch := make(chan int)`（缓冲大小省略或为 0），收发需 “同步”。
- **有缓冲通道**：`ch := make(chan int, 3)`（缓冲大小为 3，可暂存 3 个 `int` 类型数据），收发可 “异步”。

⚠️ 注意：未初始化的通道（如 `var ch chan int`）默认值为 `nil`，对 `nil` 通道执行收发操作会**永久阻塞**，因此必须初始化后再使用。

### 2. 发送与接收操作

通道的收发都使用 `<−` 操作符，方向由操作符与通道的相对位置决定：

- **发送数据**：`通道 <- 数据`（操作符在通道右侧，数据流向通道）。

  go

  ```go
  ch := make(chan int)
  ch <- 10 // 向通道发送整数 10
  ```

- **接收数据**：`变量 <- 通道`（操作符在通道左侧，数据从通道流出）。
  有三种接收形式：

  - 直接接收并赋值：`val := <-ch`

  - 接收但忽略值：`<-ch`

  - 带 “是否有效” 判断（检测通道是否关闭且无数据）：

    ```
    val, ok := <-ch
    ```

    - `ok = true`：通道正常，`val` 为有效数据；
    - `ok = false`：通道**已关闭且无剩余数据**，`val` 为对应类型的零值（如 `int` 的 `0`）。

### 3. 关闭通道

使用 `close` 函数关闭通道，语法：`close(通道)`。

- 关闭后**不能再向通道发送数据**，否则触发 `panic`；
- 关闭后**仍可从通道接收剩余数据**，直到数据被取完；
- 数据取完后，继续接收会得到**类型零值 + `ok=false`**。

示例：

go

```go
ch := make(chan int, 2)
ch <- 10
ch <- 20
close(ch)

// 接收剩余数据
fmt.Println(<-ch) // 输出：10
fmt.Println(<-ch) // 输出：20
val, ok := <-ch   // val=0（int 零值），ok=false
fmt.Println(val, ok)
```

## 二、无缓冲通道 vs 有缓冲通道

### 1. 无缓冲通道（阻塞通道）

- 缓冲大小为 0（或省略），

  发送和接收必须 “同步”

  ：

  - 发送方执行 `ch <- 数据` 时，会阻塞直到有接收方执行 `<-ch`；
  - 接收方执行 `<-ch` 时，会阻塞直到有发送方执行 `ch <- 数据`。

示例（同步通信）：

go

```go
func main() {
    ch := make(chan int) // 无缓冲通道
    go func() {
        fmt.Println("子协程：准备发送数据")
        ch <- 100 // 阻塞，直到主协程接收
        fmt.Println("子协程：发送完成")
    }()

    fmt.Println("主协程：准备接收数据")
    val := <-ch // 阻塞，直到子协程发送
    fmt.Println("主协程：收到数据", val)
    time.Sleep(time.Second) // 防止程序提前退出
}
```

输出顺序：

plaintext

```plaintext
主协程：准备接收数据
子协程：准备发送数据
子协程：发送完成
主协程：收到数据 100
```

### 2. 有缓冲通道（异步通道）

- 缓冲大小 > 0，

  发送和接收可 “异步”

  ：

  - 发送方：只要缓冲区未满，可连续发送，直到缓冲区满时阻塞；
  - 接收方：只要缓冲区非空，可连续接收，直到缓冲区空时阻塞。

示例（异步通信）：

```go
func main() {
    ch := make(chan int, 2) // 缓冲大小为 2
    ch <- 10  // 缓冲区未满，发送不阻塞
    ch <- 20  // 缓冲区仍未满，发送不阻塞
    // ch <- 30 // 缓冲区已满，取消注释会阻塞

    fmt.Println(<-ch) // 输出：10（缓冲区非空，接收不阻塞）
    fmt.Println(<-ch) // 输出：20
    // fmt.Println(<-ch) // 缓冲区空，取消注释会阻塞
}
```

## 三、通道的高级用法

### 1. `for range` 遍历通道

`for range` 会**持续从通道接收数据**，直到通道被关闭且所有数据取完，自动退出循环。

示例：

```go
func main() {
    ch := make(chan int, 3)
    ch <- 10
    ch <- 20
    ch <- 30
    close(ch) // 发送方关闭通道，通知接收方“数据发完”

    for val := range ch {
        fmt.Println(val)
    }
    // 输出：10 → 20 → 30（通道关闭后，循环自动退出）
}
```

### 2. 单向通道

可将通道**限制为 “只发送” 或 “只接收”**，增强代码安全性（如函数参数仅允许特定操作）。

- 只发送通道：`chan<- 元素类型`（只能发送，不能接收）；
- 只接收通道：`<-chan 元素类型`（只能接收，不能发送）。

示例（函数参数限制通道方向）：

go

```go
// 生产者：只能向通道发送数据
func producer(ch chan<- int) {
    ch <- 100
    ch <- 200
    close(ch)
}

// 消费者：只能从通道接收数据
func consumer(ch <-chan int) {
    for val := range ch {
        fmt.Println("消费：", val)
    }
}

func main() {
    ch := make(chan int, 2)
    go producer(ch)
    consumer(ch)
}
```

### 3. `select` 与通道（多路复用）

`select` 可同时监听**多个通道的收发操作**，类似 “多路复用器”，常用于：

- 实现**非阻塞收发**；
- 为通道操作添加**超时控制**；
- 处理 “多个通道中任意一个就绪” 的场景。

语法类似 `switch`，但 `case` 必须是通道的收发操作：

```go
select {
case val := <-ch1:
    fmt.Println("从 ch1 收到：", val)
case ch2 <- 20:
    fmt.Println("向 ch2 发送成功")
case <-time.After(time.Second): // 超时控制
    fmt.Println("操作超时")
default:
    fmt.Println("所有通道都未就绪")
}
```

示例（超时控制）：

```go
func main() {
    ch := make(chan int)
    select {
    case <-ch:
        fmt.Println("收到数据")
    case <-time.After(1 * time.Second):
        fmt.Println("等待超时，通道无数据")
    }
}
```

输出：`等待超时，通道无数据`（因 `ch` 无数据，超时后触发 `case`）。

## 四、基于通道的并发模式

### 1. 生产者 - 消费者模式

生产者持续向通道发送数据，消费者从通道读取并处理，通过通道解耦生产和消费逻辑。

示例：

go

```go
func producer(ch chan<- int) {
    for i := 0; i < 5; i++ {
        ch <- i
        fmt.Println("生产：", i)
        time.Sleep(300 * time.Millisecond)
    }
    close(ch) // 生产完毕，关闭通道
}

func consumer(ch <-chan int) {
    for val := range ch {
        fmt.Println("消费：", val)
        time.Sleep(500 * time.Millisecond)
    }
}

func main() {
    ch := make(chan int, 3) // 带缓冲，平衡生产和消费速度
    go producer(ch)
    consumer(ch)
}
```

### 2. 工作池（Worker Pool）模式

预先创建固定数量的 worker 协程，从 “任务通道” 中获取任务执行，避免频繁创建协程的开销。

示例：

go

```go
func worker(id int, tasks <-chan int, results chan<- int) {
    for task := range tasks {
        fmt.Printf("Worker %d 处理任务：%d\n", id, task)
        results <- task * task // 模拟任务结果
        time.Sleep(500 * time.Millisecond)
    }
}

func main() {
    tasks := make(chan int, 10)   // 任务通道
    results := make(chan int, 10) // 结果通道

    // 创建 3 个 worker
    for i := 1; i <= 3; i++ {
        go worker(i, tasks, results)
    }

    // 提交 9 个任务
    for i := 0; i < 9; i++ {
        tasks <- i
    }
    close(tasks) // 任务提交完毕

    // 收集结果
    for i := 0; i < 9; i++ {
        fmt.Println("结果：", <-results)
    }
}
```

## 五、注意事项与常见陷阱

### 1. 避免永久阻塞

- 无缓冲通道收发未配对：发送后无接收方，或接收前无发送方，导致协程永久阻塞（甚至死锁）。
- 有缓冲通道满 / 空后未及时处理：发送方在缓冲区满时阻塞，或接收方在缓冲区空时阻塞。

解决：

- 用 `select + timeout` 避免永久等待；
- 合理设置有缓冲通道的容量，匹配生产和消费速度。

### 2. 通道关闭的最佳实践

- 由**发送方**关闭通道（发送方最清楚 “何时无数据可发”）；
- 接收方通过 `for range` 或 `val, ok := <-ch` 判断通道是否关闭，不要主动关闭不属于自己的通道；
- 关闭已关闭的通道会触发 `panic`，需确保关闭操作仅执行一次。

### 3. 资源泄漏

若通道未关闭且无收发操作，依赖该通道的协程可能永久阻塞，导致**goroutine 泄漏**。

解决：

- 用 `context` 传递 “取消信号”，让协程能主动退出；
- 确保通道最终会被关闭，或收发操作有明确的 “结束条件”。
