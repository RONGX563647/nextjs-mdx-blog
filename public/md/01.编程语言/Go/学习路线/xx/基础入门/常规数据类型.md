Go 中的 “常规数据结构” 主要分为 **内置核心数据结构**（语言层面原生支持，高频使用）和 **标准库常用数据结构**（`container` 等包提供，满足特定场景需求）。这些结构是开发者日常开发的基础，需从 “底层原理、核心操作、使用场景、注意事项” 四个维度系统理解。

# 一、Go 内置核心数据结构（高频使用）

内置数据结构是 Go 语言原生支持的，无需依赖第三方库，包括 **数组、切片、映射、结构体、通道**，覆盖 90% 以上的常规开发场景。

## 1. 数组（Array）：固定长度的连续内存容器

### （1）定义与本质

数组是 **长度固定、元素类型相同的连续内存块**，长度是数组类型的一部分（如 `[3]int` 和 `[5]int` 是完全不同的类型）。

- 本质：底层是操作系统的连续内存，元素按索引顺序存储，支持 **随机访问**（通过索引直接定位，时间复杂度 O (1)）。

### （2）底层原理

- 内存布局：数组在内存中是连续的，例如 `[3]int{1,2,3}` 在 64 位系统中占 24 字节（每个 `int` 8 字节，3×8=24），地址依次递增。
- 零值初始化：未显式初始化的数组，元素会被赋予对应类型的零值（如 `[2]string` 零值为 `["", ""]`）。

### （3）核心操作（初始化、访问、遍历）

#### ① 初始化方式

| 方式            | 语法示例                         | 说明                                                 |
| --------------- | -------------------------------- | ---------------------------------------------------- |
| 显式长度 + 元素 | `var arr [3]int = [3]int{1,2,3}` | 明确长度和元素，类型可省略（`arr := [3]int{1,2,3}`） |
| 自动推导长度    | `arr := [...]int{4,5,6}`         | 用 `...` 让编译器根据元素数量推导长度                |
| 指定索引初始化  | `arr := [5]int{1:2, 3:4}`        | 索引 1 赋值 2，索引 3 赋值 4，其余零值               |

#### ② 访问与修改

通过索引访问（0 开始），支持直接修改元素值（数组是值类型，修改的是当前数组副本的元素）：

go











```go
arr := [3]int{1,2,3}
fmt.Println(arr[1]) // 访问索引1，输出 2
arr[1] = 10         // 修改索引1的值
fmt.Println(arr)    // 输出 [1 10 3]
```

#### ③ 遍历方式

- 普通 `for` 循环：通过索引遍历，适合需要索引的场景；
- `for-range` 循环：直接获取索引和值，代码更简洁。

go











```go
arr := [3]string{"a", "b", "c"}
// 普通for循环
for i := 0; i < len(arr); i++ {
    fmt.Printf("arr[%d] = %s\n", i, arr[i])
}
// for-range循环
for idx, val := range arr {
    fmt.Printf("idx: %d, val: %s\n", idx, val)
}
```

### （4）使用场景与注意事项

#### 适用场景

- 存储固定数量的同类型数据（如存储 RGB 三色值 `[3]int{255,0,0}`）；
- 对性能要求极高的场景（连续内存访问，缓存命中率高）。

#### 注意事项

- **长度固定**：数组长度一旦定义无法修改，如需动态扩容，需手动转换为切片；
- **值类型特性**：数组赋值或传参时会复制整个数组（若数组过大，性能开销高），如需避免复制，可传递数组指针（`*[3]int`）；
- **与切片的区别**：数组是 “值类型 + 固定长度”，切片是 “引用类型 + 动态长度”，日常开发中切片使用远多于数组。

## 2. 切片（Slice）：动态长度的数组引用（Go 最常用）

### （1）定义与本质

切片是 **基于数组的动态视图**，本身不存储数据，而是通过 “指针（指向底层数组）、长度（len，当前元素数）、容量（cap，底层数组最大可容纳元素数）” 三个字段间接操作数组。

- 本质：引用类型，底层依赖数组实现，支持动态扩容，兼顾灵活性和性能。

### （2）底层原理

切片的底层结构（`runtime.slice`）：

go











```go
type slice struct {
    array unsafe.Pointer // 指向底层数组的指针
    len   int            // 当前元素个数
    cap   int            // 底层数组的容量（最大可容纳元素数）
}
```

- 扩容机制：当

   

  ```
  len == cap
  ```

   

  时，调用

   

  ```
  append
  ```

   

  会触发扩容，扩容策略：

  1. 若 `cap < 1024`：新容量 = 原容量 × 2；
  2. 若 `cap ≥ 1024`：新容量 = 原容量 × 1.25（避免容量增长过快）；
  3. 扩容后会创建新的底层数组，将原数组数据复制到新数组，切片指针指向新数组（原数组若无人引用，会被 GC 回收）。

### （3）核心操作（初始化、增删改查、扩容）

#### ① 初始化方式

| 方式                | 语法示例                                  | 说明                                                         |
| ------------------- | ----------------------------------------- | ------------------------------------------------------------ |
| 字面量初始化        | `s := []int{1,2,3}`                       | 直接指定元素，len=cap=3                                      |
| `make` 初始化       | `s := make([]int, 3, 5)`                  | `make(类型, 长度, 容量)`，len=3，cap=5                       |
| 数组 / 切片切片操作 | `arr := [5]int{1,2,3,4,5}; s := arr[1:3]` | 从数组 / 切片截取，s 的 len=2，cap=4（原数组从索引 1 到末尾的长度） |

#### ② 核心操作

- **追加元素（`append`）**：最常用操作，动态添加元素，触发扩容时自动处理底层数组；
- **切片截取（`s[low:high:max]`）**：控制新切片的容量（`max` 限制新切片的 cap 为 `max - low`，避免修改原切片底层数组）；
- **复制（`copy`）**：将一个切片的元素复制到另一个切片，返回实际复制的元素数（需提前初始化目标切片长度）。

go











```go
// 1. append 追加元素
s := []int{1,2,3}
s = append(s, 4)          // 追加单个元素，s变为 [1,2,3,4]，len=4，cap=6（原cap=3，<1024，翻倍为6）
s = append(s, 5,6)        // 追加多个元素，s变为 [1,2,3,4,5,6]
s = append(s, []int{7,8}...) // 追加切片，需用 ... 展开

// 2. 切片截取（low:high:max）
s1 := s[1:3:4]            // low=1, high=3, max=4 → s1 = [2,3]，len=2，cap=3（4-1=3）
s1[0] = 20                // 修改s1，会影响原切片s（共享底层数组）
fmt.Println(s)            // 输出 [1,20,3,4,5,6,7,8]

// 3. copy 复制（避免共享底层数组）
target := make([]int, len(s1)) // 目标切片需提前初始化长度
copyCount := copy(target, s1)  // 复制s1到target，返回复制的元素数（2）
target[0] = 100               // 修改target，不影响s1和s
fmt.Println(s1, target)       // 输出 [20,3] [100,3]
```

#### ③ 遍历

与数组一致，支持普通 `for` 和 `for-range` 循环（因切片是引用类型，遍历效率与数组相当）。

### （4）使用场景与注意事项

#### 适用场景

- 存储动态数量的同类型数据（如接口返回的列表、用户输入的多条数据）；
- 作为函数参数（引用类型，传参时仅复制切片的三个字段，性能高）；
- 实现栈、队列等数据结构（基于切片的尾插 / 尾删操作）。

#### 注意事项

- **共享底层数组风险**：切片截取会共享原数组，修改新切片可能影响原切片，如需独立切片，需用 `copy` 或 `append` 创建新切片；
- **避免空切片与 nil 切片混淆**：`var s []int`（nil 切片，len=0, cap=0, array=nil）和 `s := []int{}`（空切片，len=0, cap=0, array = 非 nil），两者在 `if s == nil` 判断时结果不同，但功能上均可直接 `append`；
- **不要修改 `len` 和 `cap`**：切片的 `len` 和 `cap` 是只读的，需通过 `append`、切片截取等操作间接修改。

## 3. 映射（Map）：键值对哈希表（Key-Value 存储）

### （1）定义与本质

映射是 **基于哈希表（Hash Table）的键值对存储结构**，支持通过键（Key）快速查找值（Value），时间复杂度平均 O (1)。

- 本质：哈希表实现，通过 “哈希函数将 Key 映射到哈希桶，解决哈希冲突用链地址法（桶内元素用链表存储）”。

### （2）底层原理

Go 映射的底层结构（`runtime.hmap`）：

go











```go
type hmap struct {
    count     int            // 键值对数量
    flags     uint8          // 状态标志（如是否正在扩容）
    B         uint8          // 哈希桶数量 = 2^B
    noverflow uint16         // 溢出桶数量
    hash0     uint32         // 哈希种子（随机值，避免哈希碰撞攻击）
    buckets    unsafe.Pointer // 哈希桶数组指针
    oldbuckets unsafe.Pointer // 扩容时的旧桶数组
    // ... 其他字段
}
```

- 哈希桶（`bmap`）：每个桶存储 8 个键值对，若桶满则用 “溢出桶” 扩展（链表结构）；
- 扩容机制：当负载因子（`count / (2^B)`）超过阈值（默认 6.5），会触发扩容（新桶数量为原桶的 2 倍），扩容过程是 “渐进式” 的（避免一次性复制大量数据导致 STW 过长）。

### （3）核心操作（初始化、增删改查、遍历）

#### ① 初始化方式

- 字面量初始化：直接指定键值对，适合已知初始数据的场景；
- `make` 初始化：指定类型和初始容量（非必须，仅优化性能，避免频繁扩容）；
- 注意：**不能直接声明 `var m map[int]string` 后直接赋值**（此时 m 是 nil，需用 `make` 初始化后才能赋值）。

go











```go
// 1. 字面量初始化
m1 := map[string]int{"Alice": 25, "Bob": 30}

// 2. make 初始化（初始容量10，后续可自动扩容）
m2 := make(map[int]string, 10)
m2[1] = "a" // 赋值
m2[2] = "b"

// 错误示例：nil map 不能赋值
var m3 map[string]bool
// m3["test"] = true // 编译通过，但运行时 panic: assignment to entry in nil map
```

#### ② 核心操作

- **新增 / 修改**：`m[key] = value`（若 key 已存在则修改值，不存在则新增）；
- **查询**：`value, ok := m[key]`（用 `ok` 判断 key 是否存在，避免 “零值混淆”，如 `m["Bob"]` 返回 0 可能是值为 0 或 key 不存在）；
- **删除**：`delete(m, key)`（若 key 不存在，`delete` 无操作，不会报错）；
- **获取长度**：`len(m)`（返回键值对数量，时间复杂度 O (1)）。

go











```go
m := map[string]int{"Alice": 25}

// 新增
m["Charlie"] = 35
fmt.Println(m) // 输出 map[Alice:25 Charlie:35]

// 修改
m["Alice"] = 26
fmt.Println(m["Alice"]) // 输出 26

// 查询（判断key是否存在）
val, ok := m["Bob"]
if ok {
    fmt.Println("Bob's age:", val)
} else {
    fmt.Println("Bob not found") // 输出 Bob not found
}

// 删除
delete(m, "Charlie")
fmt.Println(m) // 输出 map[Alice:26]

// 长度
fmt.Println(len(m)) // 输出 1
```

#### ③ 遍历

通过 `for-range` 循环，支持遍历键、值或键值对，**遍历顺序不固定**（哈希表的特性，每次遍历可能不同）：

go











```go
m := map[string]int{"a":1, "b":2, "c":3}

// 遍历键值对
for k, v := range m {
    fmt.Printf("k: %s, v: %d\n", k, v)
}

// 仅遍历键
for k := range m {
    fmt.Println("k:", k)
}

// 仅遍历值（键用 _ 忽略）
for _, v := range m {
    fmt.Println("v:", v)
}
```

### （4）使用场景与注意事项

#### 适用场景

- 存储键值对数据（如用户 ID 到用户信息的映射、配置项的键值配置）；
- 快速查找 / 删除场景（如缓存、字典查询）。

#### 注意事项

- **键的要求**：Key 必须是 “可比较类型”（如 `int`、`string`、`struct`、`pointer`），不可为 “不可比较类型”（`slice`、`map`、`function`），否则编译报错；
- **并发安全**：默认 `map` 是非并发安全的，多个 Goroutine 同时读写会触发运行时错误（`fatal error: concurrent map writes`），并发场景需用 `sync.Map`（标准库）或加锁（`sync.Mutex`）；
- **不可取地址**：不能对 `m[key]` 取地址（`&m["a"]` 编译报错），因 `m[key]` 可能是临时值，地址不固定；
- **零值映射**：`var m map[int]string` 是 nil 映射，可查询（返回零值）和删除（无操作），但不能赋值，需 `make` 初始化后才能赋值。

## 4. 结构体（Struct）：自定义复合类型（模拟 “类”）

### （1）定义与本质

结构体是 **将多个不同类型的字段（Field）组合成的自定义复合类型**，无继承，通过 “组合” 实现代码复用，是 Go 实现面向对象（OOP）思想的核心。

- 本质：值类型，字段按内存对齐规则存储（优化 CPU 访问效率），支持为结构体绑定方法（Method），模拟 “类的成员函数”。

### （2）底层原理

- 内存布局：结构体的字段在内存中连续存储，但会按 “内存对齐” 规则填充空白字节（如 `struct{ a bool; b int }` 在 64 位系统中占 16 字节：`bool` 占 1 字节，填充 7 字节，`int` 占 8 字节）；
- 内存对齐规则：字段的偏移量必须是字段大小的整数倍，结构体总大小必须是最大字段大小的整数倍（减少 CPU 多周期读取内存的开销）。

### （3）核心操作（定义、初始化、字段访问、方法绑定）

#### ① 结构体定义

用 `type 结构体名 struct { 字段名 类型; ... }` 定义，字段首字母大写表示 “导出”（包外可见），小写表示 “私有”（仅包内可见）：

go











```go
// 定义结构体（Person）
type Person struct {
    Name string // 导出字段（首字母大写，包外可访问）
    Age  int    // 导出字段
    addr string // 私有字段（首字母小写，仅包内可访问）
}

// 嵌套结构体（组合实现复用，模拟“继承”）
type Student struct {
    Person  // 匿名嵌套（嵌入 Person 结构体，Student 可直接访问 Person 的导出字段）
    ID      int
    Major   string
}
```

#### ② 初始化方式

| 方式             | 语法示例                                                     | 说明                                         |
| ---------------- | ------------------------------------------------------------ | -------------------------------------------- |
| 字段名初始化     | `p := Person{Name: "Alice", Age: 25}`                        | 明确指定字段名，顺序可任意，未指定字段为零值 |
| 顺序初始化       | `p := Person{"Bob", 30, "Beijing"}`                          | 按结构体字段定义顺序初始化，必须包含所有字段 |
| 指针初始化       | `p := &Person{Name: "Charlie"}`                              | 返回结构体指针，避免值拷贝（推荐）           |
| 嵌套结构体初始化 | `s := Student{Person: Person{Name: "Dave"}, ID: 1001, Major: "CS"}` | 匿名嵌套需指定外层字段名（Person）           |

#### ③ 字段访问

通过 `结构体变量.字段名` 访问，匿名嵌套的结构体字段可直接访问（若字段名冲突，需通过嵌套结构体名访问）：

go











```go
// 初始化 Student
s := Student{
    Person: Person{Name: "Dave", Age: 20},
    ID:     1001,
    Major:  "Computer Science",
}

// 访问字段：直接访问匿名嵌套的 Person 字段
fmt.Println(s.Name, s.Age) // 输出 Dave 20（等价于 s.Person.Name）

// 访问嵌套结构体的私有字段（仅包内可访问）
s.Person.addr = "Shanghai" // 若在同一包内，可访问 addr

// 指针访问（结构体指针用 . 访问字段，无需 ->，Go 语法糖）
sp := &s
fmt.Println(sp.ID, sp.Major) // 输出 1001 Computer Science
```

#### ④ 方法绑定

为结构体绑定方法（类似类的成员函数），通过 “接收者”（Receiver）指定方法属于哪个结构体：

- **值接收者**：方法内修改的是结构体副本，不影响原结构体；
- **指针接收者**：方法内修改的是原结构体（推荐用于修改结构体或结构体较大时避免拷贝）。

go











```go
// 为 Person 绑定值接收者方法（不修改原结构体）
func (p Person) GetName() string {
    return p.Name
}

// 为 Person 绑定指针接收者方法（修改原结构体）
func (p *Person) SetAge(newAge int) {
    p.Age = newAge // 指针接收者，修改原结构体的 Age
}

func main() {
    p := Person{Name: "Alice", Age: 25}
    fmt.Println(p.GetName()) // 输出 Alice（值接收者调用）

    p.SetAge(26)             // 指针接收者调用，Go 自动转换为 (&p).SetAge(26)
    fmt.Println(p.Age)       // 输出 26（原结构体被修改）
}
```

### （4）使用场景与注意事项

#### 适用场景

- 封装复杂数据（如用户信息、订单信息、配置项）；
- 实现面向对象思想（通过结构体 + 方法模拟类，通过嵌套实现组合复用）；
- 定义接口实现（结构体实现接口的所有方法，自动成为接口类型）。

#### 注意事项

- **值类型特性**：结构体赋值或传参时会复制整个结构体（若结构体较大，性能开销高），推荐传递结构体指针；
- **字段导出规则**：仅首字母大写的字段可被其他包访问，私有字段需通过包内方法暴露（如 `func (p *Person) GetAddr() string { return p.addr }`）；
- **内存对齐优化**：结构体字段顺序影响总大小，建议将小字段放在一起（如 `struct{ a bool; b bool; c int }` 比 `struct{ a bool; c int; b bool }` 小 8 字节，64 位系统）。

## 5. 通道（Channel）：并发安全的通信管道（Goroutine 通信）

### （1）定义与本质

通道是 **Goroutine 间同步与通信的管道**，遵循 “通过通信共享内存，而非通过共享内存通信”（CSP 理论），是 Go 并发模型的核心。

- 本质：同步原语，底层是 “环形缓冲区（有缓冲）” 或 “直接同步队列（无缓冲）”，支持并发安全的发送（`ch <- val`）和接收（`val := <-ch`）操作。

### （2）底层原理

通道的底层结构（`runtime.hchan`）：

go











```go
type hchan struct {
    qcount   uint           // 缓冲区元素数量
    dataqsiz uint           // 缓冲区容量（有缓冲通道）
    buf      unsafe.Pointer // 缓冲区指针（环形数组）
    sendx    uint           // 发送索引（下一个发送位置）
    recvx    uint           // 接收索引（下一个接收位置）
    recvq    waitq          // 阻塞的接收 Goroutine 队列
    sendq    waitq          // 阻塞的发送 Goroutine 队列
    lock     mutex          // 互斥锁（保证并发安全）
    // ... 其他字段
}
```

- 无缓冲通道：`dataqsiz=0`，无缓冲区，发送方与接收方必须同时就绪（同步通信），否则阻塞；
- 有缓冲通道：`dataqsiz>0`，缓冲区未满可发送，缓冲区未空可接收，发送 / 接收方仅在缓冲区满 / 空时阻塞。

### （3）核心操作（初始化、发送、接收、关闭）

#### ① 初始化方式

必须用 `make(chan 类型, 缓冲大小)` 初始化，缓冲大小为 0 时是无缓冲通道，>0 时是有缓冲通道：

go











```go
// 无缓冲通道（同步通信）
ch1 := make(chan int)

// 有缓冲通道（异步通信，容量为 3）
ch2 := make(chan string, 3)
```

#### ② 核心操作

- **发送**：`ch <- val`（将 val 发送到通道，类型需与通道一致）；
- **接收**：`val := <-ch`（从通道接收值，若通道关闭且无数据，接收零值）；`val, ok := <-ch`（用 `ok` 判断通道是否关闭，`ok=false` 表示通道关闭且无数据）；
- **关闭**：`close(ch)`（关闭通道，仅发送方可关闭，关闭后不能再发送，否则 panic；接收方仍可接收剩余数据）；
- **获取长度 / 容量**：`len(ch)`（缓冲区元素数量），`cap(ch)`（缓冲区容量，无缓冲通道 cap=0）。

go











```go
// 有缓冲通道示例
ch := make(chan int, 2)

// 发送数据（缓冲区未满，不阻塞）
ch <- 1
ch <- 2
fmt.Println(len(ch), cap(ch)) // 输出 2 2

// 接收数据（缓冲区未空，不阻塞）
val1 := <-ch
fmt.Println(val1) // 输出 1

// 关闭通道
close(ch)

// 接收剩余数据
val2, ok := <-ch
fmt.Println(val2, ok) // 输出 2 true

// 通道关闭后接收（ok=false，返回零值）
val3, ok := <-ch
fmt.Println(val3, ok) // 输出 0 false

// 通道关闭后发送（panic）
// ch <- 3 // 运行时 panic: send on closed channel
```

#### ③ 并发通信示例

通过通道实现 Goroutine 间同步通信，避免共享内存与锁：

go











```go
// 子 Goroutine 发送数据
func sendData(ch chan<- int) { // chan<- 表示仅发送通道
    for i := 1; i <= 3; i++ {
        ch <- i // 发送数据
        time.Sleep(100 * time.Millisecond)
    }
    close(ch) // 发送完毕，关闭通道
}

// 主 Goroutine 接收数据
func main() {
    ch := make(chan int, 1)
    go sendData(ch)

    // 接收通道数据（直到通道关闭）
    for val := range ch { // for-range 遍历通道，通道关闭后自动退出循环
        fmt.Println("Received:", val)
    }
    fmt.Println("All data received")
}
// 输出：
// Received: 1
// Received: 2
// Received: 3
// All data received
```

### （4）使用场景与注意事项

#### 适用场景

- Goroutine 间同步通信（如子 Goroutine 处理结果传递给主 Goroutine）；
- 并发控制（如用无缓冲通道实现 “等待所有 Goroutine 完成”）；
- 数据传递（如流水线作业，多个 Goroutine 通过通道传递数据）。

#### 注意事项

- **发送 / 接收阻塞**：无缓冲通道发送方阻塞直到有接收方，接收方阻塞直到有发送方；有缓冲通道发送方阻塞直到缓冲区有空间，接收方阻塞直到缓冲区有数据；
- **关闭通道规则**：仅发送方可关闭通道，接收方关闭会 panic；关闭已关闭的通道会 panic；
- **通道类型**：支持 “仅发送通道（`chan<- T`）” 和 “仅接收通道（`<-chan T`）”，用于函数参数时限制通道操作，提升类型安全；
- **避免通道泄漏**：若 Goroutine 阻塞在发送 / 接收通道，且通道无其他 Goroutine 操作，会导致 Goroutine 泄漏（需确保通道有明确的关闭逻辑或超时机制）。

# 二、Go 标准库常用数据结构（`container` 包等）

Go 标准库的 `container` 包提供了三种常用数据结构：**双向链表（list）、堆（heap）、环形链表（ring）**，满足特定场景需求（如频繁插入删除、优先级队列）。

## 1. 双向链表（`container/list`）

### （1）定义与本质

`container/list` 实现了 **双向链表（Doubly Linked List）**，每个节点（`*list.Element`）包含前驱、后继指针和数据，支持在链表两端或任意节点插入 / 删除元素（时间复杂度 O (1)）。

- 本质：双向链表，内存非连续，适合频繁在中间插入 / 删除的场景（数组 / 切片在中间插入删除需移动元素，时间复杂度 O (n)）。

### （2）核心操作（初始化、增删改查、遍历）

#### ① 初始化

用 `list.New()` 创建新链表，返回 `*list.List` 指针：

go











```go
import "container/list"

// 初始化链表
l := list.New()
```

#### ② 核心操作

- **插入元素**：`PushFront(v)`（头部插入）、`PushBack(v)`（尾部插入）、`InsertBefore(v, e *list.Element)`（指定节点前插入）、`InsertAfter(v, e *list.Element)`（指定节点后插入）；
- **删除元素**：`Remove(e *list.Element)`（删除指定节点，返回节点数据）；
- **获取节点**：`Front()`（获取头部节点）、`Back()`（获取尾部节点）；
- **遍历**：通过 `for e := l.Front(); e != nil; e = e.Next()` 遍历链表节点，`e.Value` 获取节点数据。

go











```go
l := list.New()

// 插入元素
l.PushBack(1)        // 尾部插入 1 → [1]
l.PushFront(0)       // 头部插入 0 → [0,1]
e := l.PushBack(2)   // 尾部插入 2，返回节点 e → [0,1,2]
l.InsertBefore(1.5, e) // 在 e（值 2）前插入 1.5 → [0,1,1.5,2]

// 删除元素
l.Remove(l.Front())  // 删除头部节点（0）→ [1,1.5,2]

// 遍历链表
for e := l.Front(); e != nil; e = e.Next() {
    fmt.Println(e.Value) // 输出 1, 1.5, 2
}

// 获取长度
fmt.Println(l.Len()) // 输出 3
```

### （3）使用场景与注意事项

#### 适用场景

- 频繁在链表中间插入 / 删除元素（如日志队列、任务队列，需动态调整顺序）；
- 存储不确定长度的数据，且对随机访问要求低（链表随机访问时间复杂度 O (n)）。

#### 注意事项

- **节点有效性**：删除节点后，该节点指针（`*list.Element`）失效，不可再使用；
- **类型断言**：节点数据 `e.Value` 是 `interface{}` 类型，需通过类型断言转换为具体类型（如 `val, ok := e.Value.(int)`）；
- **性能对比**：链表的插入 / 删除效率高于切片（O (1) vs O (n)），但随机访问效率低于切片（O (n) vs O (1)），需根据场景选择。

## 2. 堆（`container/heap`）

### （1）定义与本质

`container/heap` 实现了 **堆（Heap）** 数据结构，默认是 “最小堆”（堆顶元素是最小值），需用户自定义类型实现 `heap.Interface` 接口才能使用。

- 本质：基于数组（切片）实现的完全二叉树，支持 “插入元素” 和 “弹出堆顶元素” 操作，时间复杂度均为 O (log n)，适合实现优先级队列。

### （2）核心原理：`heap.Interface` 接口

使用 `heap` 包需自定义类型（通常是切片）实现以下 5 个方法（`heap.Interface` 继承 `sort.Interface`，需实现 `Len()`、`Less()`、`Swap()`，再额外实现 `Push()`、`Pop()`）：

go











```go
type Interface interface {
    sort.Interface
    Push(x interface{}) // 向堆中添加元素（append 到切片）
    Pop() interface{}   // 从堆中移除并返回最后一个元素（切片最后一个元素）
}
```

- ```
  sort.Interface
  ```

   

  方法：

  - `Len() int`：返回堆的长度；
  - `Less(i, j int) bool`：判断堆中索引 i 的元素是否小于索引 j 的元素（最小堆返回 `h[i] < h[j]`，最大堆返回 `h[i] > h[j]`）；
  - `Swap(i, j int)`：交换堆中索引 i 和 j 的元素。

### （3）核心操作（自定义堆、插入、弹出、堆化）

#### ① 实现最小堆示例（优先级队列）

go











```go
import (
    "container/heap"
    "fmt"
)

// 自定义最小堆类型（基于 int 切片）
type MinHeap []int

// 实现 sort.Interface 方法
func (h MinHeap) Len() int           { return len(h) }
func (h MinHeap) Less(i, j int) bool { return h[i] < h[j] } // 最小堆：h[i] < h[j]
func (h MinHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

// 实现 heap.Interface 的 Push 方法（append 到切片）
func (h *MinHeap) Push(x interface{}) {
    *h = append(*h, x.(int)) // 类型断言为 int
}

// 实现 heap.Interface 的 Pop 方法（返回切片最后一个元素）
func (h *MinHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1] // 截断切片，移除最后一个元素
    return x
}

// 使用最小堆
func main() {
    // 初始化堆
    h := &MinHeap{3, 1, 2}
    heap.Init(h) // 堆化（将切片转换为堆结构）
    fmt.Println("Init heap:", *h) // 输出 [1 3 2]（堆化后，堆顶是最小值 1）

    // 插入元素
    heap.Push(h, 0)
    fmt.Println("After push 0:", *h) // 输出 [0 1 2 3]

    // 弹出堆顶元素（最小值）
    for h.Len() > 0 {
        x := heap.Pop(h).(int)
        fmt.Printf("Pop: %d, heap: %v\n", x, *h)
    }
    // 输出：
    // Pop: 0, heap: [1 3 2]
    // Pop: 1, heap: [2 3]
    // Pop: 2, heap: [3]
    // Pop: 3, heap: []
}
```

#### ② 实现最大堆

只需修改 `Less()` 方法，返回 `h[i] > h[j]`：

go











```go
func (h MaxHeap) Less(i, j int) bool { return h[i] > h[j] } // 最大堆：h[i] > h[j]
```

### （4）使用场景与注意事项

#### 适用场景

- 优先级队列（如任务调度，优先级高的任务先执行）；
- Top K 问题（如获取前 10 个最大元素，用最小堆存储，效率高于排序）；
- 堆排序（通过插入所有元素后弹出，实现排序，时间复杂度 O (n log n)）。

#### 注意事项

- **必须初始化**：自定义堆后，需调用 `heap.Init(h)` 进行堆化，否则插入 / 弹出操作会出错；
- **类型断言**：`Push()` 和 `Pop()` 的参数 / 返回值是 `interface{}` 类型，需通过类型断言转换为具体类型；
- **堆结构维护**：`heap.Push()` 和 `heap.Pop()` 会自动维护堆结构（调整元素位置），不可直接修改切片后使用。

## 3. 环形链表（`container/ring`）

### （1）定义与本质

`container/ring` 实现了 **环形链表（Circular List）**，与双向链表不同，环形链表无首尾节点（最后一个节点的后继是第一个节点），适合循环访问场景。

- 本质：固定长度的环形结构，初始化时指定长度，不可动态增删（长度固定），支持循环遍历、旋转、移动等操作。

### （2）核心操作（初始化、遍历、旋转、移动）

#### ① 初始化

用 `ring.New(n int)` 创建长度为 n 的环形链表，返回 `*ring.Ring` 指针（指向环形链表的任意节点，因环形无首尾）：

go











```go
import "container/ring"

// 初始化长度为 3 的环形链表
r := ring.New(3)
```

#### ② 核心操作

- **设置 / 获取节点数据**：`r.Value = v`（设置当前节点数据），`r.Value`（获取当前节点数据）；
- **遍历**：`r.Do(func(x interface{}) { ... })`（遍历环形链表的所有节点，执行函数）；
- **移动节点**：`r.Move(n int)`（将当前节点移动 n 步，n 为正向前移，负向后移）；
- **旋转**：`r.Rotate(n int)`（将环形链表旋转 n 步，相当于移动当前节点的位置）；
- **链接环形链表**：`r.Link(s *ring.Ring)`（将环形链表 s 链接到 r 之后，形成新的环形链表，长度为 r.Len () + s.Len ()）。

go











```go
// 初始化长度为 3 的环形链表
r := ring.New(3)

// 设置节点数据
for i := 0; i < r.Len(); i++ {
    r.Value = i + 1
    r = r.Next() // 移动到下一个节点
}

// 遍历环形链表
fmt.Print("Ring elements: ")
r.Do(func(x interface{}) {
    fmt.Printf("%d ", x) // 输出 1 2 3
})
fmt.Println()

// 移动节点（当前节点从 1 移动到 3，n=2 步）
r = r.Move(2)
fmt.Println("Current element after move 2:", r.Value) // 输出 3

// 旋转环形链表（旋转 1 步，相当于将当前节点的前一个节点移到末尾）
r.Rotate(1)
fmt.Print("Ring after rotate 1: ")
r.Do(func(x interface{}) {
    fmt.Printf("%d ", x) // 输出 3 1 2
})
```

### （3）使用场景与注意事项

#### 适用场景

- 循环访问场景（如定时任务轮询、环形缓冲区、约瑟夫环问题）；
- 固定长度的循环数据存储（如最近 5 条日志的循环展示）。

#### 注意事项

- **长度固定**：环形链表初始化后长度不可修改，如需动态增删，需使用双向链表（`container/list`）；
- **当前节点**：`*ring.Ring` 指针始终指向 “当前节点”，所有操作（`Next()`、`Move()`）都是基于当前节点；
- **遍历终止**：环形链表无首尾，`r.Do()` 会遍历所有节点后自动终止，手动遍历需通过 `r.Next()` 并计数，避免无限循环。

# 三、Go 常规数据结构对比与选择

| 数据结构         | 底层实现           | 核心特性               | 时间复杂度（增删查）         | 适用场景                                    |
| ---------------- | ------------------ | ---------------------- | ---------------------------- | ------------------------------------------- |
| 数组（Array）    | 连续内存           | 固定长度，值类型       | 查 O (1)，增删 O (n)         | 固定数量同类型数据，高性能随机访问          |
| 切片（Slice）    | 数组引用           | 动态长度，引用类型     | 尾增删 O (1)，中间增删 O (n) | 动态数量同类型数据，日常开发首选            |
| 映射（Map）      | 哈希表             | 键值对，无序           | 增删查平均 O (1)             | 键值对存储，快速查找 / 删除（如缓存、字典） |
| 结构体（Struct） | 连续内存（对齐）   | 自定义复合类型，值类型 | 字段访问 O (1)               | 封装复杂数据（如用户信息、订单）            |
| 通道（Channel）  | 环形缓冲区 / 队列  | 并发安全通信           | 发送接收 O (1)               | Goroutine 间同步通信、并发控制              |
| 双向链表（list） | 双向链表节点       | 动态增删，非连续内存   | 增删 O (1)，查 O (n)         | 频繁中间增删，如日志队列、任务队列          |
| 堆（heap）       | 切片（完全二叉树） | 优先级，最小 / 最大堆  | 插入弹出 O (log n)           | 优先级队列、Top K 问题、堆排序              |
| 环形链表（ring） | 环形节点           | 固定长度，循环访问     | 移动 O (1)，查 O (n)         | 循环访问场景，如定时任务轮询、环形缓冲区    |

**选择原则**：

1. 优先使用内置数据结构（切片、映射、结构体），避免过度依赖标准库（如能用切片实现的栈 / 队列，无需用 `container/list`）；
2. 高频随机访问选切片 / 数组，高频中间增删选链表，键值对存储选映射，并发通信选通道；
3. 性能敏感场景：优先考虑切片（尾操作）、数组（固定长度），避免频繁使用链表（缓存命中率低）。

掌握以上数据结构，即可覆盖 Go 日常开发的绝大多数场景，后续可结合具体业务需求（如分布式场景、高并发场景）深入学习进阶数据结构（如跳表、布隆过滤器）。
