## 滑动窗口

~~~markdown
滑动窗口（Sliding Window）是一种在数组或字符串上高效解决子串或子数组问题的算法技巧。它通过维护一个动态的窗口（子数组或子串），在遍历数据的过程中不断调整窗口的大小和位置，从而在一次遍历中完成对问题的求解，时间复杂度通常为O(n)。

### 基本思路
1. **窗口初始化**：确定窗口的左右边界，通常初始化为0。
2. **窗口滑动**：右边界向右扩展，收集元素；当窗口内元素不满足条件时，左边界向右收缩，移除元素。
3. **状态维护**：在窗口滑动过程中，维护与问题相关的状态（如元素计数、和、最大值等）。

### 典型应用场景
- 求满足条件的最长/最短子串
- 子数组的和/乘积问题
- 字符串匹配问题

**思路**：使用哈希表记录字符出现的位置，当右指针遇到重复字符时，左指针直接跳到上次出现位置的下一个位置。

**思路**：滑动窗口内维护出现次数最多的字符，窗口大小减去最多字符数 ≤k 时合法，否则收缩左边界。

#### 5. [567. 字符串的排列](https://leetcode-cn.com/problems/permutation-in-string/)
**问题**：判断字符串s2是否包含s1的排列。  
**思路**：固定窗口大小为s1的长度，比较窗口内字符频率是否与s1相同。

#### 6. [438. 找到字符串中所有字母异位词](https://leetcode-cn.com/problems/find-all-anagrams-in-a-string/)
**问题**：找出字符串s中p的所有异位词的起始索引。  
**思路**：与上一题类似，使用固定窗口滑动，统计字符频率。

**问题**：给定数组，第i个元素是股票第i天的价格，求最大利润。

**问题**：从一排水果中收集最多两种类型的水果，每种类型可以收集多个，求最多能收集的水果数。

**思路**：固定窗口大小为k，维护字符频率哈希表，检查窗口内是否无重复。

### 滑动窗口模板
以下是解决滑动窗口问题的通用模板：

```java
public int slidingWindow(String s, String t) {
    // 初始化左右指针
    int left = 0, right = 0;
    // 初始化结果
    int result = 0;
    
    // 用于记录目标字符及其频率
    Map<Character, Integer> targetMap = new HashMap<>();
    for (char c : t.toCharArray()) {
        targetMap.put(c, targetMap.getOrDefault(c, 0) + 1);
    }
    
    // 用于记录当前窗口内字符及其频率
    Map<Character, Integer> windowMap = new HashMap<>();
    // 记录窗口内满足条件的字符个数
    int valid = 0;
    
    while (right < s.length()) {
        // 移入窗口的字符
        char c = s.charAt(right);
        // 右移窗口
        right++;
        
        // 更新窗口内数据
        if (targetMap.containsKey(c)) {
            windowMap.put(c, windowMap.getOrDefault(c, 0) + 1);
            if (windowMap.get(c).equals(targetMap.get(c))) {
                valid++;
            }
        }
        
        // 判断左侧窗口是否需要收缩
        while (window needs shrink) {
            // 移出窗口的字符
            char d = s.charAt(left);
            // 左移窗口
            left++;
            
            // 更新窗口内数据
            if (targetMap.containsKey(d)) {
                if (windowMap.get(d).equals(targetMap.get(d))) {
                    valid--;
                }
                windowMap.put(d, windowMap.get(d) - 1);
            }
        }
        
        // 更新结果
        result = Math.max(result, right - left);
    }
    
    return result;
}
```

1. 明确窗口内需要维护的状态
2. 合理调整窗口的扩展和收缩条件
3. 高效维护窗口状态的变化
~~~

~~~markdown
### 例题1：[3. 无重复字符的最长子串](https://leetcode-cn.com/problems/longest-substring-without-repeating-characters/)
**题目描述**：给定一个字符串 `s`，找出其中不含有重复字符的最长子串的长度。

**示例**：
- 输入: `s = "abcabcbb"`
- 输出: 3 
- 解释: 无重复字符的最长子串是 `"abc"`，长度为3。

**解题思路**：
1. **窗口维护**：使用双指针 `left` 和 `right` 维护一个滑动窗口，确保窗口内的字符不重复。
2. **字符频率统计**：使用哈希表记录每个字符在窗口中出现的次数。
3. **窗口滑动规则**：
   - 右指针 `right` 不断向右扩展，将字符加入窗口。
   - 若新加入的字符导致重复（即该字符在窗口中已存在），则收缩左指针 `left`，直到窗口内不再有重复字符。
4. **结果记录**：每次窗口合法时，更新最长子串的长度。

**Java代码实现**：
```java
public int lengthOfLongestSubstring(String s) {
    int left = 0, right = 0;
    int maxLen = 0;
    Map<Character, Integer> window = new HashMap<>();
    
    while (right < s.length()) {
        char c = s.charAt(right);
        right++;
        // 更新窗口内字符频率
        window.put(c, window.getOrDefault(c, 0) + 1);
        
        // 当窗口内出现重复字符时，收缩左指针
        while (window.get(c) > 1) {
            char d = s.charAt(left);
            left++;
            window.put(d, window.get(d) - 1);
        }
        
        // 记录当前合法窗口的最大长度
        maxLen = Math.max(maxLen, right - left);
    }
    
    return maxLen;
}
```

**步骤解析**：
- **初始化**：`left = 0`, `right = 0`，哈希表 `window` 为空，`maxLen = 0`。
- **扩展窗口**：右指针 `right` 遍历字符串，将字符加入窗口并更新频率。
- **收缩窗口**：当窗口内出现重复字符（如 `window.get(c) > 1`），收缩左指针并更新频率，直到重复字符被移除。
- **记录结果**：每次窗口合法时（无重复字符），计算窗口长度 `right - left` 并更新最大值。

**复杂度分析**：
- **时间复杂度**：O(n)，每个字符最多被访问两次（右指针扩展和左指针收缩）。
- **空间复杂度**：O(k)，k为字符集大小（如ASCII为256）。


### 例题2：[76. 最小覆盖子串](https://leetcode-cn.com/problems/minimum-window-substring/)
**题目描述**：给定两个字符串 `s` 和 `t`，在 `s` 中找到包含 `t` 所有字符的最小窗口。

**示例**：
- 输入: `s = "ADOBECODEBANC"`, `t = "ABC"`
- 输出: `"BANC"`
- 解释: 最小覆盖窗口是 `"BANC"`，包含了 `t` 中的 `A`, `B`, `C`。

**解题思路**：
1. **目标字符统计**：使用哈希表 `target` 统计 `t` 中每个字符的频率。
2. **窗口状态维护**：使用哈希表 `window` 统计当前窗口内字符的频率，并用 `valid` 记录窗口内满足条件的字符种类数。
3. **窗口滑动规则**：
   - 右指针扩展窗口，更新窗口状态。
   - 当窗口包含所有目标字符（`valid == target.size()`）时，尝试收缩左指针以寻找更小的窗口。
4. **结果记录**：记录最小窗口的起始位置和长度。

**Java代码实现**：
```java
public String minWindow(String s, String t) {
    Map<Character, Integer> target = new HashMap<>();
    for (char c : t.toCharArray()) {
        target.put(c, target.getOrDefault(c, 0) + 1);
    }
    
    int left = 0, right = 0;
    int valid = 0;
    int start = 0, minLen = Integer.MAX_VALUE;
    Map<Character, Integer> window = new HashMap<>();
    
    while (right < s.length()) {
        char c = s.charAt(right);
        right++;
        
        // 更新窗口内字符频率
        if (target.containsKey(c)) {
            window.put(c, window.getOrDefault(c, 0) + 1);
            if (window.get(c).equals(target.get(c))) {
                valid++;
            }
        }
        
        // 当窗口满足条件时，收缩左指针
        while (valid == target.size()) {
            // 更新最小窗口
            if (right - left < minLen) {
                start = left;
                minLen = right - left;
            }
            
            char d = s.charAt(left);
            left++;
            
            // 更新窗口状态
            if (target.containsKey(d)) {
                if (window.get(d).equals(target.get(d))) {
                    valid--;
                }
                window.put(d, window.get(d) - 1);
            }
        }
    }
    
    return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
}
```

**步骤解析**：
- **初始化目标哈希表**：统计 `t` 中字符频率。
- **扩展窗口**：右指针遍历 `s`，将字符加入窗口并更新频率。若该字符是目标字符且频率达标，`valid` 加1。
- **收缩窗口**：当 `valid == target.size()` 时，窗口包含所有目标字符，尝试收缩左指针并更新最小窗口。
- **结果处理**：返回最小窗口子串，若不存在则返回空字符串。

**复杂度分析**：
- **时间复杂度**：O(n)，n为 `s` 的长度。每个字符最多被访问两次。
- **空间复杂度**：O(k)，k为 `t` 中不同字符的个数。


### 例题3：[209. 长度最小的子数组](https://leetcode-cn.com/problems/minimum-size-subarray-sum/)
**题目描述**：给定一个含有 `n` 个正整数的数组和一个正整数 `target`，找出数组中满足其和 ≥ `target` 的长度最小的连续子数组。

**示例**：
- 输入: `target = 7`, `nums = [2,3,1,2,4,3]`
- 输出: 2
- 解释: 子数组 `[4,3]` 是该条件下的长度最小的子数组，长度为2。

**解题思路**：
1. **窗口维护**：使用双指针维护滑动窗口，窗口内元素的和需满足 ≥ `target`。
2. **窗口滑动规则**：
   - 右指针扩展窗口，累加元素和。
   - 当窗口和 ≥ `target` 时，收缩左指针并更新最小长度，直到窗口和 < `target`。
3. **结果记录**：维护最小子数组的长度。

**Java代码实现**：
```java
public int minSubArrayLen(int target, int[] nums) {
    int left = 0, right = 0;
    int sum = 0;
    int minLen = Integer.MAX_VALUE;
    
    while (right < nums.length) {
        // 扩展窗口
        sum += nums[right];
        right++;
        
        // 当窗口和满足条件时，收缩左指针
        while (sum >= target) {
            // 更新最小长度
            minLen = Math.min(minLen, right - left);
            // 收缩左指针
            sum -= nums[left];
            left++;
        }
    }
    
    return minLen == Integer.MAX_VALUE ? 0 : minLen;
}
```

**步骤解析**：
- **初始化**：`left = 0`, `right = 0`, `sum = 0`, `minLen = Integer.MAX_VALUE`。
- **扩展窗口**：右指针遍历数组，累加元素到 `sum`。
- **收缩窗口**：当 `sum ≥ target` 时，尝试收缩左指针并更新 `minLen`，同时减去移出窗口的元素。
- **结果处理**：若未找到符合条件的子数组，返回0；否则返回最小长度。

**复杂度分析**：
- **时间复杂度**：O(n)，每个元素最多被访问两次。
- **空间复杂度**：O(1)，仅需常数级额外空间。


### 滑动窗口总结
1. **适用场景**：求满足条件的最长/最短子串、子数组，或需要维护动态区间的问题。
2. **核心逻辑**：
   - 右指针扩展窗口，收集元素。
   - 左指针收缩窗口，排除不满足条件的元素。
   - 维护窗口状态（如元素频率、和、有效元素数等）。
3. **注意事项**：
   - 窗口收缩的条件需根据题目要求灵活调整。
   - 哈希表和计数器是常用的状态维护工具。
   - 处理边界条件（如空字符串、无解情况）。

通过多练习此类题目，可以熟练掌握滑动窗口的解题技巧。
~~~
