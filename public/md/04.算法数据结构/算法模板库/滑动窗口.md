滑动窗口算法是双指针技术的一种高级变体，专门用于解决**连续子数组 / 子串问题**。其核心思想是通过两个指针（`left` 和 `right`）维护一个 “窗口”（即区间 `[left, right]`），通过**动态调整窗口范围**（扩张或收缩），在线性时间内高效求解问题。

### 一、滑动窗口的核心原理

滑动窗口的本质是**用 “局部遍历” 替代 “全局遍历”**，避免对重复区间的冗余计算。具体来说：

- 窗口由 `left`（左边界）和 `right`（右边界）定义，初始时通常都指向序列起点（`left = right = 0`）。
- `right` 负责 “扩张窗口”：向右移动以包含更多元素，直到窗口满足特定条件（如包含目标字符、和达到阈值等）。
- `left` 负责 “收缩窗口”：当窗口不满足条件（或需要优化结果，如找最小长度）时，向左移动以缩小窗口范围，释放多余元素。
- 整个过程中，通过实时维护窗口内的 “状态”（如元素和、字符频率等），同步更新最优解。

### 二、滑动窗口的分类与适用场景

根据窗口大小是否固定，滑动窗口可分为两类，适用场景和处理逻辑差异显著：

#### 1. 固定大小窗口（窗口长度不变）

**特征**：窗口长度 `k` 是固定的（如 “长度为 3 的子数组最大和”），核心是**在窗口滑动时高效更新窗口状态**。
**处理逻辑**：

- 初始化窗口：`left=0`，`right=k-1`，计算初始窗口的状态（如和、乘积）。
- 滑动窗口：`left` 和 `right` 同时向右移动（每次移动 1 步），通过 “减去离开窗口的元素，加上进入窗口的元素” 更新状态，避免重新计算整个窗口。

**典型例题**：

- **LeetCode 643. 子数组最大平均数 I**：给定数组和整数 `k`，求长度为 `k` 的子数组的最大平均数。
  思路：计算初始窗口和，然后每次滑动时减去 `nums[left]` 并加上 `nums[right+1]`，同步更新最大和。

go



```go
func findMaxAverage(nums []int, k int) float64 {
    currentSum := 0
    // 初始化窗口
    for i := 0; i < k; i++ {
        currentSum += nums[i]
    }
    maxSum := currentSum
    // 滑动窗口
    for right := k; right < len(nums); right++ {
        currentSum += nums[right] - nums[right-k] // 减去离开的元素，加上进入的元素
        if currentSum > maxSum {
            maxSum = currentSum
        }
    }
    return float64(maxSum) / float64(k)
}
```

#### 2. 可变大小窗口（窗口长度动态变化）

**特征**：窗口长度不固定，需根据条件动态扩张或收缩（如 “最长无重复字符子串”“最小覆盖子串”）。这类问题更复杂，也是滑动窗口的核心应用场景。
**处理逻辑**：

- 扩张：`right` 向右移动，将新元素纳入窗口，更新窗口状态（如字符频率、和等）。
- 收缩：当窗口不满足条件（或需要优化结果）时，`left` 向右移动，移除元素并更新状态，直到窗口重新满足条件。
- 关键：明确 “满足条件” 的定义（如 “窗口包含所有目标字符”“窗口和≥target”），并设计高效的状态更新方式。

### 三、可变窗口的典型题型分类（附 LeetCode 例题）

可变窗口问题是滑动窗口的重点，根据 “条件类型” 可分为以下 6 类，每类对应不同的解题策略：

#### 1. 子串 / 子数组的 “最值” 问题（最长 / 最短）

**问题特征**：寻找满足条件的最长或最短连续子串 / 子数组（如 “最长无重复字符子串”“最短子数组和≥target”）。
**核心策略**：

- 最长：扩张窗口至不满足条件，收缩时记录最大长度。
- 最短：扩张窗口至满足条件，收缩时尝试缩小窗口并记录最小长度。

**例题 1：LeetCode 3. 无重复字符的最长子串**
目标：找到字符串中不含重复字符的最长子串长度。
思路：用哈希表记录字符最后出现的位置，`right` 扩张时若遇重复字符，`left` 跳至 “重复位置 + 1”（保证窗口内无重复），同步更新最长长度。

go

```go
func lengthOfLongestSubstring(s string) int {
    lastOccur := make(map[byte]int) // 记录字符最后出现的索引
    maxLen, left := 0, 0
    for right := 0; right < len(s); right++ {
        c := s[right]
        // 若c已出现且在当前窗口内（lastOccur[c] >= left），则移动left到重复位置+1
        if pos, ok := lastOccur[c]; ok && pos >= left {
            left = pos + 1
        }
        // 更新最长长度
        if right-left+1 > maxLen {
            maxLen = right - left + 1
        }
        lastOccur[c] = right // 更新c的最后出现位置
    }
    return maxLen
}
```

**例题 2：LeetCode 209. 长度最小的子数组**
目标：找到和≥target 的最短子数组长度（若不存在返回 0）。
思路：`right` 扩张至和≥target，然后收缩 `left` 尝试减小窗口，记录最小长度。

go

```go
func minSubArrayLen(target int, nums []int) int {
    minLen := len(nums) + 1 // 初始化为一个大于可能值的数
    currentSum, left := 0, 0
    for right := 0; right < len(nums); right++ {
        currentSum += nums[right]
        // 当和≥target时，收缩left以寻找更短窗口
        for currentSum >= target {
            windowLen := right - left + 1
            if windowLen < minLen {
                minLen = windowLen
            }
            currentSum -= nums[left]
            left++
        }
    }
    if minLen == len(nums)+1 {
        return 0
    }
    return minLen
}
```

#### 2. 子串包含 “目标集合” 问题

**问题特征**：寻找包含所有目标元素（或满足特定计数条件）的子串（如 “最小覆盖子串”“找到字符串中所有字母异位词”）。
**核心策略**：

- 用哈希表记录目标元素的 “需求计数” 和窗口内的 “当前计数”。
- 扩张窗口：当窗口元素满足 “当前计数≥需求计数” 时，标记为 “有效窗口”。
- 收缩窗口：在有效窗口内收缩 `left`，尝试找到最小长度或所有可能解。

**例题：LeetCode 76. 最小覆盖子串**
目标：找到 `s` 中包含 `t` 所有字符的最小子串（字符可无序，数量需匹配）。
思路：用 `need` 记录 `t` 中字符的需求，`window` 记录窗口内字符的当前计数，通过 `valid` 标记是否满足所有需求，收缩时更新最小子串。

go

```go
import "math"

func minWindow(s string, t string) string {
    need, window := make(map[byte]int), make(map[byte]int)
    for i := 0; i < len(t); i++ {
        need[t[i]]++
    }
    left, right, valid := 0, 0, 0 // valid：满足需求的字符种类数
    start, minLen := 0, math.MaxInt32

    for right < len(s) {
        c := s[right]
        right++
        // 若c是需要的字符，更新window计数
        if need[c] > 0 {
            window[c]++
            // 当window中c的数量达到需求，valid+1
            if window[c] == need[c] {
                valid++
            }
        }

        // 当所有字符都满足需求，开始收缩left
        for valid == len(need) {
            // 更新最小子串
            if right-left < minLen {
                start = left
                minLen = right - left
            }
            // 移除left指向的字符
            d := s[left]
            left++
            if need[d] > 0 {
                // 若移除后window中d的数量小于需求，valid-1
                if window[d] == need[d] {
                    valid--
                }
                window[d]--
            }
        }
    }

    if minLen == math.MaxInt32 {
        return ""
    }
    return s[start : start+minLen]
}
```

#### 3. 子数组和的 “计数 / 存在性” 问题

**问题特征**：统计和为 `k` 的子数组个数，或判断是否存在和为 `k` 的子数组（如 “和为 K 的子数组”“连续的子数组和”）。
**核心策略**：

- 对于非负数组：可用滑动窗口直接计算和，扩张时累加，收缩时递减（因非负，和随窗口增大单调递增）。
- 对于含负数数组：滑动窗口不适用（和可能波动），需用前缀和 + 哈希表，但部分特殊场景（如判断是否存在）仍可优化。

**例题：LeetCode 560. 和为 K 的子数组**
目标：统计和为 `k` 的连续子数组的个数（数组可含负数）。
说明：此题为含负数的情况，严格来说不适合滑动窗口（因和不单调），但可通过 “前缀和 + 哈希表” 解决（作为对比）。

go

```go
// 前缀和+哈希表解法（非滑动窗口，但常用于对比）
func subarraySum(nums []int, k int) int {
    prefixSum := make(map[int]int)
    prefixSum[0] = 1 // 前缀和为0的情况（子数组从0开始）
    sum, count := 0, 0
    for i := 0; i < len(nums); i++ {
        sum += nums[i]
        // 若存在前缀和为 sum-k，则子数组和为k
        if cnt, ok := prefixSum[sum-k]; ok {
            count += cnt
        }
        prefixSum[sum]++
    }
    return count
}
```

**特殊场景**：若数组全为非负数，可直接用滑动窗口统计（和随窗口增大单调递增，收缩时和减小）。

#### 4. 字符串匹配问题（异位词 / 回文）

**问题特征**：寻找与目标字符串 “结构相似” 的子串（如字母异位词、回文子串）。
**核心策略**：

- 字母异位词：通过哈希表记录字符频率，窗口内频率与目标一致则匹配。
- 回文子串：结合中心扩展法（双指针的变种），对每个中心向两边扩张，判断是否为回文。

**例题：LeetCode 438. 找到字符串中所有字母异位词**
目标：找到 `s` 中所有是 `p` 的字母异位词的子串（字符相同，顺序不同）。
思路：固定窗口长度为 `len(p)`，用哈希表对比窗口与 `p` 的字符频率，频率一致则记录起始索引。

go

```go
func findAnagrams(s string, p string) []int {
    res := []int{}
    lenS, lenP := len(s), len(p)
    if lenS < lenP {
        return res
    }

    // 初始化目标频率和窗口频率
    target, window := [26]int{}, [26]int{}
    for i := 0; i < lenP; i++ {
        target[p[i]-'a']++
        window[s[i]-'a']++
    }
    if target == window {
        res = append(res, 0)
    }

    // 滑动窗口（长度固定为lenP）
    for i := lenP; i < lenS; i++ {
        // 移除左边离开的字符，加入右边进入的字符
        window[s[i-lenP]-'a']--
        window[s[i]-'a']++
        if target == window {
            res = append(res, i-lenP+1)
        }
    }
    return res
}
```

#### 5. 限定条件的子数组问题（如最多 k 个不同元素）

**问题特征**：寻找最多包含 `k` 个不同元素的最长子数组 / 子串（如 “最长连续不重复子数组（最多 k 个重复）”）。
**核心策略**：

- 用哈希表记录窗口内元素的频率，当不同元素数超过 `k` 时，收缩 `left` 直到满足条件，同步更新最长长度。

**例题：LeetCode 340. 至多包含 K 个不同字符的最长子串**
目标：找到最多包含 `k` 个不同字符的最长子串长度。

go

```go
func lengthOfLongestSubstringKDistinct(s string, k int) int {
    if k == 0 {
        return 0
    }
    freq := make(map[byte]int) // 记录窗口内字符频率
    maxLen, left := 0, 0
    for right := 0; right < len(s); right++ {
        c := s[right]
        freq[c]++ // 扩张窗口，加入字符c

        // 当不同字符数超过k，收缩left
        for len(freq) > k {
            d := s[left]
            freq[d]--
            if freq[d] == 0 {
                delete(freq, d) // 移除频率为0的字符
            }
            left++
        }

        // 更新最长长度
        if right-left+1 > maxLen {
            maxLen = right-left+1
        }
    }
    return maxLen
}
```

#### 6. 数组中 “区间操作” 问题（如替换后的最长重复字符）

**问题特征**：允许对窗口内元素进行有限次修改（如替换 `k` 个字符），寻找最长有效区间。
**核心策略**：

- 用哈希表记录窗口内 “最多出现的字符次数”，当 `窗口长度 - 最多次数 ≤ k` 时，可通过替换使窗口内字符全相同，否则收缩窗口。

**例题：LeetCode 424. 替换后的最长重复字符**
目标：最多替换 `k` 个字符，使子串中所有字符相同，求最长长度。

go



```go
func characterReplacement(s string, k int) int {
    freq := make(map[byte]int)
    maxLen, maxFreq, left := 0, 0, 0
    for right := 0; right < len(s); right++ {
        c := s[right]
        freq[c]++
        // 更新窗口内最多出现的字符次数
        if freq[c] > maxFreq {
            maxFreq = freq[c]
        }

        // 当需要替换的次数（窗口长度 - maxFreq）>k时，收缩left
        for right-left+1 - maxFreq > k {
            d := s[left]
            freq[d]--
            left++
        }

        // 更新最长长度（窗口始终满足替换条件）
        if right-left+1 > maxLen {
            maxLen = right-left+1
        }
    }
    return maxLen
}
```

### 四、滑动窗口的优化技巧

1. **哈希表高效记录状态**：对于字符频率、元素计数等，用哈希表（或数组，针对有限字符集如小写字母）实时维护，避免每次重新计算。
2. **避免不必要的收缩**：在求 “最长” 问题时，窗口一旦满足条件，无需收缩至不满足，只需记录当前长度并继续扩张（如无重复字符子串）。
3. **固定窗口的 “增量更新”**：对于固定长度窗口，通过 “减去离开元素 + 加上进入元素” 更新状态，时间复杂度降至 O (n)。
4. **处理重复元素**：在需要去重的场景（如无重复子串），用哈希表记录元素最后出现的位置，直接跳转到冲突位置 + 1，减少收缩次数。

### 五、滑动窗口与其他算法的对比

| 算法            | 适用场景                            | 时间复杂度 | 空间复杂度                       |
| --------------- | ----------------------------------- | ---------- | -------------------------------- |
| 滑动窗口        | 连续子数组 / 子串，需动态维护区间   | O(n)       | O (1) 或 O (k)（k 为字符集大小） |
| 前缀和 + 哈希表 | 子数组和问题（含负数）              | O(n)       | O(n)                             |
| 动态规划        | 子数组 / 子串的最优解（非连续也可） | O(n²)      | O (n) 或 O (n²)                  |

**总结**：滑动窗口是处理 “连续区间” 问题的最优选择，尤其当区间需要动态调整且可通过局部状态判断时，效率远高于暴力法或动态规划。

### 六、如何快速识别滑动窗口问题？

当题目满足以下特征时，优先考虑滑动窗口：

1. 输入为线性结构（数组、字符串、链表）。
2. 问题涉及 “连续子序列”（子数组、子串）。
3. 需要求解 “最值”（最长 / 最短）、“计数”（符合条件的数量）或 “存在性”（是否有满足条件的区间）。
4. 可通过 “扩张 - 收缩” 窗口维护某种状态（如和、频率、重复次数）。

滑动窗口的核心是 “动态维护窗口状态”，关键在于明确 “扩张与收缩的条件”。掌握这种思想后，大多数连续子序列问题都能在 O (n) 时间内解决，极大提升算法效率。



滑动窗口算法的高效性，源于对 “窗口状态” 的精准维护和 “指针移动” 的策略性设计。要真正掌握这一算法，需深入理解其底层技巧 —— 如何通过最小化状态更新成本、优化指针移动逻辑，将复杂问题转化为线性时间求解。以下从**状态设计、指针移动、边界处理、性能优化**四个维度，拆解滑动窗口的核心技巧。

### 一、窗口状态的 “极简设计” 技巧

滑动窗口的核心是通过维护 “窗口状态”（如元素和、字符频率、最大 / 最小值等）判断窗口是否满足条件。状态设计的核心原则是：**只记录必要信息，避免冗余计算**。

#### 1. 状态变量的选择：按需取舍

根据问题目标选择最精简的状态变量，避免记录无关信息：

- **子串包含问题**（如最小覆盖子串）：需记录 “目标字符的需求计数” 和 “窗口内的当前计数”，以及 “满足需求的字符种类数”（`valid`）。无需记录所有字符，只需关注目标字符。
- **子数组和问题**：只需记录 “当前窗口的和”，无需记录每个元素。
- **重复字符问题**（如最长无重复子串）：只需记录 “字符最后出现的位置”，无需记录所有出现位置。
- **替换 k 个字符问题**（如最长重复字符）：只需记录 “窗口内出现次数最多的字符频率”（`maxFreq`），无需记录所有字符的频率细节。

**示例**：在 “最长重复字符替换” 中，`maxFreq` 无需实时更新（即使收缩窗口后 `maxFreq` 可能变小），因为我们只关心 “可能的最长窗口”—— 即使 `maxFreq` 不是当前窗口的真实最大值，也不会错过最优解（因为收缩后的窗口长度一定小于之前的最长值）。这种 “懒惰更新” 减少了状态维护成本。

#### 2. 状态存储的优化：数组 vs 哈希表

- 有限字符集

  （如小写字母、数字）：用数组替代哈希表，提升访问速度。例如处理 26 个小写字母时，用

  

  ```
  [26]int
  ```

   

  记录频率，比

  

  ```
  map[byte]int
  ```

   

  效率更高。

  go

  

  ```go
  // 字母频率统计：数组比哈希表更高效
  freq := [26]int{}
  for _, c := range s {
      freq[c-'a']++
  }
  ```

- **无限字符集**（如任意字符串、整数）：必须用哈希表，但可通过 “延迟删除” 减少操作（如当元素频率为 0 时不立即删除，仅在判断大小时忽略）。

### 二、指针移动的 “策略性收缩” 技巧

滑动窗口的指针移动（尤其是 `left` 的收缩）是性能关键。低效的收缩会导致时间复杂度退化（如 O (n²)），而策略性收缩可保持 O (n) 复杂度。

#### 1. 收缩的 “必要性判断”：避免无效操作

- **求最长窗口**（如最长无重复子串）：仅当窗口不满足条件时收缩（如出现重复字符），收缩至满足条件后立即停止，继续扩张 `right`。无需收缩到最小，因为更长的窗口可能在后续出现。
- **求最短窗口**（如最小覆盖子串）：一旦窗口满足条件，立即收缩 `left` 以尝试更小的有效窗口，直到不满足条件再继续扩张 `right`。

**反例**：在 “最长无重复子串” 中，若遇到重复字符后，`left` 一步一步移动（而非直接跳至重复位置 + 1），会导致 O (n²) 时间复杂度。正确做法是利用哈希表记录字符最后出现的位置，直接跳转：

go



```go
// 高效收缩：直接跳至重复位置+1
if pos, ok := lastOccur[c]; ok && pos >= left {
    left = pos + 1 // 一步到位，避免逐次移动
}
```

#### 2. 收缩的 “单调性利用”：基于问题特性设计规则

滑动窗口的有效性依赖于 “问题的单调性”—— 即窗口扩张 / 收缩时，条件的满足状态具有可预测性：

- **非负数组的子数组和**：窗口和随 `right` 扩张而增大，随 `left` 收缩而减小（单调），因此可通过收缩 `left` 找到最小有效窗口。
- **字符频率匹配**：窗口包含的目标字符种类随 `right` 扩张而增多，随 `left` 收缩而减少（单调），因此可通过 `valid` 变量判断是否满足条件。

**反例**：若数组含负数，子数组和不单调（扩张可能使和减小，收缩可能使和增大），滑动窗口不再适用（需用前缀和 + 哈希表）。

#### 3. 固定窗口的 “增量更新”：避免重复计算

固定长度窗口（如长度为 `k`）的核心技巧是**增量更新状态**：每次滑动时，通过 “减去离开窗口的元素，加上进入窗口的元素” 更新状态，而非重新计算整个窗口。

**示例**：求长度为 `k` 的子数组最大和：

go

```go
// 初始窗口和
currentSum := sum(nums[0...k-1])
maxSum := currentSum
// 滑动时增量更新（O(1) per step）
for right := k; right < len(nums); right++ {
    currentSum += nums[right] - nums[right-k] // 减去离开的，加上进入的
    maxSum = max(maxSum, currentSum)
}
```

若每次滑动都重新计算 `sum(nums[right-k+1...right])`，时间复杂度会从 O (n) 退化到 O (nk)。

### 三、边界与特殊情况的 “无死角处理” 技巧

滑动窗口的边界条件（如窗口为空、数组长度为 0、目标长度大于数组长度）容易被忽略，需针对性处理。

#### 1. 初始化边界：明确窗口起点

- 可变窗口：初始 `left=0, right=0`（窗口为空），`right` 先扩张。
- 固定窗口：若 `k > len(nums)`，直接返回空结果（如 “找到所有字母异位词” 中，若 `len(p) > len(s)`，则无结果）。

#### 2. 循环终止条件：避免越界

- 可变窗口：`right < len(s)`（`right` 遍历至末尾即可，`left` 无需单独遍历）。
- 固定窗口：`right < len(s)`（滑动次数为 `len(s)-k`，需确保 `right` 不越界）。

#### 3. 结果修正：处理 “无有效窗口” 的情况

- 求最短窗口时，若始终无有效窗口，需返回 0 或空串（如 `minLen` 仍为初始值时）。
- 求最长窗口时，若数组为空，直接返回 0。

**示例**：最小覆盖子串中，若未找到有效窗口，返回空串：

go

```go
if minLen == math.MaxInt32 {
    return ""
}
```

### 四、性能优化的 “细节把控” 技巧

滑动窗口的时间复杂度理论上是 O (n)，但实现细节可能导致实际效率差异，需关注以下优化点：

#### 1. 减少哈希表操作：延迟删除与批量更新

哈希表的插入 / 删除操作耗时，可通过 “延迟删除” 优化：当元素频率减为 0 时，不立即从哈希表中删除，仅在判断窗口内不同元素数量时忽略（适用于 “最多 k 个不同元素” 问题）。

#### 2. 避免重复计算：缓存中间结果

- 对于需要频繁计算的窗口属性（如最大 / 最小值），可通过辅助数据结构（如单调队列）实时维护，避免每次遍历窗口求解（如 “滑动窗口最大值” 问题）。

#### 3. 早停策略：提前退出无效循环

在某些场景下，当窗口无法继续优化时，可提前退出：

- 求最长窗口时，若剩余元素长度不足以超过当前最长值，可提前终止。

  go

  ```go
  // 剩余元素不足以超过当前最长，提前退出
  if right + (maxLen - (right - left)) >= len(s) {
      break
  }
  ```

### 五、技巧总结：滑动窗口的 “四字诀”

1. **准**：状态设计精准，只记录必要信息（如 `valid` 标记、`maxFreq`）。
2. **快**：指针移动高效，利用单调性直接跳转（如重复字符时 `left` 跳至冲突位置 + 1）。
3. **省**：状态更新增量，固定窗口避免重复计算（如增量求和）。
4. **全**：边界处理全面，覆盖空数组、无有效窗口等特殊情况。

滑动窗口的本质是 “用局部遍历替代全局遍历”，而上述技巧的核心是 “最小化局部遍历的成本”。掌握这些技巧后，无论是简单的子数组和问题，还是复杂的最小覆盖子串问题，都能在 O (n) 时间内高效求解。
