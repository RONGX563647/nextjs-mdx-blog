##### 一、Java 核心认知

#### 1. Java 语言特性

| 特性 | 说明 |
|:------|:------|
| **🌐 跨平台性** | 一次编写，多处运行。JVM 将字节码翻译成对应系统的机器指令 |
| **🎯 面向对象** | 一切皆对象，支持封装、继承、多态 |
| **💪 健壮性** | 强类型检查、异常处理、自动垃圾回收 |
| **🔒 安全性** | 字节码验证、安全管理器、沙箱机制 |
| **⚡ 多线程** | 语言级别内置多线程支持 |

#### 2. Java 三大技术平台

| 平台 | 全称 | 定位 | 应用场景 |
|:------|:------|:------|:----------|
| **☕ Java SE** | Standard Edition | 核心与基础 | 桌面应用、控制台程序、学习基础 |
| **🏢 Java EE** | Enterprise Edition | 企业级解决方案 | 电商、金融、政务等服务器端开发 |

#### 3. JDK/JRE/JVM 关系

```text

├── JRE（Java运行环境）

```

> - 安装 JDK 后无需再装 JRE（JDK 已包含）
> - 不同操作系统的 JVM 不同，但字节码文件完全一致

##### 二、环境搭建

#### 1. 下载与安装 JDK

- **📦 推荐版本**：JDK 8 或 JDK 11（企业主流，兼容性最好）
- **🔗 下载地址**：Oracle 官网 / 华为镜像（下载更快）
- **⚠️ 安装注意**：
  - 路径不要有中文或空格（如 `D:\Java\jdk1.8.0_301`）
  - 记住安装路径，后续配置需要

---

#### 2. 配置环境变量

**🪟 Windows 系统配置步骤**：

1. 右键「此电脑」→「属性」→「高级系统设置」→「环境变量」

2. 新建系统变量 `JAVA_HOME`：
   - 变量名：`JAVA_HOME`
   - 变量值：`D:\Java\jdk1.8.0_301`（你的 JDK 安装路径）

3. 编辑系统变量 `Path`：
   - 新建条目：`%JAVA_HOME%\bin`

> **💡 配置原理**：
> - `JAVA_HOME`：告诉系统 JDK 安装位置
> - `Path`：让系统在任何目录都能识别 `javac`、`java` 命令

---

#### 3. 验证安装

打开 CMD，执行以下命令：

```bash
##### 验证 Java 编译器
javac -version

##### 验证 Java 运行时
java -version
```

**✅ 成功标志**：显示版本号（如 `javac 1.8.0_301`）

> **⚠️ 常见问题**：
> - 提示"不是内部或外部命令"→ 检查 `JAVA_HOME` 和 `Path` 配置
> - 配置后仍失败→ 重启 CMD 窗口

#### 3. 核心要求（必须遵守）

| 要求 | 说明 | ❌ 错误示例 |
|:------|:------|:----------|
| **📝 类名与文件名一致** | `public class HelloWorld` 对应 `HelloWorld.java` | 类名 `Hello` 文件名 `hello.java` |
| **⚙️ main 方法固定** | `public static void main(String[] args)` | 拼写错误或参数不对 |
| **💬 语句以分号结束** | 每条语句末尾加 `;` | `System.out.println("hello")` |

##### 四、基础语法规范

#### 1. 注释类型

```java
// 单行注释：解释单行代码

/*
 * 多行注释：解释代码块
 * 可以写多行
 */

/**
 * 文档注释：生成 API 文档
 * @author 作者名
 * @version 版本号
 */
```

> **💡 注释使用建议**：
> - 单行注释：解释简单逻辑
> - 多行注释：解释复杂算法或代码块
> - 文档注释：用于生成 API 文档

---

#### 2. 标识符命名规范

| 类型 | 规范 | ✅ 示例 |
|:------|:------|:------|
| **📦 类名** | 大驼峰（首字母大写）| `HelloWorld`、`StudentManager` |
| **🔧 方法/变量** | 小驼峰（首字母小写）| `getName`、`studentAge` |
| **🔢 常量** | 全大写，下划线分隔 | `MAX_VALUE`、`PI` |
| **📁 包名** | 全小写，域名倒写 | `com.company.project` |

> **⚠️ 命名规范**：
> - 见名知意：变量名要能表达其用途
> - 避免拼音：使用英文单词
> - 遵循约定：保持代码风格一致

#### 3. 8 种基本数据类型

| 类型 | 关键字 | 占用内存 | 取值范围 | 默认值 |
|:------|:--------|:----------|:----------|:--------|
| 字节型 | `byte` | 1 字节 | -128 ~ 127 | 0 |
| 短整型 | `short` | 2 字节 | -32768 ~ 32767 | 0 |
| 整型 | `int` | 4 字节 | 约 -21亿 ~ 21亿 | 0 |
| 长整型 | `long` | 8 字节 | 很大 | 0L |
| 单精度浮点 | `float` | 4 字节 | 约 ±3.4E38 | 0.0F |
| 双精度浮点 | `double` | 8 字节 | 约 ±1.7E308 | 0.0D |
| 字符型 | `char` | 2 字节 | 0 ~ 65535 | '\u0000' |
| 布尔型 | `boolean` | 1 字节 | true / false | false |

> **⚠️ 特别注意**：
> - `long` 类型赋值超过 `int` 范围时必须加 `L`：`long num = 10000000000L;`
> - `float` 类型赋值必须加 `F`：`float f = 3.14F;`

##### 五、程序执行流程详解

```
┌─────────────────────────────────────────────────────────────┐
│                     Java 程序执行流程                        │
├─────────────────────────────────────────────────────────────┤
│  1. 编写源代码    │  HelloWorld.java（人类可读）            │
│        ↓          │                                         │
│  2. 编译（javac） │  javac HelloWorld.java                  │
│        ↓          │                                         │
│  3. 生成字节码    │  HelloWorld.class（JVM可读，跨平台）    │
│        ↓          │                                         │
│  4. 运行（java）  │  java HelloWorld                        │
│        ↓          │                                         │
│  5. JVM 加载执行  │  类加载 → 验证 → 准备 → 解析 → 初始化   │
│        ↓          │                                         │
│  6. 输出结果      │  Hello, Java!                           │
└─────────────────────────────────────────────────────────────┘
```

> **💡 核心理解**：
> - `.java` 文件：源代码，人类编写和阅读
> - `.class` 文件：字节码，JVM 可执行，与平台无关
> - **跨平台本质**：不同系统安装对应的 JVM，执行同一份字节码

#### ❓ 问答

##### Q1：为什么说 Java 是跨平台的？

**答**：Java 程序编译后生成字节码（`.class`），而非特定系统的机器码。不同操作系统（Windows、Linux、Mac）只需安装对应的 JVM，JVM 将字节码翻译成该系统的机器指令执行。因此，"一次编写，到处运行"。

---

##### Q2：JDK 和 JRE 的区别是什么？

**答**：
- **JDK（Java Development Kit）**：Java 开发工具包，包含 JRE 和开发工具（javac、java、javadoc 等），用于开发 Java 程序。
- **JRE（Java Runtime Environment）**：Java 运行环境，包含 JVM 和核心类库，用于运行 Java 程序。
- **关系**：JDK ⊃ JRE ⊃ JVM。开发需要 JDK，运行只需要 JRE。

---

##### Q3：环境变量的作用是什么？为什么要配置 JAVA_HOME 和 Path？

**答**：
- **环境变量**：告诉操作系统在哪里找到可执行程序。
- **JAVA_HOME**：指向 JDK 的安装根目录，便于其他程序（如 IDE、Maven）找到 JDK。
- **Path**：添加 `%JAVA_HOME%\bin` 后，系统在任何目录都能识别 `javac`、`java` 等命令，无需进入 JDK 的 bin 目录执行。

---

##### Q4：Java 程序从编写到运行的完整流程是什么？

**答**：
1. **编写**：使用文本编辑器或 IDE 编写 `.java` 源代码文件
2. **编译**：使用 `javac` 命令将源代码编译为 `.class` 字节码文件
3. **运行**：使用 `java` 命令启动 JVM，加载并执行字节码
4. **执行**：JVM 将字节码翻译成机器指令，程序运行并输出结果

---

##### Q5：public class 的类名必须与文件名一致吗？

**答**：**必须一致**（包括大小写）。如果类声明为 `public class HelloWorld`，则文件名必须是 `HelloWorld.java`。一个源文件只能有一个 `public` 类，但可以有多个非 `public` 类。
