1.3 跨域配置

WebConfig.java - CORS 跨域配置：

@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("http://localhost:5173")
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .allowCredentials(true);
    }
}

1.4 前端 API 封装

api/request.js - Axios 封装：

import axios from 'axios'
import { ElMessage } from 'element-plus'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000
})

// 请求拦截器
request.interceptors.request.use(
  config => {
    const userInfo = localStorage.getItem('userInfo')
    if (userInfo) {
      config.headers['Authorization'] = 'Bearer ' + JSON.parse(userInfo).token
    }
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

// 响应拦截器
request.interceptors.response.use(
  response => {
    return response.data
  },
  error => {
    ElMessage.error(error.response?.data?.message || '请求失败')
    return Promise.reject(error)
  }
)

export default request

4.2 体系结构视图要点

  • 前端层：Vue 3 + Vite + Element Plus + Axios
  • 后端层：Spring Boot + Spring MVC + JPA
  • 中间件层：RabbitMQ 消息队列
  • 数据存储：MySQL 数据库
  • 区块链核心：Block、Transaction、Wallet、Blockchain、Node

4.3 开发视图要点

  • Controller 层：处理 HTTP 请求，调用 Service 层
  • Service 层：业务逻辑实现，事务管理
  • Model 层：数据模型定义（Block、Transaction、Wallet、User）
  • Core 层：区块链核心逻辑（Blockchain、HashUtil）
  • Network 层：P2P 网络通信（Node、NodeInfo）
  • Config 层：RabbitMQ 配置、CORS 配置

4.4 部署视图要点

  • 开发环境：JDK 17、Node.js 16+、Maven 3.8+、RabbitMQ 3.x
  • 后端部署：./mvnw spring-boot:run 或 java -jar
  • 前端部署：npm run dev（开发）或 npm run build（生产）
  • 服务端口：前端 5173，后端 8080，RabbitMQ 5672/15672

五、常见问题与解决方案

5.1 后端启动失败

# 问题：端口被占用
# 解决：修改 application.yml 中的 server.port
server:
  port: 8081

# 问题：RabbitMQ 连接失败
# 解决：确保 RabbitMQ 服务已启动
# Windows: net start RabbitMQ
# 或: rabbitmq-service start

5.2 前端无法访问后端 API

# 问题：跨域错误
# 解决：确保 WebConfig 中配置了 CORS
# 检查 application.yml 中前端地址是否正确

# 问题：API 请求 404
# 解决：检查 Controller 的 @RequestMapping 路径
# 确保前端 axios 的 baseURL 配置正确

5.3 挖矿耗时过长

# 问题：挖矿难度过高导致等待时间太长
# 解决：降低 difficulty 值
# 在 Blockchain 构造函数中修改:
this.difficulty = 2;  // 从 4 降低到 2

# 或在 application.yml 中配置:
blockchain:
  difficulty: 2

5.4 消息队列消息未消费

# 问题：消费者未启动
# 解决：确保 RabbitMQ 服务运行正常
# 检查消费者类的 @RabbitListener 注解是否正确

# 问题：消息未路由到队列
# 解决：检查 Exchange 和 Queue 的绑定关系
# 访问 http://localhost:15672 查看绑定状态
