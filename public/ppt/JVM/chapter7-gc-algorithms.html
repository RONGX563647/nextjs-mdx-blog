第七章：垃圾回收算法 - 深入底层原理

不同的垃圾回收算法有不同的特点和适用场景。本章深入探讨算法实现、三色标记理论和底层数据结构。

7.1 标记-清除算法（Mark-Sweep）

  • 执行阶段：标记阶段（从GC Roots扫描存活对象）+ 清除阶段（回收未标记对象）
  • 标记实现：使用Bitmap（位图）标记对象，1=存活，0=可回收
  • 底层数据结构：Mark Bitmap（标记位图），每个对象对应1个bit
  • 优点：实现简单、无需额外空间、不移动对象
  • 致命缺点：产生大量内存碎片，导致大对象无法分配
  • 空间复杂度：O(1)额外空间（位图），但碎片导致利用率降低
  • 使用场景：CMS收集器的老年代回收

// 标记-清除算法伪代码
void markSweep() {
  // 标记阶段 - DFS遍历所有存活对象
  for (root : gcRoots) {
    mark(root); // 递归标记所有可达对象
  }

  // 清除阶段 - 扫描堆内存，回收未标记对象
  for (obj : heap) {
    if (!isMarked(obj)) {
      free(obj); // 释放内存，加入空闲链表
    } else {
      unmark(obj); // 清除标记，为下次GC准备
    }
  }
}

7.2 标记-复制算法（Mark-Copy）

  • 核心思想：将内存分为两块，每次只使用一块，GC时将存活对象复制到另一块
  • 底层实现：Eden + Survivor0 + Survivor1（8:1:1比例）
  • 指针碰撞分配：空闲内存连续，使用指针分配，仅需CAS操作
  • 优点：无内存碎片、分配效率极高、实现简单
  • 缺点：可用内存缩小为一半（实际新生代为90%）
  • 复制开销：仅复制存活对象，新生代存活率低（通常<10%）
  • 使用场景：所有新生代收集器（Serial Copy、ParNew、PS Scavenge、G1 Young）

7.3 标记-整理算法（Mark-Compact）

  • 核心思想：标记存活对象后，将所有存活对象向一端移动，然后清理边界外内存
  • 实现难点：需要更新所有引用对象的地址（指针）
  • 三步实现（Lisp2算法）：
      ① 标记阶段：计算每个存活对象的新地址
      ② 更新引用：遍历所有引用，更新为新地址
      ③ 移动对象：复制对象到新位置
  • 优点：无内存碎片、内存利用率高
  • 缺点：移动对象开销大、需要暂停所有线程（STW）
  • 使用场景：老年代回收（Serial Old、Parallel Old）

// 标记-整理算法伪代码（Lisp2简化版）
void markCompact() {
  // 阶段1：标记存活对象
  markPhase();

  // 阶段2：计算新地址
  address newAddress = heapStart;
  for (obj : heap) {
    if (isMarked(obj)) {
      setForwardingPointer(obj, newAddress); // 设置转发指针
      newAddress += obj.size;
    }
  }

  // 阶段3：更新所有引用
  updateAllReferences(); // 遍历GC Roots和对象引用

  // 阶段4：移动对象
  for (obj : heap) {
    if (isMarked(obj)) {
      move(obj, getForwardingPointer(obj));
    }
  }
}

  • 初始状态：所有对象都是白色
  • 标记过程：从GC Roots开始，根对象变为灰色 → 扫描灰色对象 → 变为黑色
  • 并发标记问题：用户线程和GC线程并发执行，可能导致对象状态变化
  • 两种破坏情况（导致漏标）：
      ① 黑色对象引用指向白色对象（赋值插入）
      ② 灰色对象到白色对象的引用被断开
  • 解决方案：增量更新（Incremental Update）或SATB（Snapshot At The Beginning）
// 三色标记过程伪代码
void concurrentMark() {
  // 1. 初始标记（STW） - 标记GC Roots直接引用的对象
  for (root : gcRoots) {
    markGray(root);
  }

  // 2. 并发标记 - 与用户线程并发执行
  while (hasGrayObjects()) {
    grayObj = getGrayObject();
    for (ref : grayObj.references) {
      if (isWhite(ref)) {
        markGray(ref); // 白色→灰色
      }
    }
    markBlack(grayObj); // 灰色→黑色
  }

  // 3. 最终标记（STW） - 处理并发期间的变化
  processSatbLog(); // 或处理增量更新日志

  // 4. 清理阶段
  sweepWhiteObjects(); // 回收所有白色对象
}

7.5 读写屏障（Read/Write Barrier）- 并发GC核心机制

  • 读屏障：在读取对象引用时插入的代码，用于维护并发标记的正确性
  • 写屏障：在修改对象引用时插入的代码，用于记录引用变化
  • 底层实现：通过JVM编译器在字节码层面自动插入
  • 开销：每次读写操作额外增加几纳秒开销，但可实现并发GC
  • SATB（初始快照）：写屏障记录引用删除，使用SATB日志
  • 增量更新：写屏障记录引用新增，CMS采用此方案

void writeBarrier(Object* obj, int fieldOffset, Object* newValue) {
  Object* oldValue = obj->getField(fieldOffset);

  // SATB写屏障：如果删除了到白色对象的引用，记录到日志

  // 执行实际赋值
  obj->setField(fieldOffset, newValue);

7.6 记忆集（Remembered Set）与卡表（Card Table）

  • 记忆集：记录从非收集区指向收集区的引用，避免全堆扫描
  • 卡表：记忆集的一种实现，将堆内存划分为固定大小的"卡页"（通常512字节）
  • 底层数据结构：byte数组（Card Table），每个byte对应一个卡页
  • 卡页状态：0=干净（无跨代引用），非0=脏（有跨代引用）
  • 写屏障更新卡表：当老年代引用新生代对象时，标记对应卡页为"脏"
  • GC时处理：只需扫描脏卡页中的对象，大幅降低扫描开销

// 写屏障更新卡表
void oop_write_barrier(Object* oldObj, Object* newObj) {
  if (isOldGeneration(oldObj) && isNewGeneration(newObj)) {
    // 老年代引用新生代，标记对应卡页为脏
    size_t cardIndex = (uintptr_t)oldObj >> 9; // 除以512
    cardTable[cardIndex] = 1; // 标记为脏卡页
  }
}

// Young GC时扫描脏卡页
void youngGC() {
  for (card : cardTable) {
    if (card == DIRTY) { // 只处理脏卡页
      scanCard(card); // 扫描跨代引用
    }
  }
  // 之后执行常规的Young GC标记-复制
}

7.7 TLAB（Thread Local Allocation Buffer）- 线程本地分配缓冲区

  • 目的：避免多线程在Eden区竞争分配内存时的同步开销
  • 实现原理：每个线程预先分配一小块Eden区，优先在TLAB中分配
  • 默认大小：约Eden区的1%，可通过-XX:TLABSize调整
  • 分配流程：TLAB有空间 → 直接分配（无锁）；TLAB满 → 申请新TLAB或直接在Eden分配
  • 优点：极大提升多线程对象分配性能
  • 控制参数：-XX:+UseTLAB（默认开启）、-XX:TLABSize

7.8 分代收集理论（Generational Collection）

  • 弱分代假说：绝大多数对象朝生夕死，熬过GC的次数越多越难回收
  • 强分代假说：熬过越多次GC的对象，越有可能是存活对象
  • 跨代引用假说：跨代引用相对较少，新生代主要被新生代对象引用
  • 区域划分：新生代（Young）+ 老年代（Old），不同代使用不同算法
  • 对象年龄：对象头中的age字段（4位，最大15），每次Minor GC+1
  • 晋升规则：年龄达到阈值（-XX:MaxTenuringThreshold，默认15）→ 老年代
  • 动态年龄判定：如果同龄对象总大小超过Survivor区50%，年龄≥该年龄的对象直接晋升

💡 算法选择策略

新生代：使用标记-复制算法，因为新生代存活率低（通常<10%），复制开销小
老年代：使用标记-整理或标记-清除算法，因为老年代存活率高，复制开销大
现代JVM：采用分代收集理论，新生代和老年代使用不同的算法组合
并发GC：基于三色标记法，使用读写屏障实现并发标记，减少STW时间
