阶段三：区块链节点管理

一、节点管理概述

区块链节点管理是整个系统的核心功能模块，登录后用户可对自己专属的区块链节点进行全生命周期管理。根据需求文档，节点管理包含以下核心功能：

序号 功能 说明
1 节点信息展示 可视化展示节点地址、钱包地址等基础信息
2 区块链查询 查看完整区块链数据、区块详情
3 交易创建 发起区块链交易，生成交易数据
4 挖矿操作 执行挖矿功能，打包交易生成新区块
5 余额查询 查看矿工钱包余额
6 链长度查询 查询当前区块链总长度
7 数据防篡改验证 校验区块链数据完整性，验证区块是否被篡改
8 邻居节点管理 添加邻居节点
9 节点同步 查看、执行区块链节点数据同步操作

3.3 Block 区块模型

Block.java - 区块数据结构：

@Data
public class Block {
    private String hash;
    private String previousHash;
    private List<Transaction> transactions;
    private long timestamp;
    private int nonce;
    private int index;

    public Block(int index, String previousHash, List<Transaction> transactions) {
        this.index = index;
        this.previousHash = previousHash;
        this.transactions = transactions != null ? transactions : new ArrayList<>();
        this.timestamp = System.currentTimeMillis();
        this.nonce = 0;
        this.hash = calculateHash();
    }

    public String calculateHash() {
        String data = index + previousHash + timestamp + nonce + transactions.toString();
        return HashUtil.sha256(data);
    }

    public void mineBlock(int difficulty) {
        String target = new String(new char[difficulty]).replace('\0', '0');
        while (!hash.substring(0, difficulty).equals(target)) {
            nonce++;
            hash = calculateHash();
        }
        System.out.println("区块挖掘完成！Hash：" + hash);
    }
}

3.3.1 HashUtil 工具类

HashUtil.java - SHA-256 哈希工具类：

public class HashUtil {
    public static String sha256(String data) {
        try {
            MessageDigest digest = MessageDigest.getInstance("SHA-256");
            byte[] hash = digest.digest(data.getBytes());
            StringBuilder hexString = new StringBuilder();
            for (byte b : hash) {
                String hex = Integer.toHexString(0xff & b);
                if (hex.length() == 1) hexString.append('0');
                hexString.append(hex);
            }
            return hexString.toString();
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException("SHA-256 算法不可用", e);
        }
    }
}

3.4 Transaction 交易模型

Transaction.java - 交易数据结构：

@Data
public class Transaction {
    private String transactionId;
    private String senderAddress;
    private String recipientAddress;
    private double amount;
    private String signature;
    private LocalDateTime timestamp;
    private int nonce;

    public Transaction() {
        this.transactionId = UUID.randomUUID().toString();
        this.timestamp = LocalDateTime.now();
        this.nonce = 0;
    }

    public Transaction(String senderAddress, String recipientAddress, double amount) {
        this.transactionId = UUID.randomUUID().toString();
        this.senderAddress = senderAddress;
        this.recipientAddress = recipientAddress;
        this.amount = amount;
        this.timestamp = LocalDateTime.now();
        this.nonce = 0;
    }

    public String calculateHash() {
        return HashUtil.sha256(senderAddress + recipientAddress + amount + timestamp + nonce);
    }

    @Override
    public String toString() {
        return senderAddress + " -> " + recipientAddress + ": " + amount;
    }
}

说明：

  • 使用 Lombok 的 @Data 注解自动生成 getter/setter 方法
  • transactionId：使用 UUID 生成的唯一交易标识
  • senderAddress 和 recipientAddress：交易双方地址
  • amount：交易金额
  • signature：数字签名（可选，用于验证交易真实性）
  • timestamp：交易创建时间
  • toString()：重写方法，方便日志输出

4.2 Service 实现

BlockchainService.java - 区块链业务逻辑层：

@Service
public class BlockchainService {
    private static final Logger logger = LoggerFactory.getLogger(BlockchainService.class);

    @Autowired
    private Blockchain blockchain;

    @Autowired
    private Node node;
    
    private Wallet wallet = new Wallet();

    public Map<String, Object> getNodeInfo() {
        Map<String, Object> info = new HashMap<>();
        info.put("nodeId", node != null ? node.getNodeId() : "local-node");
        info.put("host", node != null ? node.getHost() : "localhost");
        info.put("port", node != null ? node.getPort() : 8080);
        info.put("lastSeen", System.currentTimeMillis());
        return info;
    }

    public Map<String, Object> getStats() {
        Map<String, Object> stats = new HashMap<>();
        stats.put("chainLength", blockchain.getChain().size());
        stats.put("isValid", blockchain.isChainValid());
        stats.put("peerCount", node != null ? node.getPeerNodes().size() : 0);
        stats.put("walletAddress", wallet.getAddress());
        stats.put("walletBalance", getBalance(wallet.getAddress()));
        stats.put("miningReward", blockchain.getMiningReward());
        stats.put("difficulty", blockchain.getDifficulty());
        return stats;
    }

    public Transaction createTransaction(String senderAddress, 
                                          String recipientAddress, 
                                          double amount) {
        Transaction transaction = new Transaction(senderAddress, recipientAddress, amount);
        blockchain.addTransaction(transaction);
        
        if (node != null) {
            node.broadcastTransaction(transaction);
        }
        
        logger.info("交易创建成功：{} -> {} : {}", 
            senderAddress, recipientAddress, amount);
        return transaction;
    }

    public void receiveTransaction(Transaction transaction) {
        blockchain.addTransaction(transaction);
        logger.info("接收到交易：{}", transaction);
    }

    public boolean mineBlock(String minerAddress) {
        boolean success = blockchain.minePendingTransactions(minerAddress);
        
        if (success && node != null) {
            Block latestBlock = blockchain.getLatestBlock();
            node.broadcastBlock(latestBlock);
        }
        return success;
    }

    public double getBalance(String address) {
        return blockchain.getBalance(address);
    }

    public boolean isChainValid() {
        return blockchain.isChainValid();
    }

    public void addPeer(String peerAddress) {
        if (node != null) {
            node.addPeer(peerAddress);
        }
    }

    public void removePeer(String peerAddress) {
        if (node != null) {
            node.removePeer(peerAddress);
        }
    }

    public List<String> getPeers() {
        return node != null ? node.getPeerNodes() : new ArrayList<>();
    }

    public void syncWithNetwork() {
        if (node != null) {
            node.syncWithNetwork();
        }
    }

    public boolean receiveBlock(Block block) {
        Block latestBlock = blockchain.getLatestBlock();
        
        if (block.getPreviousHash().equals(latestBlock.getHash())) {
            blockchain.getChain().add(block);
            logger.info("接收到新区块并添加到链");
            
            if (node != null) {
                node.broadcastBlock(block);
            }
            return true;
        } else {
            logger.warn("区块链接不匹配，可能需要同步");
            return false;
        }
    }
}

说明：

  • 使用 @Autowired 自动注入 Blockchain 和 Node 实例
  • Wallet 直接实例化，每个服务有自己的钱包
  • 提供节点信息查询、统计信息、交易创建、挖矿、余额查询等方法
  • 所有方法都进行了空值检查，确保在节点未初始化时也能正常工作

4.3 Blockchain 核心逻辑

public class Blockchain {
    private List<Block> chain;
    private List<Transaction> pendingTransactions;
    private int difficulty;
    private double miningReward;

    public void addTransaction(Transaction transaction) {
        // 验证交易地址
        if (transaction.getSenderAddress() == null || 
            transaction.getRecipientAddress() == null) {
            throw new IllegalArgumentException("交易地址不能为空");
        }
        // 验证交易金额
        if (transaction.getAmount() <= 0) {
            throw new IllegalArgumentException("交易金额必须大于0");
        }
        // 添加到待处理队列
        pendingTransactions.add(transaction);
        logger.info("交易已添加到待处理队列: {}", transaction.getTransactionId());
    }
}

5.3 Blockchain 挖矿逻辑

public boolean minePendingTransactions(String minerAddress) {
    if (pendingTransactions.isEmpty()) {
        logger.warn("没有待处理的交易");
        return false;
    }
    
    // 创建新区块
    Block newBlock = new Block(
        chain.size(),                    // 区块索引
        getLatestBlock().getHash(),      // 前一区块哈希
        pendingTransactions              // 打包所有待处理交易
    );
    
    // 执行工作量证明（挖矿）
    newBlock.mineBlock(difficulty);
    
    // 将新区块添加到链上
    chain.add(newBlock);
    logger.info("新区块已添加到链: {}", newBlock.getHash());
    
    // 给矿工发放奖励
    Transaction rewardTransaction = new Transaction("0", minerAddress, miningReward);
    pendingTransactions = new ArrayList<>();  // 清空待处理队列
    pendingTransactions.add(rewardTransaction); // 添加奖励交易
    
    return true;
}

5.4 工作量证明（PoW）详解

public void mineBlock(int difficulty) {
    // 目标：哈希值前 difficulty 位必须为 '0'
    String target = new String(new char[difficulty]).replace('\0', '0');
    
    // 不断尝试不同的 nonce 值
    while (!hash.substring(0, difficulty).equals(target)) {
        nonce++;
        hash = calculateHash();
    }
    
    System.out.println("区块挖掘完成! Hash: " + hash);
}

挖矿流程说明：

  1. 设定目标难度（difficulty = 4，即哈希值前 4 位必须为 '0'）
  2. 从 nonce = 0 开始不断递增尝试
  3. 每次递增后重新计算区块哈希值
  4. 当哈希值满足目标条件时，挖矿完成
  5. 难度越高，需要的计算量越大

6.2 Blockchain 余额计算逻辑

public double getBalance(String address) {
    double balance = 0;
    
    // 遍历区块链上的所有区块
    for (Block block : chain) {
        // 遍历每个区块中的所有交易
        for (Transaction transaction : block.getTransactions()) {
            // 如果该地址是收款方，增加余额
            if (transaction.getRecipientAddress().equals(address)) {
                balance += transaction.getAmount();
            }
            // 如果该地址是付款方，减少余额
            if (transaction.getSenderAddress().equals(address)) {
                balance -= transaction.getAmount();
            }
        }
    }
    
    return balance;
}

余额计算原理：

  • 遍历整条区块链的所有区块
  • 对每个区块中的所有交易进行累加计算
  • 作为收款方时增加相应金额
  • 作为付款方时减少相应金额
  • 初始余额为 0，通过挖矿获得奖励后余额增加

7.2 Blockchain 验证逻辑

public boolean isChainValid() {
    // 从第二个区块开始验证（跳过创世区块）
    for (int i = 1; i < chain.size(); i++) {
        Block currentBlock = chain.get(i);
        Block previousBlock = chain.get(i - 1);
        
        // 验证当前区块的哈希值是否正确
        if (!currentBlock.getHash().equals(currentBlock.calculateHash())) {
            logger.error("区块 {} 的哈希值无效", i);
            return false;
        }
        
        // 验证前一区块哈希值是否匹配
        if (!currentBlock.getPreviousHash().equals(previousBlock.getHash())) {
            logger.error("区块 {} 的前一区块哈希值不匹配", i);
            return false;
        }
    }
    return true;
}

防篡改原理：

  1. 每个区块的哈希值由其内容（索引、前一哈希、时间戳、nonce、交易数据）计算得出
  2. 如果修改某个区块的数据，该区块的哈希值会立即改变
  3. 由于后续区块存储了前一区块的哈希值，前一区块哈希值改变会导致后续所有区块的 previousHash 不匹配
  4. 要成功篡改，需要重新计算被修改区块及其后所有区块的工作量证明，这在计算上几乎不可能完成

8.3 Node 网络节点实现

public class Node {
    private String nodeId;
    private String host;
    private int port;
    private List<String> peerNodes;
    private ConcurrentHashMap<String, NodeInfo> knownNodes;

    public Node(String host, int port) {
        this.nodeId = java.util.UUID.randomUUID().toString();
        this.host = host;
        this.port = port;
        this.peerNodes = new ArrayList<>();
        this.knownNodes = new ConcurrentHashMap<>();
        logger.info("节点创建成功: {}:{}", host, port);
    }

    public void addPeer(String peerAddress) {
        if (!peerNodes.contains(peerAddress)) {
            peerNodes.add(peerAddress);
            logger.info("已添加节点: {}", peerAddress);
        }
    }

    public void removePeer(String peerAddress) {
        peerNodes.remove(peerAddress);
        knownNodes.remove(peerAddress);
        logger.info("已移除节点: {}", peerAddress);
    }

    public List<String> getPeerNodes() { return peerNodes; }
}

public boolean receiveBlock(Block block) {
    Block latestBlock = blockchain.getLatestBlock();
    
    // 验证区块链接关系
    if (block.getPreviousHash().equals(latestBlock.getHash())) {
        blockchain.getChain().add(block);
        logger.info("接收到新区块并添加到链");
        
        // 继续向其他邻居广播
        if (node != null) {
            node.broadcastBlock(block);
        }
        return true;
    } else {
        logger.warn("区块链接不匹配，可能需要同步");
        return false;
    }
}

9.3 Node 同步逻辑

public void syncWithNetwork() {
    logger.info("开始与网络同步...");
    
    for (String peer : peerNodes) {
        // 从邻居节点获取区块链
        List<Block> peerChain = requestChainFromPeer(peer);
        
        if (peerChain != null && peerChain.size() > 0) {
            logger.info("从节点 {} 获取到链，长度: {}", peer, peerChain.size());
            
            // 如果邻居的链更长，则替换当前链
            if (peerChain.size() > chain.size()) {
                // 验证邻居链的有效性
                if (isValidChain(peerChain)) {
                    chain = peerChain;
                    logger.info("已同步更长的有效链");
                }
            }
        }
    }
    logger.info("同步完成");
}

public List<Block> requestChainFromPeer(String peerAddress) {
    try {
        return WebClient.create("http://" + peerAddress)
            .get()
            .uri("/api/blockchain/chain")
            .retrieve()
            .bodyToMono(List.class)
            .block();
    } catch (Exception e) {
        logger.error("从节点 {} 获取链失败: {}", peerAddress, e.getMessage());
        removePeer(peerAddress);
        return null;
    }
}

private boolean isValidChain(List<Block> chainToValidate) {
    for (int i = 1; i < chainToValidate.size(); i++) {
        Block currentBlock = chainToValidate.get(i);
        Block previousBlock = chainToValidate.get(i - 1);
        
        if (!currentBlock.getHash().equals(currentBlock.calculateHash())) {
            return false;
        }
        if (!currentBlock.getPreviousHash().equals(previousBlock.getHash())) {
            return false;
        }
    }
    return true;
}

9.4 同步原理

P2P 网络同步遵循"最长链原则"：

  1. 节点启动时，向所有已知的邻居节点请求区块链数据
  2. 比较各邻居节点的链长度
  3. 如果发现更长的有效链，则用该链替换当前链
  4. 验证新链的完整性（每个区块的哈希值和链接关系）
  5. 同步完成后，节点拥有最新的区块链数据
