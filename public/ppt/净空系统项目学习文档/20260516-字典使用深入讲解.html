📖 第一章：字典概述

字典是 JeecgBoot 框架中的核心功能之一，用于统一管理系统中的各种枚举值、状态码、类型标识等固定选项数据。 通过字典机制，可以实现数据标准化、界面友好化和维护便捷化。

1.1 什么是字典

📋 字典定义

字典（Dictionary） 是一种用于存储键值对映射关系的数据结构，在 JeecgBoot 中特指系统级别的 固定选项数据集合。每个字典包含一个字典定义（sys_dict）和多个字典项（sys_dict_item）。

举例说明： "性别"字典包含两个字典项：{"男": "1", "女": "0"}， 数据库中存储值 "1"，前端显示文本 "男"。

1.2 字典的作用

  • 数据标准化：统一管理系统中的枚举值、状态码等固定选项
  • 界面友好化：数据库存储代码值，前端显示友好文本
  • 维护便捷化：集中管理，一处修改，多处生效
  • 国际化支持：为多语言系统提供基础数据支持
  • 缓存优化：字典数据加载后缓存，减少数据库查询

1.3 应用场景

场景 字典 Code 字典项示例 应用位置
性别 sex 男/女 用户管理、员工信息
状态 status 启用/禁用 所有业务表状态字段
删除标记 del_flag 已删除/未删除 逻辑删除字段
告警级别 alarm_level 一般/严重/紧急 预警信息管理
净空等级 clearance_level 优秀/良好/一般/警告/危险 净空检测报告
不平衡等级 unbalance_level 无影响/轻微/中度/严重/紧急 风机不平衡监测

🗄️ 第二章：数据库表结构

JeecgBoot 的字典系统由两张核心表组成：sys_dict（字典定义表）和 sys_dict_item（字典项表），采用主从表结构存储字典数据。

2.1 sys_dict 表（字典定义表）

sys_dict 表用于存储字典的定义信息，即字典的元数据。

2.1.1 表结构

CREATE TABLE `sys_dict` (
  `id` varchar(32) NOT NULL COMMENT '主键 ID',
  `dict_name` varchar(100) NOT NULL COMMENT '字典名称',
  `dict_code` varchar(100) NOT NULL COMMENT '字典编码',
  `description` varchar(255) DEFAULT NULL COMMENT '描述',
  `del_flag` int(1) DEFAULT NULL COMMENT '删除状态',
  `create_by` varchar(32) DEFAULT NULL COMMENT '创建人',
  `create_time` datetime DEFAULT NULL COMMENT '创建时间',
  `update_by` varchar(32) DEFAULT NULL COMMENT '更新人',
  `update_time` datetime DEFAULT NULL COMMENT '更新时间',
  `type` int(1) UNSIGNED ZEROFILL DEFAULT 0 COMMENT '字典类型 0:string,1:number',
  PRIMARY KEY (`id`),
  UNIQUE INDEX `uk_sd_dict_code` (`dict_code`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='字典表';

2.1.2 字段说明

字段名 类型 长度 必填 说明
id VARCHAR 32 是 主键 ID，使用 ASSIGN_ID 生成策略
dict_name VARCHAR 100 是 字典名称，如"性别"、"状态"
dict_code VARCHAR 100 是 字典编码，如"sex"、"status"，唯一索引
description VARCHAR 255 否 字典描述信息
del_flag INT 1 否 删除状态（0-未删除，1-已删除）
type INT 1 否 字典类型（0-string 类型，1-number 类型）
create_by VARCHAR 32 否 创建人
create_time DATETIME - 否 创建时间
update_by VARCHAR 32 否 更新人
update_time DATETIME - 否 更新时间

2.2 sys_dict_item 表（字典项表）

sys_dict_item 表用于存储字典的具体项，即字典的值列表。

2.2.1 表结构

CREATE TABLE `sys_dict_item` (
  `id` varchar(32) NOT NULL COMMENT '主键 ID',
  `dict_id` varchar(32) DEFAULT NULL COMMENT '字典 ID（外键关联 sys_dict.id）',
  `item_text` varchar(100) NOT NULL COMMENT '字典项文本（显示值）',
  `item_value` varchar(100) NOT NULL COMMENT '字典项值（存储值）',
  `description` varchar(255) DEFAULT NULL COMMENT '描述',
  `sort_order` int(10) DEFAULT NULL COMMENT '排序',
  `status` int(11) DEFAULT NULL COMMENT '状态（1-启用，0-禁用）',
  `create_by` varchar(32) DEFAULT NULL COMMENT '创建人',
  `create_time` datetime DEFAULT NULL COMMENT '创建时间',
  `update_by` varchar(32) DEFAULT NULL COMMENT '更新人',
  `update_time` datetime DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_sdi_role_dict_id` (`dict_id`) USING BTREE,
  INDEX `idx_sdi_role_sort_order` (`sort_order`) USING BTREE,
  INDEX `idx_sdi_status` (`status`) USING BTREE,
  INDEX `idx_sdi_dict_val` (`dict_id`, `item_value`) USING BTREE
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='字典项表';

2.2.2 字段说明

字段名 类型 长度 必填 说明
id VARCHAR 32 是 主键 ID
dict_id VARCHAR 32 是 外键，关联 sys_dict 表的 id 字段
item_text VARCHAR 100 是 字典项文本，前端显示的友好文本
item_value VARCHAR 100 是 字典项值，数据库存储的实际值
description VARCHAR 255 否 字典项描述
sort_order INT 10 否 排序，用于控制字典项的显示顺序
status INT 11 否 状态（1-启用，0-禁用）
create_by VARCHAR 32 否 创建人
create_time DATETIME - 否 创建时间
update_by VARCHAR 32 否 更新人
update_time DATETIME - 否 更新时间

3.3.2 SysDictServiceImpl 实现（核心方法）

package org.jeecg.modules.system.service.impl;

import org.jeecg.common.constant.CacheConstant;
import org.jeecg.modules.system.entity.SysDict;
import org.jeecg.modules.system.entity.SysDictItem;
import org.jeecg.modules.system.mapper.SysDictMapper;
import org.jeecg.modules.system.mapper.SysDictItemMapper;
import org.jeecg.modules.system.model.DictModel;
import org.jeecg.modules.system.service.ISysDictService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.cache.annotation.Cacheable;
import org.springframework.stereotype.Service;

import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;

import java.util.*;
import java.util.stream.Collectors;

/**
 * 字典服务实现类
 */
@Service
public class SysDictServiceImpl extends ServiceImpl<SysDictMapper, SysDict> implements ISysDictService {

    @Autowired
    private SysDictMapper sysDictMapper;

    @Autowired
    private SysDictItemMapper sysDictItemMapper;

    /**
     * 通过查询指定 code 获取字典（带缓存）
     * @param code 字典编码
     * @return 字典项列表
     */
    @Override
    @Cacheable(value = CacheConstant.SYS_DICT_CACHE, key = "#code", unless = "#result == null")
    public List<DictModel> queryDictItemsByCode(String code) {
        log.debug("无缓存 dictCache 的时候调用这里！");
        return sysDictMapper.queryDictItemsByCode(code);
    }

    /**
     * 查询有效的数据字典项（带缓存）
     * @param code 字典编码
     * @return 有效的字典项列表
     */
    @Override
    @Cacheable(value = CacheConstant.SYS_ENABLE_DICT_CACHE, key = "#code", unless = "#result == null")
    public List<DictModel> queryEnableDictItemsByCode(String code) {
        return sysDictMapper.queryEnableDictItemsByCode(code);
    }

    /**
     * 登录加载系统字典（一次性加载所有字典）
     * @return 所有字典数据
     */
    @Override
    public Map<String, List<DictModel>> queryAllDictItems() {
        Map<String, List<DictModel>> res = new HashMap(5);
        List<SysDict> ls = sysDictMapper.selectList(null);
        
        // 查询所有启用的字典项（status=1）
        LambdaQueryWrapper<SysDictItem> queryWrapper = new LambdaQueryWrapper<SysDictItem>();
        queryWrapper.eq(SysDictItem::getStatus, 1);
        queryWrapper.orderByAsc(SysDictItem::getSortOrder);
        List<SysDictItem> sysDictItemList = sysDictItemMapper.selectList(queryWrapper);

        // 按字典 ID 分组
        for (SysDict d : ls) {
            List<DictModel> dictModelList = sysDictItemList.stream()
                .filter(s -> d.getId().equals(s.getDictId()))
                .map(item -> {
                    DictModel dictModel = new DictModel();
                    dictModel.setText(item.getItemText());
                    dictModel.setValue(item.getItemValue());
                    return dictModel;
                }).collect(Collectors.toList());
            res.put(d.getDictCode(), dictModelList);
        }

        log.debug("-------登录加载系统字典-----" + res.toString());
        return res;
    }
}

📝 第四章：@Dict 注解详解

@Dict 注解是 JeecgBoot 字典机制的核心， 用于在实体类字段上标注字典信息，实现自动翻译功能。

4.1 注解定义

package org.jeecg.common.aspect.annotation;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

/**
 * 字典注解
 * @author: dangzhenghui
 * @date: 2019 年 03 月 17 日 - 下午 9:37:16
 */
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Dict {
    /**
     * 数据 code（必填）
     * 用于指定字典编码，如"sex"、"status"
     */
    String dicCode();

    /**
     * 数据 Text（可选）
     * 用于表字典查询时指定显示文本字段
     */
    String dicText() default "";

    /**
     * 数据字典表（可选）
     * 用于指定表字典的表名，支持复杂查询
     */
    String dictTable() default "";
}

📋 注解参数说明

参数 必填 说明 示例
dicCode 是 字典编码，对应 sys_dict.dict_code "sex"、"status"
dicText 否 表字典的显示文本字段 "realname"、"org_name"
dictTable 否 表字典的表名 "sys_user"、"sys_org"

4.3 表字典查询

表字典是一种特殊的字典类型，用于关联其他业务表的数据。 通过 dictTable、dicText、 dicCode 三个参数实现跨表关联查询。

4.3.1 表字典查询流程

┌──────────────┐ │ 前端请求 │ │ /api/list │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ Controller │ │ queryList() │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ Service │ │ page(page, wrapper) │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ MyBatis-Plus │ │ 执行查询 │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ 返回结果 │ │ 包含原始值 │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ Dict 注解处理 │ │ 解析@Dict │ └──────┬───────┘ │ ▼ ┌─────────────────────────────┐ │ 表字典查询 │ │ SELECT name FROM │ │ patrol_fan_scope │ │ WHERE id = region │ └──────┬──────────────────────┘ │ ▼ ┌──────────────┐ │ 翻译结果 │ │ region -> 风场名称 │ └──────┬───────┘ │ ▼ ┌──────────────┐ │ 返回前端 │ │ JSON 数据 │ └──────────────┘

4.3.2 表字典 SQL 示例

-- 原始查询
SELECT
    id,
    fan_no,
    fan_name,
    region,  -- 存储的是风场 ID
    create_by  -- 存储的是用户 username
FROM
    patrol_fan_info

-- 字典翻译后的结果（逻辑上）
SELECT
    a.id,
    a.fan_no,
    a.fan_name,
    b.name AS region_text,  -- 翻译后的风场名称
    c.realname AS create_by_text  -- 翻译后的创建人文名称
FROM
    patrol_fan_info a
LEFT JOIN patrol_fan_scope b ON a.region = b.id
LEFT JOIN sys_user c ON a.create_by = c.username

5.1 JDictSelectUtil 工具类

5.1.1 获取字典数据

/**
 * 通过 code 获取字典数组
 * @param {string} dictCode - 字典编码
 * @param {object} params - 查询参数（可选）
 * @returns {Promise} - 字典数据数组
 */
export async function getDictItems(dictCode, params) {
    // 优先从缓存中读取字典配置
    if (getDictItemsFromCache(dictCode)) {
        let desformDictItems = getDictItemsFromCache(dictCode).map(item => ({
            ...item,
            label: item.text
        }))
        return desformDictItems
    }

    // 缓存中没有，就请求后台
    return await ajaxGetDictItems(dictCode, params).then(({success, result}) => {
        if (success) {
            let res = result.map(item => ({...item, label: item.text}))
            console.log('------- 从 DB 中获取到了字典 -------dictCode : ', dictCode, res)
            return Promise.resolve(res)
        } else {
            console.error('getDictItems error: : ', res)
            return Promise.resolve([])
        }
    }).catch((res) => {
        console.error('getDictItems error: ', res)
        return Promise.resolve([])
    })
}

5.3 字典翻译

前端接收后端返回的数据后，需要根据字典配置将值（value）翻译为文本（text）。

5.3.1 字典翻译方法

/**
 * 字典翻译工具方法
 * @param {string} dictCode - 字典编码
 * @param {string} value - 要翻译的值
 * @param {Array} dictData - 字典数据数组
 * @returns {string} - 翻译后的文本
 */
export function translateDictValue(dictCode, value, dictData) {
    if (!dictData || !Array.isArray(dictData)) {
        return value
    }
    const item = dictData.find(d => d.value === String(value))
    return item ? item.text : value
}

// 使用示例
import { getDictItems, translateDictValue } from '@/components/dict/JDictSelectUtil'

// 在组件中使用
export default {
  methods: {
    async getStatusText(statusValue) {
        const statusDict = await getDictItems('status')
        return translateDictValue('status', statusValue, statusDict)
    }
  }
}

6.3 常见问题

❓ 问题 1：字典数据不生效

现象：实体类字段上添加了 @Dict 注解，但查询结果没有翻译。

原因：

  • JeecgBoot 的字典翻译是在 Controller 层统一处理的，需要继承 JeecgController
  • 如果是自定义的 Service，需要手动调用字典翻译方法
  • 字典编码拼写错误，检查 dicCode 是否与 sys_dict.dict_code 一致

解决方案：确保 Controller 继承 JeecgController，或手动调用翻译方法。

❓ 问题 2：表字典查询失败

现象：使用 @Dict(dictTable="...") 后查询报错。

原因：

  • 表名或字段名拼写错误
  • 关联字段数据类型不匹配
  • 表字典格式不正确（应该是：表名，显示字段，关联字段）

解决方案：检查 dictTable、dicText、dicCode 三个参数的格式和值。

❓ 问题 3：字典缓存不更新

现象：修改了字典数据，但查询结果还是旧值。

原因：字典数据有 Redis 缓存，修改后缓存没有清除。

解决方案：

  • 手动清除 Redis 缓存： redisTemplate.delete(CacheConstant.SYS_DICT_CACHE + ":" + dictCode)
  • 重启服务，缓存会自动重新加载
  • 修改 SysDictServiceImpl 中的缓存策略
💡 最佳实践：
  • 字典编码使用英文命名，遵循蛇形命名法（如 clearance_level）
  • 字典项值使用大写英文或数字（如 EXCELLENT、1）
  • 合理设置 sort_order 控制字典项显示顺序
  • 对于不常变化的字典数据，启用 Redis 缓存优化性能
