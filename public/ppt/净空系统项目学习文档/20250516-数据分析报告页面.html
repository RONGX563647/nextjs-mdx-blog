org.jeecg.modules.fengji.info.controller.DeviceStatsController 是数据分析报告的核心控制器，提供日/周/月/小时统计数据查询接口和分表管理接口。

📖 控制器设计要点

  • 异常处理：try-catch 捕获所有异常，返回友好的错误信息

2.2 服务层 DeviceDailyStatsServiceImpl

服务实现层是业务逻辑的核心，负责统计数据计算、聚合和缓存。

2.2.1 日报统计计算

/**
 * 计算指定日期的日报表统计
 * 从 record_info 分表中聚合数据到 device_daily_stats 表
 */
@Transactional(rollbackFor = Exception.class)
public void calculateDailyStats(Date date) {
    LocalDate localDate = date.toInstant()
        .atZone(ZoneId.systemDefault())
        .toLocalDate();
    
    // 1. 获取所有设备列表
    List<FengdianDeviceInfo> devices = fengdianDeviceInfoService.list(
        new QueryWrapper<FengdianDeviceInfo>().eq("del_flag", "0")
    );

    // 2. 遍历每个设备计算统计
    for (FengdianDeviceInfo device : devices) {
        calculateDeviceDailyStats(device, localDate);
    }

    log.info("日报表统计计算完成，日期：{}", localDate);
}

/**
 * 计算单个设备的日报表统计
 */
private void calculateDeviceDailyStats(FengdianDeviceInfo device, LocalDate date) {
    String deviceNo = device.getDeviceNo();
    
    // 1. 从分表查询当天的所有检测记录
    List<RecordInfo> records = shardingRecordService.queryRecordInfoByEquipmentAndTime(
        deviceNo,
        Date.from(date.atStartOfDay(ZoneId.systemDefault()).toInstant()),
        Date.from(date.plusDays(1).atStartOfDay(ZoneId.systemDefault()).toInstant())
    );

    // 2. 统计计算
    int totalRecords = records.size();
    int abnormalRecords = 0;
    BigDecimal totalClearance = BigDecimal.ZERO;
    BigDecimal minClearance = null;
    BigDecimal maxClearance = null;

    for (RecordInfo record : records) {
        // 累加净空值
        if (record.getClearanceHeight() != null) {
            totalClearance = totalClearance.add(record.getClearanceHeight());
            
            // 更新最小值
            if (minClearance == null || record.getClearanceHeight().compareTo(minClearance) < 0) {
                minClearance = record.getClearanceHeight();
            }
            
            // 更新最大值
            if (maxClearance == null || record.getClearanceHeight().compareTo(maxClearance) > 0) {
                maxClearance = record.getClearanceHeight();
            }
        }
        
        // 统计异常记录
        if (record.getIsAbnormal() != null && record.getIsAbnormal() == 1) {
            abnormalRecords++;
        }
    }

    // 3. 计算平均值
    BigDecimal avgClearance = totalRecords > 0 
        ? totalClearance.divide(BigDecimal.valueOf(totalRecords), 2, RoundingMode.HALF_UP)
        : BigDecimal.ZERO;

    // 4. 获取阈值配置
    ClearanceThresholdVO thresholds = getThresholds();
    
    // 5. 计算净空等级分布
    int excellentCount = 0, goodCount = 0, mediumCount = 0, warningCount = 0, dangerCount = 0;
    
    for (RecordInfo record : records) {
        if (record.getClearanceHeight() != null) {
            int level = calculateUnbalanceLevel(record.getClearanceHeight(), thresholds);
            switch (level) {
                case 1: excellentCount++; break;
                case 2: goodCount++; break;
                case 3: mediumCount++; break;
                case 4: warningCount++; break;
                case 5: dangerCount++; break;
            }
        }
    }

    // 6. 保存或更新日报表记录
    DeviceDailyStats stats = new DeviceDailyStats();
    stats.setEquipmentCode(deviceNo);
    stats.setStatsDate(date);
    stats.setTotalRecords(totalRecords);
    stats.setAbnormalRecords(abnormalRecords);
    stats.setAvgClearanceValue(avgClearance);
    stats.setMinClearanceValue(minClearance);
    stats.setMaxClearanceValue(maxClearance);
    stats.setExcellentCount(excellentCount);
    stats.setGoodCount(goodCount);
    stats.setMediumCount(mediumCount);
    stats.setWarningCount(warningCount);
    stats.setDangerCount(dangerCount);
    
    deviceDailyStatsService.saveOrUpdate(stats);
}
💡 关键设计点：
  • 事务支持：使用 @Transactional 确保数据一致性
  • 分表查询：根据日期自动路由到对应的 record_info 分表
  • 预计算字段：excellentCount 等字段避免查询时重复计算
  • 空值处理：min/max 初始为 null，避免空指针异常

2.2.2 小时级统计查询

/**
 * 获取设备小时统计列表（按小时聚合）
 * @return 0-23 点的统计数据列表
 */
public List<DeviceDailyStats> getHourlyStats(String deviceNo, String windFarmCode, Date date) {
    LocalDate localDate = date.toInstant()
        .atZone(ZoneId.systemDefault())
        .toLocalDate();
    
    // 查询指定日期的所有小时统计
    LambdaQueryWrapper<DeviceDailyStats> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(DeviceDailyStats::getStatsDate, localDate);
    queryWrapper.eq(DeviceDailyStats::getStatsHour, hour); // 0-23
    
    if (deviceNo != null && !deviceNo.isEmpty()) {
        queryWrapper.eq(DeviceDailyStats::getEquipmentCode, deviceNo);
    }
    
    if (windFarmCode != null && !windFarmCode.isEmpty()) {
        // 需要通过关联查询获取风场下的设备
        List<String> deviceNos = getDeviceNosByWindFarm(windFarmCode);
        queryWrapper.in(DeviceDailyStats::getEquipmentCode, deviceNos);
    }
    
    queryWrapper.orderByAsc(DeviceDailyStats::getStatsHour);
    
    return deviceDailyStatsMapper.selectList(queryWrapper);
}

2.4 分表管理策略

record_info_YYYYMMDD     # 检测记录分表

mqtt_log_20250516        # 2025 年 5 月 16 日的 MQTT 日志

2.4.2 分表管理接口

/**
 * 删除指定日期之前的分表
 * URL: DELETE /api/stats/sharding/clean
 * 参数：daysToKeep (保留天数，默认 30 天)
 */
@DeleteMapping("/sharding/clean")
public Result<Map<String, Object>> cleanShardingTables(
        @RequestParam(defaultValue = "30") int daysToKeep) {
    try {
        LocalDate beforeDate = LocalDate.now().minusDays(daysToKeep);
        
        int recordInfoDeleted = shardingRecordService.deleteRecordInfoTablesBeforeDate(beforeDate);
        int mqttLogDeleted = shardingRecordService.deleteMqttLogTablesBeforeDate(beforeDate);
        
        Map<String, Object> result = new HashMap<>();
        result.put("recordInfoTablesDeleted", recordInfoDeleted);
        result.put("mqttLogTablesDeleted", mqttLogDeleted);
        result.put("beforeDate", beforeDate.toString());
        
        return Result.OK(result);
    } catch (Exception e) {
        log.error("清理分表失败", e);
        return Result.error("清理失败");
    }
}

/**
 * 预创建未来几天的分表
 * URL: POST /api/stats/sharding/precreate
 * 参数：days (未来天数，默认 7 天)
 */

public Result<String> preCreateTables(@RequestParam(defaultValue = "7") int days) {
    try {

    } catch (Exception e) {
        log.error("预创建分表失败", e);
        return Result.error("预创建失败");
    }
}
📌 分表管理要点：
  • 自动分表：系统根据检测时间自动路由到对应的分表
  • 定期清理：保留最近 30 天数据，避免数据库膨胀
  • 预创建表：提前创建未来 7 天的分表，避免写入失败

      if (warningCount && warningCount.count > 0) {
        problems.push(`有${warningCount.count}台风机处于警告等级，需要重点关注`)
      }
      
      if (problems.length === 0) {
        problems.push('当前无明显问题，继续保持良好监测')
      }
      
      return problems
    },

     * 自动生成改进建议
     */
    improvementSuggestions() {

第五章：核心功能详解

5.1 日报统计计算

5.1.1 计算时机

  • 定时任务：每天凌晨 00:05 自动计算前一天的日报表
  • 手动触发：通过 /api/stats/calculate 接口手动计算指定日期
  • 实时计算：每小时自动计算当前小时的统计（可选）

5.1.2 计算流程

1. 获取所有设备列表
   └─ SELECT * FROM fengdian_device_info WHERE del_flag = '0'

2. 遍历每个设备
   └─ for each device in devices:

3.   从分表查询当天记录
     └─ SELECT * FROM record_info_YYYYMMDD 
        WHERE equipment_code = ? AND detection_time >= ? AND detection_time < ?

4.   统计计算
     ├─ totalRecords = count(records)
     ├─ abnormalRecords = count(records WHERE is_abnormal = 1)
     ├─ avgClearanceValue = AVG(clearance_height)
     ├─ minClearanceValue = MIN(clearance_height)
     ├─ maxClearanceValue = MAX(clearance_height)
     └─ excellentCount/goodCount/mediumCount/warningCount/dangerCount

5.   保存或更新日报表
     └─ INSERT INTO device_daily_stats (...) VALUES (...)
        ON DUPLICATE KEY UPDATE ...

5.2 周报/月报统计

5.2.1 周报统计

/**
 * 获取周报统计
 * @param year 年份
 * @param weekOfYear 周数 (1-52)
 */
public Map<String, Object> getWeeklyStats(String deviceNo, int year, int weekOfYear) {
    // 1. 计算该周的起止日期
    LocalDate firstDayOfWeek = LocalDate.of(year, 1, 1)
        .with(WeekFields.of(Locale.CHINA).weekOfWeekBasedYear(), weekOfYear)
        .with(WeekFields.of(Locale.CHINA).dayOfWeek(), 1);
    
    LocalDate lastDayOfWeek = firstDayOfWeek.plusDays(6);
    
    // 2. 查询该周的所有日报表
    List<DeviceDailyStats> dailyStats = deviceDailyStatsService.getDailyStats(
        deviceNo, null,
        Date.from(firstDayOfWeek.atStartOfDay(ZoneId.systemDefault()).toInstant()),
        Date.from(lastDayOfWeek.plusDays(1).atStartOfDay(ZoneId.systemDefault()).toInstant())
    );
    
    // 3. 汇总计算
    Map<String, Object> weeklyStats = new HashMap<>();
    weeklyStats.put("totalRecords", dailyStats.stream()
        .mapToInt(s -> s.getTotalRecords() != null ? s.getTotalRecords() : 0)
        .sum());
    
    weeklyStats.put("avgClearance", dailyStats.stream()
        .filter(s -> s.getAvgClearanceValue() != null)
        .mapToDouble(s -> s.getAvgClearanceValue().doubleValue())
        .average().orElse(0.0));
    
    // ... 其他统计字段
    
    return weeklyStats;
}

convertHourlyTrendData(statsList) {
  return statsList.map(item => ({
    time: `${String(item.statsHour).padStart(2, '0')}:00`,
    value: item.avgClearanceValue,
    min: item.minClearanceValue,
    max: item.maxClearanceValue
  }))
}

5.4 净空等级自动计算

5.4.1 等级计算公式

/**
 * 根据净空值和阈值配置计算等级
 * @return 1=优秀，2=良好，3=一般，4=警告，5=危险
 */
private int calculateUnbalanceLevel(BigDecimal clearance, ClearanceThresholdVO thresholds) {
    double value = clearance.doubleValue();
    
    if (value >= thresholds.getExcellent()) {
        return 1;  // 优秀
    } else if (value >= thresholds.getGood()) {
        return 2;  // 良好
    } else if (value >= thresholds.getMedium()) {
        return 3;  // 一般
    } else if (value >= thresholds.getWarning()) {
        return 4;  // 警告
    } else {
        return 5;  // 危险
    }
}

6.2 Redis 缓存优化

6.2.1 缓存策略

/**
 * 使用 Redis 缓存小时统计数据
 * Key 格式：stats:hourly:{deviceNo}:{date}:{hour}
 * TTL: 24 小时
 */
public DeviceDailyStats getHourlyStatsFromRedis(String deviceNo, Date date, int hour) {
    String key = String.format("stats:hourly:%s:%tF:%d", deviceNo, date, hour);
    String json = redisTemplate.opsForValue().get(key);
    
    if (json != null) {
        return JSON.parseObject(json, DeviceDailyStats.class);
    }
    
    // 缓存未命中，从数据库查询
    DeviceDailyStats stats = getByDeviceAndDate(deviceNo, date, hour);
    
    // 写入缓存
    if (stats != null) {
        redisTemplate.opsForValue().set(key, 
            JSON.toJSONString(stats), 
            24, TimeUnit.HOURS
        );
    }
    
    return stats;
}

6.2.2 缓存更新时机

  • 定时计算：每小时结束后 5 分钟，计算并缓存该小时的统计
  • 手动刷新：调用 /api/stats/calculate 后更新缓存
  • 缓存失效：24 小时后自动过期

6.3.1 导出 PDF

/**
 * 导出报告为 PDF
 * 使用 html2canvas + jsPDF 实现
 */
async exportReport() {
  const element = this.$refs.reportContent
  const canvas = await html2canvas(element, {
    scale: 2,
    useCORS: true
  })
  
  const imgData = canvas.toDataURL('image/png')
  const pdf = new jsPDF('p', 'mm', 'a4')
  const imgWidth = 210
  const imgHeight = (canvas.height * imgWidth) / canvas.width
  
  pdf.addImage(imgData, 'PNG', 0, 0, imgWidth, imgHeight)
  pdf.save(`净空检测报告_${this.dateRange[0].format('YYYYMMDD')}.pdf`)
}
