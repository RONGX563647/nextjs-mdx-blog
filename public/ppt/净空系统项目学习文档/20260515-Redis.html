第一章：Redis在项目中的定位

1.1 为什么需要Redis

风电净空监控系统是一个典型的 物联网数据采集与分析系统，具有以下业务特点：

📖 项目业务特点与数据规模
数据类型 频率/规模 特点 Redis应用场景
设备状态上报 50-100台设备，每秒多次 高频、实时性要求高 状态缓存（防抖）
净空检测记录 每台设备500-2000条/天 大量写入、需统计 Hash统计（Lua脚本）
最新识别结果 前端轮询查询 实时性要求极高 String缓存（60秒）
趋势图表数据 按时间排序查询 有序、范围查询 ZSet缓存（1000条）
系统日志 每次操作产生日志 异步写入、批量处理 Stream缓冲队列
字典数据 变更频率低、查询频繁 读多写少 @Cacheable缓存

如果所有数据都直接读写MySQL，会遇到以下严重问题：

  • 数据库连接池耗尽：50台设备每秒上报状态，占用大量连接
  • 查询性能差：实时数据查询延迟高，前端轮询卡顿
  • 写入瓶颈：统计数据频繁更新导致行锁竞争
  • 日志写入压力大：系统日志直接写库影响业务性能

1.2 Redis vs MySQL对比

Redis作为内存数据库，在项目中的定位是 缓存层 + 消息队列 + 分布式协调：

对比维度 MySQL Redis 本项目使用策略
存储介质 磁盘（SSD/HDD） 内存 热数据用Redis，冷数据用MySQL
读取延迟 1-10ms <0.1ms（快100倍） 实时状态查询走Redis
写入性能 数百TPS 数万TPS（快100倍） 高频状态更新走Redis
数据结构 表结构（关系型） String/Hash/ZSet/Stream 根据场景选择合适类型
持久化 强持久化（WAL） 可选RDB/AOF 缓存允许丢失，日志用Stream持久化
原子操作 事务（ACID） Lua脚本（原子执行） 统计更新使用Lua脚本
分布式支持 主从/分库分表 集群/哨兵/Redisson 单机Redis + Redisson分布式锁

1.3 项目Redis架构概览

本项目Redis架构分为 四个层次，各层职责清晰：

┌─────────────────────────────────────────────────────────┐
│                    应用层 (Application)                    │
│  MqttStatusManager | DeviceDailyStats | LogBufferService  │

└────────────────────┬────────────────────────────────────┘
                     │ 调用

└────────────────────┬────────────────────────────────────┘
                     │ 依赖

💡 架构设计原则：
  • 分层清晰：应用层不直接操作连接，通过框架层封装
  • 配置集中：所有Redis配置集中在application-dev.yml和RedissonConfig
  • 类型安全：使用StringRedisTemplate避免序列化问题
  • 降级容错：Redis不可用时自动降级到数据库（三级降级）

第二章：Redis配置与连接

2.1 application-dev.yml配置

项目的Redis配置集中在 jeecg-boot-module-system/src/main/resources/application-dev.yml 文件中： 

# ============================================
# Redis 配置（开发环境）
# ============================================
spring:
  redis:
    # 数据库编号：使用DB 3，与其他业务隔离
    # Redis默认有16个数据库（DB 0-15）
    database: 3
    
    # Redis服务器地址（开发环境本地）
    host: 127.0.0.1
    port: 6379
    
    # 密码（开发环境为空字符串）
    # ⚠️ 注意：空字符串会导致Redisson自动配置AUTH失败
    password: ""
    
    # ============================================
    # Lettuce 连接池配置
    # Lettuce基于Netty，线程安全，支持异步
    # ============================================
    lettuce:
      pool:
        # 最大连接数：8个连接足够应对当前并发
        # 计算公式：(核心数 * 2) + 有效磁盘数
        # 当前并发不高，8个足够；高并发场景可设为20-50
        max-active: 8
        
        # 最大空闲连接：保持8个空闲连接，避免频繁创建
        # 与max-active保持一致，避免连接频繁创建销毁
        max-idle: 8
        
        # 最小空闲连接：0表示允许全部回收
        # 节省资源，低峰期自动回收
        min-idle: 0
        
        # 获取连接最大等待时间：-1表示无限等待
        # ⚠️ 建议设为3000ms避免线程长时间阻塞
        max-wait: -1ms
        
        # 关闭超时时间
        shutdown-timeout: 100ms
  
  # ============================================
  # Redisson 配置（用于分布式锁和Session存储）
  # Redisson提供分布式锁、MapCache、Session管理等
  # ============================================
  redisson:
    config: |
      singleServerConfig:
        address: "redis://127.0.0.1:6379"
        database: 3
        # 连接池大小
        connectionPoolSize: 8
        # 最小空闲连接数
        connectionMinimumIdleSize: 5
        # 空闲连接超时时间（毫秒）
        idleConnectionTimeout: 10000
        # 连接超时时间（毫秒）
        connectTimeout: 10000
📌 为什么使用Lettuce而不是Jedis？
  • 线程安全：Lettuce基于Netty，连接实例可线程安全共享；Jedis连接非线程安全，需要连接池
  • 异步支持：Lettuce原生支持异步和响应式编程（Reactive Streams）
  • 连接池优化：Lettuce的连接池基于Apache Commons Pool2，性能更好
  • Spring Boot推荐：Spring Boot 2.x默认使用Lettuce作为Redis客户端
  • 资源占用低：Lettuce单个连接即可支持高并发，Jedis需要多个连接

2.2 RedissonConfig配置类源码

项目自定义了 RedissonConfig.java 来手动配置Redisson客户端，避免自动配置的空密码问题： 

package org.jeecg.config;

import lombok.extern.slf4j.Slf4j;
import org.redisson.Redisson;
import org.redisson.api.RedissonClient;
import org.redisson.config.Config;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.util.StringUtils;

import java.io.IOException;

/**
 * Redisson手动配置，替代RedissonAutoConfiguration
 * 
 * ⚠️ 为什么需要手动配置？
 * Spring Boot自动配置在 spring.redis.password=""（空字符串）时，
 * 会尝试执行 AUTH "" 命令，导致Redis拒绝连接。
 * 手动配置可以判断密码是否为空，避免此问题。
 * 
 * @author fengji-system
 */
@Slf4j
@Configuration
public class RedissonConfig {

    // ============================================
    // 配置属性注入
    // ============================================
    
    // Redisson YAML配置（支持集群、哨兵等高级场景）
    @Value("${spring.redisson.config:}")
    private String configYaml;

    // Redis服务器地址
    @Value("${spring.redis.host:127.0.0.1}")
    private String redisHost;

    // Redis端口
    @Value("${spring.redis.port:6379}")
    private int redisPort;

    // Redis数据库编号
    @Value("${spring.redis.database:0}")
    private int redisDatabase;

    // Redis密码（可能为空字符串）
    @Value("${spring.redis.password:}")
    private String redisPassword;

    /**
     * 创建RedissonClient Bean
     * 
     * destroyMethod = "shutdown" 确保应用关闭时正确释放资源
     * 包括：关闭连接、释放线程池、清理缓存等
     * 
     * @return RedissonClient实例
     */
    @Bean(destroyMethod = "shutdown")
    public RedissonClient redissonClient() {
        // ============================================
        // 分支1：未配置YAML，使用默认单机配置
        // ============================================
        if (!StringUtils.hasText(configYaml)) {
            log.warn("spring.redisson.config 未配置，使用默认单机配置");
            
            Config config = new Config();
            config.useSingleServer()
                    .setAddress("redis://" + redisHost + ":" + redisPort)
                    .setDatabase(redisDatabase);
            
            // ⚠️ 关键：只有密码非空时才设置，避免空字符串导致AUTH失败
            if (redisPassword != null && !redisPassword.isEmpty()) {
                config.useSingleServer().setPassword(redisPassword);
            }
            
            return Redisson.create(config);
        }

        // ============================================
        // 分支2：使用YAML配置（支持集群、哨兵等高级场景）
        // ============================================
        log.info("加载 Redisson 配置，配置内容:\n{}", configYaml);
        try {
            // 从YAML解析配置
            Config config = Config.fromYAML(configYaml);
            RedissonClient client = Redisson.create(config);
            log.info("RedissonClient 初始化成功");
            return client;
        } catch (IOException e) {
            // ============================================
            // 降级：YAML解析失败，使用默认配置
            // ============================================
            log.error("Redisson YAML 配置解析失败，使用默认配置：{}", e.getMessage());
            
            Config config = new Config();
            config.useSingleServer()
                    .setAddress("redis://" + redisHost + ":" + redisPort)
                    .setDatabase(redisDatabase);
            if (redisPassword != null && !redisPassword.isEmpty()) {
                config.useSingleServer().setPassword(redisPassword);
            }
            return Redisson.create(config);
        }
    }
}
⚠️ 为什么需要手动配置Redisson？

Spring Boot自动配置在 spring.redis.password=""（空字符串）时，会尝试执行 AUTH "" 命令，导致Redis拒绝连接并抛出 ERR Client sent AUTH, but no password is set 错误。手动配置可以判断密码是否为空，避免此问题。

2.3 Lettuce连接池参数详解

参数 当前值 说明 调优建议
max-active 8 最大连接数 当前并发不高，8个足够；高并发场景可设为20-50
max-idle 8 最大空闲连接 与max-active保持一致，避免频繁创建销毁
min-idle 0 最小空闲连接 设为0允许全部回收，节省资源；高并发可设为5
max-wait -1ms 获取连接最大等待时间 -1表示无限等待；建议设为3000ms避免线程阻塞
shutdown-timeout 100ms 关闭超时 默认值即可

2.4 数据库选择与隔离

Redis默认有16个数据库（DB 0-15），本项目使用 DB 3：

📖 数据库隔离策略
数据库 用途 本项目是否使用
DB 0 默认数据库 否

第三章：Redis使用场景

3.1 设备状态缓存（MqttStatusManager）

MqttStatusManager.java 使用Redis缓存设备状态，避免频繁查询数据库，核心源码如下： 

package org.jeecg.modules.mqtt;

import org.jeecg.modules.fengji.common.DeviceStatusConstants;
import org.jeecg.modules.fengji.info.entity.FengdianDeviceInfo;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Component;
import java.util.concurrent.TimeUnit;

/**
 * MQTT设备状态管理器
 * 负责处理设备状态更新、Redis缓存和最后在线时间管理
 * 用于打破MqttClientManager和MqttMessageHandler之间的循环依赖
 * 
 * 核心功能：
 * 1. 设备状态缓存（防抖机制）
 * 2. 设备信息缓存
 * 3. 最后在线时间缓存
 * 4. 设备检查标识管理
 */
@Component
@ConditionalOnProperty(name = "mqtt.enabled", havingValue = "true")
public class MqttStatusManager {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private IFengdianDeviceInfoService fengdianDeviceInfoService;

    /**
     * 处理设备状态更新
     * 
     * ⚠️ 核心设计：防抖机制
     * 通过比较Redis缓存中的状态，避免相同状态重复写入数据库
     * 50台设备每秒上报多次，防抖可减少90%以上的数据库写入
     * 
     * @param deviceNo 设备编号
     * @param status 状态值（"0"离线/"1"在线/"2"故障）
     * @param statusType 状态类型（online/running/camera）
     * @return true-需要更新数据库，false-状态未变更，跳过
     */
    public boolean handleDeviceStatusUpdate(String deviceNo, String status, String statusType) {
        try {
            // ============================================
            // Redis Key格式：mqtt:device:status:{deviceNo}:{statusType}
            // 示例：mqtt:device:status:DEVICE001:online
            //      mqtt:device:status:DEVICE001:running
            //      mqtt:device:status:DEVICE001:camera
            // ============================================
            String redisKey = DeviceStatusConstants.REDIS_KEY_DEVICE_STATUS + deviceNo + ":" + statusType;

            // ============================================
            // 步骤1：检查Redis中是否存在相同状态（防抖）
            // ============================================
            String cachedStatus = redisTemplate.opsForValue().get(redisKey);
            if (status.equals(cachedStatus)) {
                // 状态未变更，跳过数据库更新
                logger.debug("Device status unchanged, skipping update - DeviceNo: {}, StatusType: {}, Status: {}", 
                    deviceNo, statusType, status);
                return false;
            }

            // ============================================
            // 步骤2：更新Redis缓存，设置过期时间为1小时
            // ============================================
            redisTemplate.opsForValue().set(redisKey, status, 
                DeviceStatusConstants.DEVICE_STATUS_CACHE_EXPIRE / 1000 / 60, TimeUnit.MINUTES);

            // ============================================
            // 步骤3：更新最后在线时间（24小时过期）
            // ============================================
            String lastOnlineKey = DeviceStatusConstants.REDIS_KEY_DEVICE_LAST_ONLINE + deviceNo;
            redisTemplate.opsForValue().set(lastOnlineKey, String.valueOf(System.currentTimeMillis()), 24, TimeUnit.HOURS);

            // ============================================
            // 步骤4：设置设备检查标识（用于定时任务）
            // ============================================
            FengdianDeviceInfo deviceInfo = getDeviceInfo(deviceNo);
            if (deviceInfo != null) {
                setDeviceCheckFlag(deviceInfo.getId());
            }

            logger.info("Device status updated - DeviceNo: {}, StatusType: {}, Status: {}", deviceNo, statusType, status);
            return true; // 需要更新数据库
        } catch (Exception e) {
            logger.error("Error handling device status update", e);
            // ⚠️ 降级策略：Redis操作失败时，仍然允许更新数据库
            return true;
        }
    }

    /**
     * 获取设备基础信息（从Redis缓存或数据库）
     * 
     * 缓存策略：Cache-Aside（旁路缓存）
     * 1. 先读Redis缓存
     * 2. 缓存命中则直接返回
     * 3. 缓存未命中则查数据库，并写入缓存
     * 
     * @param deviceNo 设备编号
     * @return 设备信息对象
     */
    public FengdianDeviceInfo getDeviceInfo(String deviceNo) {
        try {
            // Redis Key格式：mqtt:device:info:{deviceNo}
            // 缓存值为设备ID（主键），用于快速查询
            String cacheKey = DeviceStatusConstants.REDIS_KEY_DEVICE_INFO + deviceNo;
            String cachedInfo = redisTemplate.opsForValue().get(cacheKey);
            
            if (cachedInfo != null) {
                // 缓存命中：根据缓存的设备ID查询数据库
                FengdianDeviceInfo deviceInfo = fengdianDeviceInfoService.getById(cachedInfo);
                if (deviceInfo != null) {
                    return deviceInfo;
                }
            }
            
            // 缓存未命中：从数据库查询
            FengdianDeviceInfo deviceInfo = fengdianDeviceInfoService.lambdaQuery()
                    .eq(FengdianDeviceInfo::getDeviceNo, deviceNo)
                    .one();
            
            if (deviceInfo != null) {
                // 写入缓存，过期时间1小时
                redisTemplate.opsForValue().set(cacheKey, deviceInfo.getId(), 
                    DeviceStatusConstants.DEVICE_INFO_CACHE_EXPIRE / 1000 / 60, TimeUnit.MINUTES);
            }
            
            return deviceInfo;
        } catch (Exception e) {
            logger.error("Error getting device info", e);
            return null;
        }
    }
}

🔍 设计亮点分析

设计点 实现方式 效果
防抖机制 比较缓存状态，相同则跳过 减少90%以上数据库写入
过期时间合理 状态缓存1小时，在线时间24小时 符合业务变化频率
降级容错 Redis异常时返回true 保证业务不中断
Cache-Aside 先读缓存，未命中查库并写缓存 实现简单，缓存失败不影响业务

3.2 设备信息缓存

设备基础信息（设备编号、IP地址等）变更频率低，适合长时间缓存：

// Redis Key格式：mqtt:device:info:{deviceNo}
// 缓存值：设备ID（主键）
// 过期时间：1小时（DEVICE_INFO_CACHE_EXPIRE = 3600000ms）
public FengdianDeviceInfo getDeviceInfo(String deviceNo) {
    String cacheKey = DeviceStatusConstants.REDIS_KEY_DEVICE_INFO + deviceNo;
    String cachedInfo = redisTemplate.opsForValue().get(cacheKey);
    
    if (cachedInfo != null) {
        // 缓存命中：根据设备ID查询
        return fengdianDeviceInfoService.getById(cachedInfo);
    }
    
    // 缓存未命中：查库并写缓存
    FengdianDeviceInfo deviceInfo = fengdianDeviceInfoService.lambdaQuery()
            .eq(FengdianDeviceInfo::getDeviceNo, deviceNo)
            .one();
    
    if (deviceInfo != null) {
        redisTemplate.opsForValue().set(cacheKey, deviceInfo.getId(), 60, TimeUnit.MINUTES);
    }
    
    return deviceInfo;
}

3.3 最后在线时间缓存

设备最后在线时间更新频繁（每30秒一次），使用Redis节流避免频繁写库： 

/**
 * 更新设备最后在线时间（使用Redis缓存优化）
 * 
 * ⚠️ 核心设计：30秒节流
 * 只有间隔超过30秒才更新数据库，否则只更新Redis
 * 将每秒多次的写操作降为每30秒一次
 */
public void updateDeviceLastOnline(String deviceNo) {
    try {
        FengdianDeviceInfo deviceInfo = getDeviceInfo(deviceNo);
        if (deviceInfo == null) return;
        
        String lastOnlineKey = DeviceStatusConstants.REDIS_KEY_DEVICE_LAST_ONLINE + deviceNo;
        String lastOnlineStr = redisTemplate.opsForValue().get(lastOnlineKey);
        long currentTime = System.currentTimeMillis();
        
        // ============================================
        // 节流判断：间隔超过30秒才更新数据库
        // LAST_ONLINE_INTERVAL = 30 * 1000ms = 30秒
        // ============================================
        if (lastOnlineStr == null || 
            (currentTime - Long.parseLong(lastOnlineStr) > DeviceStatusConstants.LAST_ONLINE_INTERVAL)) {
            
            // 更新数据库（仅更新必要字段）
            fengdianDeviceInfoService.lambdaUpdate()
                    .eq(FengdianDeviceInfo::getId, deviceInfo.getId())
                    .set(FengdianDeviceInfo::getLastTime, new java.util.Date())
                    .set(FengdianDeviceInfo::getRunStatus, DeviceStatusConstants.STATUS_ONLINE)
                    .update();
            
            // 更新Redis缓存（24小时过期）
            redisTemplate.opsForValue().set(lastOnlineKey, String.valueOf(currentTime), 24, TimeUnit.HOURS);
            
            logger.info("Device last online time updated - DeviceNo: {}", deviceNo);
        } else {
            // 仅更新Redis，不写数据库
            redisTemplate.opsForValue().set(lastOnlineKey, String.valueOf(currentTime), 24, TimeUnit.HOURS);
            logger.debug("Device last online time updated in cache only - DeviceNo: {}", deviceNo);
        }
    } catch (Exception e) {
        logger.error("Error updating device last online time", e);
    }
}

  • 降低锁竞争：减少行锁持有时间，提升并发性能
  • 实时性保证：Redis缓存保证最新时间始终可用，前端查询无延迟
